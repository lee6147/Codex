요약

이번 공식 업데이트 흐름은 모델 성능 경쟁보다 운영 안전성과 협업 표면 확장에 가깝다. Claude Code는 세션 간 권한 위임 차단, fallback 모델, deny-rule glob 같은 런타임 안정 장치를 강화했고, Cursor는 Design Mode와 SDK 개선으로 UI/Canvas 기반 피드백을 더 구조화했다. OpenAI Codex는 2026-06-02 공식 발표와 Enterprise/Edu 릴리스 노트에서 역할별 플러그인, Sites, Annotations, 플러그인 공유를 전면에 배치했다.

Claude Code

2.1.166: fallback 모델, 권한 규칙, 세션 간 메시징 하드닝

확인 완료공식 changelog2026-06-06

Anthropic의 Claude Code changelog는 2.1.166에서 최대 세 개의 fallback 모델 설정, deny rule tool-name 위치의 glob 패턴, cross-session SendMessage 권한 하드닝, thinking 비활성화 옵션, API 오류 시 fallback retry를 명시했다. 2.1.167과 2.1.168도 같은 날 게시됐지만 세부 항목은 “bug fixes and reliability improvements” 수준으로만 공개됐다.

실무 의미: 장시간/다중 세션 작업에서 모델 장애, 권한 위임, 원격 세션 정체 같은 운영 리스크를 줄이는 방향이다.

Cursor

3.7: Design Mode Improvements

확인 완료공식 changelog2026-06-05

Cursor changelog는 3.7을 2026-06-05 항목으로 노출하며 Design Mode Improvements를 게시했다. Cursor browser 안의 Design Mode에서 UI 변경을 클릭, 그리기, 음성으로 지시할 수 있고, multi-select 요소와 voice input으로 에이전트에게 더 명확한 시각적 편집 맥락을 전달하는 흐름이다.

실무 의미: frontend와 내부 도구 리뷰에서 “어느 요소를 어떻게 바꿀지”를 자연어만으로 설명하는 부담이 줄어든다.

OpenAI Codex

Codex for every role, tool, and workflow

3일 창 밖 공식 맥락OpenAI 공식2026-06-02
