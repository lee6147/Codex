최신 확인 릴리스는 2026-04-23의 v2.1.119

GitHub Releases 2026-04-23 창 밖 참고

v2.1.119는 /config 영속화, PowerShell tool 자동 승인, MCP 서버 병렬 재연결, worktree 관련 수정, Windows 경고 정리처럼 실제 사용 마찰을 줄이는 항목이 많았다.

2026-04-23 품질 회고는 아직도 읽어야 하는 기준 문서

Anthropic Engineering 2026-04-23 운영 회고

Anthropic은 Claude Code 품질 저하를 세 가지 원인으로 분리해 설명했다. 기본 effort 하향, 오래된 reasoning 정리 버그, 과도한 간결화 프롬프트가 각기 다른 사용자 집합에 영향을 줬다는 구조다.

Cursor

2026-04-25 ~ 2026-04-28 공식 신규 항목 없음

Cursor changelog를 2026-04-28 기준으로 다시 확인했지만, 이번 창 안에 2026-04-25 이후 새 공식 changelog 항목은 잡히지 않았다.

따라서 Cursor는 오늘도 여전히 2026-04-24의 3.2와 2026-04-21 안정성 글이 제품 방향을 설명하는 핵심 기준이다. 제품 추진력은 강하지만, 이번 창에서는 후속 공식 보강이 없었다.

확인 상태: 확인 완료 | 출처: Cursor Changelog

최신 제품 신호는 2026-04-24의 Cursor 3.2

공식 changelog 2026-04-24 창 밖 참고

Cursor 3.2는 /multitask로 비동기 subagent를 병렬 실행하고, 개선된 worktrees 흐름과 multi-root workspaces를 묶었다. 이는 Cursor가 단순 IDE 보조를 넘어 멀티레포 작업 콘솔로 올라가고 있다는 가장 분명한 공식 신호다.

실사용 관점에서는 "한 프로젝트 안에서 고치기"보다 "여러 분기와 여러 폴더를 동시에 조정하기"가 전면으로 올라왔다. 오늘 창에는 새 항목이 없지만, 여전히 가장 큰 방향 전환 카드다.

안정성 엔지니어링 글도 여전히 유효한 참고 자료

Cursor Research 2026-04-21 기술 글

Cursor는 앱 안정성 글에서 OOM-per-session 80% 감소, OOM-per-request 73% 감소를 공개했다. Agents Window가 커질수록 메모리와 충돌률을 제품 경쟁력의 일부로 다뤄야 한다는 메시지다.

이는 3.2의 병렬성과도 연결된다. 기능 확장이 계속될수록 "얼마나 똑똑한가"만큼 "얼마나 덜 죽는가"가 중요하다는 점을 분명히 보여준다.

OpenAI Codex

Codex rate card가 2026-04-28 확인 시 다시 갱신 상태

OpenAI Help Center 2026-04-28 확인 가격 구조

2026-04-28 확인 시 Codex rate card 문서는 Updated: 2 hours ago로 표시됐다. 이 문서는 Codex 가격 체계가 메시지당 과금이 아니라 토큰 기반 크레딧 구조라는 점을 다시 못박고, Enterprise, Edu, Health, Gov까지 새 기준을 적용 범위에 넣고 있다.

OpenAI Help Center Updated: 2 days ago 접근 권한

Using Codex with your ChatGPT plan 문서는 2026-04-28 확인 시 최근 갱신 상태였고, Plus, Pro, Business, Enterprise/Edu 기본 포함, 한시적 Free/Go 포함, 일부 2x rate limits, Codex app/CLI/IDE/web 진입 경로를 한 문서에서 묶어 설명한다.
