## 1. Claude Code

### 강점

- 깊은 CLI 작업, 긴 코드베이스 탐색, 셸 중심 실행 흐름에서 여전히 가장 강한 축에 있다.
- 최신 확인 릴리스인 `v2.1.119`는 `/config` 영속화, PowerShell tool 자동 승인, MCP 병렬 재연결, worktree 보정처럼 실제 사용 마찰을 줄이는 수정이 많다.
- 2026-04-23 품질 회고가 매우 구체적이라, 제품이 문제를 숨기기보다 원인 분해와 운영 교정으로 대응한다는 신뢰를 준다.

### 약점

- 최근 3일 공식 창 안의 신규 발표가 없어서 이번 주에는 제품 모멘텀이 조용했다.
- 품질 저하 이슈가 모델 열화가 아니라 제품 레이어의 판단과 버그에서 발생했다는 점은, 하네스와 설정 변화가 체감 성능에 크게 작용한다는 뜻이다.
- IDE 다중 표면이나 시각적 제어보다 여전히 터미널 중심 흐름에 무게가 실려 있다.

### 추천 사용 시나리오

- 대형 저장소 분석, 다중 파일 수정, 테스트 실행, 셸 중심 디버깅
- worktree, hooks, MCP, 긴 컨텍스트를 적극 쓰는 고급 사용자
- GUI보다 실행 증거와 재현 가능한 작업 흐름을 선호하는 팀

### 피해야 할 사용 시나리오

- 화면 안에서 가볍게 diff만 보며 빠르게 반복하는 IDE 중심 작업
- 최근 제품 레이어 변화에 매우 민감하고, 체감 변동성을 특히 싫어하는 조직

### 최근 업데이트 반영 관찰

- 이번 주 Claude Code의 핵심은 "새 기능 추가"가 아니라 "품질 흔들림을 어떻게 해석하고 다시 막는가"였다.
- 즉 모델보다 운영 규율과 회고의 질이 더 중요한 국면이다.

### 현재 시점 총평

Claude Code는 여전히 가장 강한 CLI형 에이전트 후보다. 다만 이번 주에는 공격적 확장보다 품질 운영 신뢰 회복이 중심이었다. 깊은 작업에는 강하지만, 제품 레이어 변경이 체감 품질에 미치는 영향도 계속 주의해서 봐야 한다.

## 2. Cursor

### 강점

- Cursor 3.2는 `/multitask`, worktrees, multi-root workspaces를 한 번에 묶으며 실제 멀티레포 에이전트 작업을 제품 표면으로 끌어올렸다.
- 2026-04-21 안정성 글은 OOM-per-session 80% 감소, OOM-per-request 73% 감소를 공개해, 앱 안정성을 기능만큼 중요한 경쟁력으로 다루고 있음을 보여준다.
- IDE 안에서 에이전트, 브라우저, diff, 여러 작업 공간을 시각적으로 다루는 흐름이 가장 자연스럽다.

### 약점

- 2026-04-25~2026-04-28 창 안의 새 공식 항목이 없어서 이번 주엔 후속 보강이 조용했다.
- 기능이 빠르게 커질수록 앱 안정성과 자원 사용량 관리가 계속 핵심 리스크로 남는다.
- 순수 CLI형 사용자에게는 제품 강점 상당수가 과할 수 있다.

### 추천 사용 시나리오

- 프런트엔드, 풀스택, 다중 폴더 프로젝트를 시각적으로 오가며 병렬 작업할 때
- 브랜치, worktree, 여러 에이전트를 한 화면에서 관리하고 싶은 개발자
- 코드 수정과 브라우저 확인을 자주 왕복하는 팀

### 피해야 할 사용 시나리오

- 완전한 터미널 중심 개발 문화
- 메모리 사용량과 앱 무게에 특히 민감한 환경

### 최근 업데이트 반영 관찰

- Cursor의 방향은 분명하다. "편한 AI IDE"에서 "멀티 작업 에이전트 작업장"으로 계속 올라가고 있다.
- 그만큼 안정성 공학과 자원 관리가 제품 정체성의 일부가 됐다.

### 현재 시점 총평

Cursor는 가장 공격적으로 병렬 에이전트 IDE를 확장하고 있다. 일상 개발의 손맛과 시각적 제어는 강하지만, 그 편의성을 유지하기 위한 안정성 엔지니어링이 계속 함께 따라와야 한다.

## 3. OpenAI Codex

### 강점

- 2026-04-28 확인 기준 rate card와 플랜 문서가 모두 최근 갱신 상태여서, 가격, 권한, 접근 표면 구성이 빠르게 공식화되고 있다.
- OpenAI Academy의 `Working with Codex`, `Automations`는 Codex를 프로젝트, 스레드, 반복 작업 단위의 실행 환경으로 설명해 진입 장벽을 낮춘다.
- app, CLI, IDE, web, 플러그인, RBAC, Compliance API까지 한 제품군 안에서 다루는 구조가 가장 넓다.

### 약점

- 이번 주 변화의 중심이 새 기능보다 가격표, 플랜, 권한, 운영 문서라서 매력을 체감하려면 제품 문서까지 함께 읽어야 한다.
- Help Center의 상대 날짜 표기는 절대 수정 시각 추적에 불편하다.
- 실제 비용 감각은 토큰 구조, fast mode, 작업 크기, 클라우드 위임 여부를 함께 봐야 해서 초보자에게는 직관적이지 않다.

### 추천 사용 시나리오

- 반복 작업 자동화, 일정 기반 점검, 배치형 코드 작업
- 권한 통제, 플러그인 통제, RBAC, Compliance API 같은 관리면이 중요한 팀
- 로컬 상호작용과 클라우드 위임을 함께 쓰려는 조직

### 피해야 할 사용 시나리오

- 즉흥적이고 짧은 IDE 보조만 원하는 개인 사용자
- 토큰 기반 비용 구조와 플랜 차이를 신경 쓰고 싶지 않은 팀

### 최근 업데이트 반영 관찰

- Codex는 이번 주 가장 뚜렷하게 "운영 표면"을 정리하고 있었다.
- 기능 화려함보다 누가 어떤 플랜에서, 어떤 권한으로, 어떤 표면에서, 어떤 비용 구조로 쓰는가가 핵심 질문이 됐다.

### 현재 시점 총평

Codex는 가장 제품군이 넓고 자동화 친화적이다. 단순 체험보다 하네스화할 때 가치가 커지고, 개인 툴을 넘어 팀 운영면까지 포함하는 제품으로 빠르게 정리되고 있다.
