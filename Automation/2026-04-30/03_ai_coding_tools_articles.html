Claude Code

Week 17 · April 20–24, 2026

공식 주간 다이제스트 확인 완료 게시 범위 2026-04-20 ~ 2026-04-24

Anthropic의 Week 17 문서는 /ultrareview 연구 프리뷰, 자동 세션 리캡, 플러그인 기반 커스텀 테마, 웹 UI 재설계를 한 묶음으로 설명한다. 단일 명령이 아니라 장기 세션과 다중 에이전트를 사람이 추적하기 쉽게 만드는 방향이 핵심이다.

실무 시사점: Claude Code의 최근 가치는 한 번 더 똑똑한 답변보다, 여러 세션을 오래 돌릴 때 덜 깨지고 더 잘 요약해 주는 운영면 강화에 있다.

Cursor

Build programmatic agents with the Cursor SDK

공식 글 확인 완료 발표일 2026-04-29

Cursor는 SDK를 공개하면서 데스크톱 앱, CLI, 웹 앱에서 쓰던 같은 에이전트 런타임을 TypeScript에서 직접 호출할 수 있게 했다. 로컬과 클라우드 VM 양쪽 실행을 지원하고, public beta로 풀었다.

실무 시사점: Cursor의 무게중심이 인터랙티브 IDE에서 조직용 프로그래머블 에이전트 인프라로 확장되고 있다. 이제 SDK가 붙는 순간 CI/CD와 내부 도구 연동이 훨씬 쉬워진다.

출처: Cursor SDK release | 메모: 공식 changelog

Multitask, Worktrees, and Multi-root Workspaces

공식 글 확인 완료 발표일 2026-04-24

Cursor 3.2는 비동기 서브에이전트 기반 /multitask, 개선된 worktrees, multi-root workspaces를 전면에 배치했다. 여러 저장소를 한 세션에 묶고, 큰 작업을 병렬로 분해하는 흐름이 제품 본체가 됐다.

실무 시사점: 3.2가 열어 준 병렬 작업 철학이 2026-04-29 SDK와 자연스럽게 연결된다. 즉, UI에서 시작한 멀티에이전트 경험이 이제 API로 넘어가고 있다.

OpenAI Codex

An open-source spec for Codex orchestration: Symphony

공식 엔지니어링 글 확인 완료 발표일 2026-04-27

OpenAI는 Symphony를 "프로젝트 관리 보드를 코딩 에이전트용 control plane으로 바꾸는 오케스트레이터"로 설명한다. 열린 작업마다 하나의 에이전트를 붙이고, 에이전트가 계속 실행되며, 사람은 결과를 검토하는 구조다.

실무 시사점: Codex의 최근 읽을거리 중 가장 중요한 것은 새 UI가 아니라, 티켓 기반 다중 에이전트 운영을 어떻게 표준화할지에 대한 설계 문서다.

출처: OpenAI Symphony | 메모: 공식 엔지니어링 글

What is Codex?

OpenAI Academy 확인 완료 발표일 2026-04-23

Academy 글은 Codex를 "실제 일을 위임할 수 있는 AI agent"로 설명하고, 코드뿐 아니라 문서, 슬라이드, 반복 작업, 정보 수집까지 작업 범위를 넓혀 소개한다. ChatGPT는 일을 생각하게 돕고, Codex는 일을 앞으로 밀어 준다는 구분도 명확하다.

실무 시사점: OpenAI가 Codex를 순수 개발자 도구보다 범용 작업 자동화 환경으로 포지셔닝하고 있음을 보여준다.

출처: What is Codex? | 메모: 공식 학습 자료

Using Codex with your ChatGPT plan / Codex rate card

운영 문서 내용 확인 완료 2026-04-30 조회

플랜 문서는 Codex를 app, CLI, IDE extension, web, cloud delegation까지 하나의 표면으로 설명하고, 한시적인 Free/Go 포함과 2x rate limits 문구를 담고 있다. Rate card 문서는 토큰 기반 크레딧 구조와 모델별 입력/출력 단가를 정리한다.

실무 시사점: Codex 관련 "읽을거리"의 상당수가 이제 기능 소개보다 사용량, 가격, 권한, 배포 방식 같은 운영면으로 이동했다. 이건 제품이 체험 단계를 지나 운영 단계로 들어갔다는 뜻이다.
