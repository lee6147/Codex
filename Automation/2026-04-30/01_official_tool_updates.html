빠른 요약

이번 3일 창에서는 Claude Code가 가장 자주 움직였고, Cursor는 제품 표면을 SDK로 넓혔으며, Codex는 "에이전트를 어떻게 조직적으로 운영할 것인가"를 문서와 설계 글로 강화했다. 즉, 세 도구 모두 단순한 코드 생성기보다 에이전트 운영 환경으로 자신을 설명하는 방향이 강해졌다.
OpenAI Codex의 경우 2026-04-27 Symphony 글은 명확한 날짜가 있으나, Help Center 문서는 상대 시각만 제공하거나 캐시가 달라 보였다. 따라서 문서 내용 자체는 확인 완료로 보되, 절대 수정 분 시각은 확인 불가로 기록했다.

Claude Code

연속 릴리스: v2.1.123

도구: Claude Code 확인 완료 발표일 2026-04-29

Anthropic GitHub 릴리스 페이지 기준으로 v2.1.123은 2026-04-29에 배포됐다. 공개 변경점은 크지 않지만, OAuth 인증이 특정 베타 비활성화 환경변수와 충돌하며 401 재시도 루프에 빠지던 문제를 바로 고쳤다.

큰 기능 추가보다도 "세션이 오래 돌 때 끊기지 않게 만드는 사후 정리"에 가까운 업데이트다. 운영 안정성 위주의 짧은 패치가 하루 단위로 이어진다는 점 자체가 Claude Code의 현재 우선순위를 보여준다.

출처: anthropics/claude-code GitHub Releases | v2.1.123

바로 전 릴리스: v2.1.122

도구: Claude Code 확인 완료 발표일 2026-04-28

v2.1.122는 Bedrock 서비스 티어 선택용 환경변수 추가, PR URL 기반 세션 재개 검색, 그리고 MCP/Structured Output/ToolSearch 주변 버그 수정을 담았다. 눈에 띄는 대형 기능보다 기업형 연결성과 세션 복원력이 중심이다.

특히 Bedrock, Vertex, ToolSearch 관련 수정은 Claude Code가 개인 CLI를 넘어 다양한 기업 인프라 위에서 동작하는 공용 하네스라는 점을 다시 드러낸다.

출처: anthropics/claude-code GitHub Releases | v2.1.122

이번 주 문맥: Week 17 다이제스트

주간 요약 확인 완료 범위 2026-04-20 ~ 2026-04-24

Anthropic의 Week 17 문서는 /ultrareview 연구 프리뷰, 자동 세션 리캡, 커스텀 테마, 웹 UI 재설계를 한 번에 묶어 설명한다. Claude Code의 주간 테마는 "더 많은 하위 에이전트와 더 긴 세션을 사람이 놓치지 않게 만드는 것"이었다.

이번 3일 창의 패치 릴리스들과 이어 보면, Claude Code는 지금 화려한 신기능보다 장기 실행 품질과 운영 복원력을 다지는 단계라고 해석하는 편이 맞다.

Cursor

Cursor SDK 공개

도구: Cursor 확인 완료 발표일 2026-04-29

Cursor는 2026-04-29 공식 changelog에서 Cursor SDK를 발표했다. 데스크톱 앱, CLI, 웹 앱에서 쓰던 같은 에이전트 런타임을 TypeScript 몇 줄로 호출할 수 있게 하며, 로컬 또는 Cursor 클라우드 VM에서 실행할 수 있다고 명시했다.

동시에 Cloud Agents API를 durable agent와 run 단위로 재구성하고, SSE 스트리밍, 재연결, 수명주기 제어를 더했다. 즉 Cursor는 더 이상 "에디터 안 에이전트"에 머무르지 않고 조직용 프로그래머블 에이전트 표면을 공식화했다.

출처: Cursor Changelog | Build programmatic agents with the Cursor SDK

직전 제품 프레임: Cursor 3.2

맥락 보강 확인 완료 발표일 2026-04-24

Cursor 3.2는 /multitask, worktrees, multi-root workspaces를 전면에 내세웠다. 비동기 서브에이전트가 큰 작업을 쪼개 병렬 처리하고, 여러 저장소를 하나의 에이전트 세션으로 묶는 구조를 제품 표면에 올렸다.

이번 SDK 출시는 이 3.2의 운영 철학을 코드 호출 API로 확장한 것으로 읽힌다. 즉 3.2가 UI에서 병렬 실행을 열었다면, SDK는 그 병렬 실행을 자동화와 파이프라인 쪽으로 밀어 넣는다.

출처: Cursor Changelog | Multitask, Worktrees, and Multi-root Workspaces

OpenAI Codex

Symphony 공개

도구: OpenAI Codex 확인 완료 발표일 2026-04-27

OpenAI는 2026-04-27 엔지니어링 글에서 Symphony를 공개했다. 핵심 설명은 "프로젝트 관리 보드를 코딩 에이전트용 control plane으로 바꾸는 오케스트레이터"이며, 열린 작업마다 하나의 에이전트가 붙어 계속 돌도록 설계했다고 밝혔다.

이 글은 Codex를 단일 세션 도구가 아니라 티켓 기반 다중 에이전트 운영 체계로 설명한다. 지난 며칠의 가장 구조적인 Codex 신호는 새 버튼이 아니라 바로 이 운영 모델 공개다.

플랜 문서 상태 확인

도구: OpenAI Codex 내용 확인 완료 2026-04-30 조회

2026-04-30 실제 조회 기준 Using Codex with your ChatGPT plan 문서는 Codex가 Plus, Pro, Business, Enterprise/Edu에 포함되며, 한시적으로 Free와 Go에도 포함된다고 안내했다. 또 Codex app, CLI, IDE extension, web, cloud delegation을 모두 하나의 제품군으로 설명한다.

다만 Help Center의 상대 표기와 캐시가 일관되지 않아 절대 수정 일시는 확인하지 못했다. 그래서 이 보고서는 문서 내용 상태는 반영하되, 상세 수정 시각은 확인 불가로 남긴다.

요금 체계 문서 상태 확인

가격/운영 내용 확인 완료 2026-04-30 조회

Codex rate card 문서는 현재 Codex 사용량이 토큰 단위 크레딧 구조에 맞춰 설명되며, GPT-5.5, GPT-5.4, GPT-5.3-Codex 등의 입력/캐시 입력/출력 토큰 단가를 분리해 보여준다. Fast mode 가산, code review 전용 모델, 사용량 패널 등 운영면의 세부 항목도 명시되어 있다.

즉 Codex는 단순 기능 소개보다 비용 예측과 조직 운영 가시성 문서를 계속 정교화하는 단계다. Symphony가 control plane이라면, 이 문서는 consumption plane을 정리하는 쪽이다.
