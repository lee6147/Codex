## 개요

- 2026-04-27~2026-04-30 공식 업데이트 창 기준으로 가장 자주 움직인 쪽은 Claude Code였다. `v2.1.121`에서 `v2.1.123`까지 연속 배포하며 인증, ToolSearch, Bedrock, 세션 재개, 메모리 누수 같은 운영 마찰을 줄였다.
- Cursor는 2026-04-29 SDK 공개로 이번 주 메시지를 분명히 했다. Cursor 3.2가 UI에서 연 멀티태스크·worktree·multi-root 철학을 이제 TypeScript API와 클라우드 에이전트 런타임으로 확장했다.
- OpenAI Codex는 2026-04-27 Symphony로 오케스트레이션 설계를 공개했고, 2026-04-30 확인 기준 플랜·요금 문서도 계속 손보고 있었다. Codex의 중심 질문은 새 기능보다 "조직이 어떻게 여러 에이전트를 통제하고 비용을 읽을 것인가"에 더 가깝다.
- GitHub Trending은 오늘도 skills, harness, code intelligence, agent workspace 계층이 강했다. 시장의 관심이 모델 성능 자체보다 에이전트를 오래, 많이, 재현 가능하게 굴리는 운영 레이어로 이동하고 있다는 뜻이다.
- 기사 레인은 최근 2일 창만으로는 도구별 읽을거리가 얇아서 2026-04-23~2026-04-30으로 넓혔다. 이번 주 읽을 만한 글은 광고성 비교보다 릴리스 운영, SDK, 오케스트레이션, 플랜/요금 같은 실무 문서였다.

## 1. Claude Code

### 강점

- 2026-04-28~2026-04-29 사이의 `v2.1.122`, `v2.1.123`는 겉보기에 작은 패치처럼 보여도 실제로는 장기 세션 운영에서 걸리는 마찰을 줄인다. Bedrock 서비스 티어 선택, PR URL 기반 세션 검색, ToolSearch 복원, OAuth 401 루프 수정은 팀 환경에서 체감이 큰 항목이다.
- Week 17 다이제스트를 같이 보면 Claude Code는 `/ultrareview`, 자동 세션 리캡, 웹 UI 재설계처럼 "여러 세션을 오래 굴리는 사람"을 위한 기능을 더 분명하게 다듬고 있다.
- 즉 Claude Code의 최근 경쟁력은 단발성 마법이 아니라 깊은 CLI 작업과 긴 실행 루프를 버티는 운영 품질에 있다.

### 약점

- 최근 읽을거리의 중심이 대부분 안정화, 운영 복원력, 연결성 수선이라서 겉으로는 조용해 보일 수 있다.
- 강점이 팀용/고급용 흐름에 몰려 있어, 가벼운 IDE 보조만 원하는 사용자에게는 가치가 덜 선명하다.
- 제품 체감은 모델 자체만이 아니라 settings, hooks, MCP, auth, tool routing 같은 주변 레이어에 크게 좌우된다.

### 추천 사용 시나리오

- 큰 저장소에서 긴 세션으로 분석, 수정, 테스트, 셸 디버깅을 이어 가는 작업
- MCP, hooks, skills, 장기 세션 복구 같은 운영 기능을 적극 활용하는 사용자
- 단일 작업을 깊게 파고들거나, 여러 CLI 세션을 병렬로 돌리는 팀

### 피해야 할 사용 시나리오

- 설정과 운영층을 거의 만지지 않고 바로 "가벼운 에디터 보조"만 기대하는 경우
- 장기 세션 안정화보다 즉시 체감되는 UI 편의가 더 중요한 경우

### 최근 업데이트 반영 관찰

- 이번 주 Claude Code의 핵심은 "새로운 장난감"이 아니라 "덜 끊기고, 덜 꼬이고, 더 길게 버티는가"다.
- 이것은 성숙기 제품의 전형적 움직임이다. 이제 승부처는 기능 추가 자체보다 운영 품질이다.

### 현재 시점 총평

Claude Code는 여전히 가장 깊은 CLI형 에이전트 후보에 가깝다. 이번 주에는 공격적 확장보다 세션 복원력과 운영 안정화가 중심이었고, 오히려 이 방향이 장기 사용자에게는 더 중요하다.

## 2. Cursor

### 강점

- 2026-04-29 SDK 공개는 Cursor가 더 이상 "좋은 AI 에디터"에 머무르지 않는다는 선언에 가깝다. 데스크톱, CLI, 웹 앱에서 쓰던 같은 런타임을 TypeScript에서 직접 호출하게 하며, 로컬과 클라우드 VM 실행을 모두 열었다.
- 2026-04-24의 Cursor 3.2는 `/multitask`, worktrees, multi-root workspaces로 멀티에이전트 작업장이라는 그림을 제품 표면에 고정했다.
- 즉 Cursor는 UI에서 비주얼하게 관리되는 에이전트 작업장과, 코드에서 호출되는 프로그래머블 에이전트 표면을 가장 빠르게 연결하고 있다.

### 약점

- 기능이 늘수록 앱 무게, 안정성, 운영 복잡도 문제도 같이 커질 수밖에 없다.
- Cursor의 최근 강점은 IDE 안에서 끝나지 않고 조직용 자동화로 뻗는 데 있는데, 순수 CLI 사용자에게는 오히려 과할 수 있다.
- 외부 실사용 검증보다 공식 문서와 공식 발표가 주된 읽을거리여서, "많은 팀이 실제로 어떻게 굴리고 있는가"에 대한 독립 검증은 상대적으로 적다.

### 추천 사용 시나리오

- 여러 작업을 시각적으로 병렬 관리하고 브랜치/worktree를 자주 넘나드는 개발자
- 멀티레포 작업과 비동기 서브에이전트 분해가 중요한 팀
- SDK로 내부 자동화나 CI/CD 파이프라인까지 연결하려는 조직

### 피해야 할 사용 시나리오

- 완전한 터미널 중심 개발 문화
- 무거운 앱 레이어나 복잡한 제품 표면을 최대한 피하고 싶은 환경

### 최근 업데이트 반영 관찰

- Cursor의 메시지는 매우 선명하다. "대화형 AI 에디터"에서 "에이전트 런타임 플랫폼"으로 올라가고 있다.
- 이번 주 포인트는 3.2가 보여준 멀티태스크 철학이 SDK로 넘어갔다는 점이다. 이제 사람의 손으로만 쓰는 제품이 아니라 코드에서 호출하는 인프라가 됐다.

### 현재 시점 총평

Cursor는 세 도구 중 인터랙티브 경험과 프로그래머블 표면을 가장 공격적으로 통합하고 있다. 손맛 좋은 UI가 강점이지만, 그만큼 안정성과 운영 복잡도 관리가 계속 과제가 된다.

## 3. OpenAI Codex

### 강점

- Symphony는 Codex를 티켓 기반 다중 에이전트 운영 체계로 설명한다. 열린 작업마다 에이전트를 붙이고, 프로젝트 관리 보드를 control plane으로 쓰는 그림은 Codex의 방향을 가장 잘 보여 준다.
- OpenAI Academy의 `What is Codex?`는 Codex를 코드 작성기에 한정하지 않고, 문서, 슬라이드, 반복 업무, 정보 수집까지 맡기는 범용 작업 에이전트로 설명한다.
- 플랜 문서와 rate card는 app, CLI, IDE extension, web, cloud delegation, 권한, 사용량, 가격을 한 제품군으로 묶어 설명한다. 즉 Codex는 기능보다 운영 체계를 먼저 읽어야 가치가 보이는 도구가 됐다.

### 약점

- 최근 중요한 읽을거리 상당수가 요금, 플랜, 권한, 운영 문서라서 제품 가치를 문서까지 함께 읽어야 파악할 수 있다.
- Help Center는 상대 시각과 캐시가 섞여 보일 때가 있어, 절대 수정 시각 추적은 불편하다.
- 비용 감각은 모델, fast mode, automations, cloud delegation, workspace credit 구조를 함께 봐야 해서 초보자에게 직관적이지 않다.

### 추천 사용 시나리오

- 반복 작업 자동화, 일정 기반 점검, 티켓 기반 오케스트레이션
- 권한 제어, 사용량 가시성, 팀 단위 배포가 중요한 조직
- 코드 작업 외 문서/리서치/파일 업데이트까지 같은 에이전트 작업 환경으로 묶고 싶은 팀

### 피해야 할 사용 시나리오

- 즉흥적이고 짧은 IDE 보조만 원하는 개인 사용자
- 가격·권한·운영 문서를 읽고 싶지 않은 팀

### 최근 업데이트 반영 관찰

- Codex는 이번 주 가장 분명하게 "운영 체계"를 제품 본체로 밀고 있다.
- 기능 하나보다 오케스트레이션 설계, 플랜 접근, 요금 구조, 사용량 가시성이 더 중요한 질문이 됐다.

### 현재 시점 총평

Codex는 세 도구 중 가장 하네스화 친화적이다. 단순 체험보다 반복 업무 자동화, 조직 권한 통제, 프로젝트 단위 운영 환경에서 가치가 커지고 있고, 이번 주는 그 방향이 더욱 또렷해진 시점이다.

- 하나의 고난도 작업을 깊게 파고드는 사람에게는 Claude Code가 가장 잘 맞는다.

## 이번 주 총평

AI 코딩 도구 시장은 점점 "누가 더 똑똑한가"보다 "누가 더 오래, 많이, 구조적으로 굴러가는가"의 경쟁으로 이동하고 있다. Claude Code는 긴 실행 루프의 복원력을 다지고 있고, Cursor는 멀티에이전트 경험을 SDK로 확장하고 있으며, Codex는 오케스트레이션과 운영 문서를 제품 핵심으로 밀고 있다. 이제 사용자는 답변 품질 비교만으로는 부족하고, 작업 구조와 운영 표면을 먼저 봐야 한다.
