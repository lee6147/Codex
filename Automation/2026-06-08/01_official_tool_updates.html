요약

이번 공식 업데이트 흐름은 "모델 성능"보다 "agent 운영 안전성"에 가깝다. Claude Code는 외부 세션 권한 전파와 fallback 실행을 다듬었고, Cursor는 시각 피드백과 자동 검토를 agent SDK와 canvas에 붙이고 있다. OpenAI Codex는 2026-06-06 alpha 릴리스가 확인되지만, 상세 changelog는 stable 0.137.0 쪽이 더 정보량이 높다.

Claude Code

2.1.168 및 2.1.167: 안정성 릴리스

확인 완료공식 changelog2026-06-06

Anthropic 공식 Claude Code changelog에는 2.1.168과 2.1.167이 모두 2026-06-06 릴리스로 표시되며, 공개 설명은 bug fixes and reliability improvements 수준이다. 세부 기능명은 공개되지 않았으므로 실무 영향은 "최신 패치 존재" 이상으로 과장하지 않는 것이 맞다.

실무 시사점: 조직 배포에서는 2.1.166 이후 버전을 같은 안정성 묶음으로 보고, headless/remote/background agent smoke test를 함께 확인하는 편이 안전하다.

출처: Claude Code changelog

2.1.166: fallback 모델과 권한 하드닝

확인 완료공식 changelog2026-06-06

2.1.166은 최대 3개 fallback 모델 설정, deny rule의 tool-name glob 지원, cross-session SendMessage 권한 차단, thinking 비활성화 제어, 예기치 않은 API 오류에 대한 fallback 재시도를 포함한다. Windows PowerShell validation hang, remote session stuck, background agent worktree crash-loop 같은 운영 버그도 함께 고쳤다.

실무 시사점: 장시간 agent 작업, remote session, background agent, Windows 터미널 작업을 쓰는 팀은 이 릴리스가 단순 패치보다 중요하다.

Cursor

3.7 Design Mode Improvements

확인 완료공식 changelog2026-06-05, strict 3일 밖

Cursor는 3.7 Design Mode Improvements에서 브라우저 안 Design Mode로 클릭, 그리기, 음성 설명을 통해 agent에게 UI 수정을 지시할 수 있다고 설명한다. 다중 선택 요소와 voice input은 frontend review를 텍스트 지시에서 시각 지시로 옮기는 업데이트다.

실무 시사점: UI 리팩터링, 디자인 QA, 반복적 화면 수정에서는 Cursor가 "코드 작성 도구"보다 "시각 피드백을 agent 입력으로 바꾸는 IDE"에 가까워진다.
