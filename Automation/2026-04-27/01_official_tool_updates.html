Claude Code

창 안에서는 신규 공식 업데이트 미확인

2026-04-24 ~ 2026-04-27 공식 신규 항목 없음

Anthropic 뉴스, 엔지니어링 글, Claude Code 릴리스 흐름을 2026-04-27 기준으로 다시 확인했지만, 이번 3일 창 안에서 새롭게 올라온 Claude Code 공식 업데이트는 확인되지 않았다.

즉 이번 비교에서 Claude Code는 “이번 주에 또 무엇이 추가됐는가”보다 “직전 릴리스와 사후 설명이 아직도 해석의 기준인가”를 봐야 하는 상태다.

최신 확인 릴리스는 2026-04-23의 v2.1.119

GitHub Releases 2026-04-23 창 밖 참고

가장 최신으로 확인되는 릴리스는 2026-04-23의 v2.1.119다. 이 릴리스에는 /config 설정 영속화, PowerShell 권한 모드 자동 승인, MCP 서버 병렬 재연결, Windows 경고 정리, worktree 관련 수정이 포함됐다.

실무적으로는 새 모델 발표보다 “Windows/PowerShell 환경에서 실제로 덜 거슬리게 돌아가게 만드는 유지보수” 성격이 강했다. 이 점은 터미널 중심 사용자에게 여전히 중요하다.

2026-04-23 사후 설명: 품질 저하 원인 3개를 공식 정리

Anthropic Engineering 2026-04-23 운영 회고

Anthropic은 최근 Claude Code 품질 저하 보고를 세 가지 원인으로 분리해 설명했다. 기본 reasoning effort를 high에서 medium으로 내린 판단, 유휴 세션의 오래된 생각을 지우는 과정의 버그, 그리고 2026-04-16에 넣은 과도한 간결화 프롬프트가 겹쳤다는 설명이다.

핵심은 “느려 보여서 바꾼 최적화가 실제 코딩 품질을 깎았고, 그 교훈을 다시 제품 운영 규칙으로 올렸다”는 점이다. 공식 회고가 이렇게 구체적이면 다음 릴리스의 해석 기준이 된다.

Cursor

Cursor 3.2: /multitask, worktrees, multi-root workspaces

공식 changelog 2026-04-24 창 안 신규

Cursor changelog의 최신 항목은 2026-04-24 공개된 3.2다. 핵심은 /multitask로 비동기 서브에이전트를 병렬 실행하고, Agents Window에서 개선된 worktree 흐름을 제공하며, 여러 폴더를 하나의 세션 대상으로 묶는 multi-root workspace를 도입한 것이다.

이 변화는 Cursor가 단순 편집기 보조에서 멀티레포 작업을 처리하는 에이전트 조정면으로 더 밀어 올리고 있다는 신호다. 특히 cross-repo 수정과 백그라운드 분기 작업을 공식 경로로 끌어올린 점이 크다.

OpenAI Codex

Codex rate card가 계속 갱신 중

OpenAI Help Center 2026-04-27 확인 가격 구조

2026-04-27 확인 시 Codex rate card 페이지는 Updated: 18 hours ago로 표시됐다. 이 문서는 Plus, Pro, Business, Enterprise, Edu, Health, Gov 전반에 걸친 Codex 크레딧 구조를 설명하며, 현재 기본 프레임이 메시지당 과금이 아니라 토큰 유형별 크레딧 사용량이라는 점을 다시 못박고 있다.

문서에는 2026-04-23부터 기존 Enterprise/Edu/Gov/Health/Teachers 고객까지 토큰 기반 요금 구조가 확대 적용됐다고 적혀 있다. 또 GPT-5.5, GPT-5.4, GPT-5.4-Mini, GPT-5.3-Codex별 입력/캐시 입력/출력 토큰당 크레딧이 명시돼 있어, Codex 비교가 이제 기능보다 운영비 예측과 더 강하게 묶인다.
