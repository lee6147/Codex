## 1. Claude Code

### 강점

- 가장 깊은 코드베이스 작업과 CLI 중심 실행 흐름에 여전히 강하다.
- 최신 확인 릴리스인 `v2.1.119`는 PowerShell 자동 승인, `/config` 영속화, MCP 병렬 재연결, worktree 보정처럼 실제 사용 마찰을 줄이는 수정이 많다.
- 2026-04-23 품질 사후 설명이 매우 구체적이라, 제품 운영이 숨기기보다 원인 분해와 교정으로 움직인다는 신뢰를 준다.

### 약점

- 최근 3일 공식 창 안의 신규 발표가 없어서 이번 주에는 “새 기능 모멘텀”이 약했다.
- 품질 저하 이슈가 제품 레이어의 최적화 판단과 프롬프트 변경에서 실제로 발생했다는 점은, 릴리스 속도가 빠른 만큼 체감 변동성도 크다는 뜻이다.
- 모델 자체보다 설정, 세션 맥락, tool-use 흐름 변화에 결과가 크게 흔들릴 수 있다.

### 추천 사용 시나리오

- 대형 저장소 분석, 다중 파일 수정, 테스트 실행, 셸 중심 작업
- IDE보다 터미널과 파일 시스템을 직접 잡고 움직이는 개발자
- MCP, hooks, worktree, 긴 컨텍스트를 적극 쓰는 팀

### 비추천에 가까운 사용 시나리오

- 눈앞의 파일을 짧게 고치고 바로 diff를 클릭하며 진행하는 가벼운 IDE 중심 작업
- 최근 버전 변화에 매우 민감하고, 제품 레이어 변동성을 싫어하는 조직

### 최신 업데이트 반영 관찰

- 이번 주 Claude Code의 핵심은 “또 무엇이 추가됐는가”보다 “품질 흔들림이 왜 생겼고 무엇을 교정했는가”였다.
- 즉 기능 경쟁보다 운영 규율과 검증 체계가 더 중요한 국면이다.

### 현재 시점 총평

Claude Code는 여전히 가장 강한 CLI형 에이전트 후보지만, 이번 주에는 공격적 확장보다 품질 운영의 신뢰 회복이 중심이었다. 깊은 작업에는 강하지만, 제품 운영 선택이 체감 품질에 미치는 영향도 그만큼 크다.

## 2. Cursor

### 강점

- 2026-04-24 공개된 Cursor 3.2는 `/multitask`, worktrees, multi-root workspaces를 하나로 묶으며 실제 멀티레포 에이전트 작업에 한 걸음 더 갔다.
- 2026-04-21 안정성 글은 OOM-per-session 80% 감소, OOM-per-request 73% 감소를 제시하며 “기능을 붙여도 버티는 앱”을 만들려는 공학적 깊이를 보여준다.
- IDE 안에서 바로 에이전트, 브라우저, diff, worktree를 오가는 작업에 가장 자연스럽다.

### 약점

- 기능이 빠르게 커지는 만큼 앱 안정성과 자원 사용량 관리가 계속 핵심 리스크다.
- 강력한 기능 상당수가 Agents Window와 IDE 표면에 묶여 있어, 순수 CLI형 워크플로를 선호하는 사람에게는 덜 맞는다.
- 권한 통제와 장기 자동화보다는 “지금 화면에서 빠르게 병렬 작업”에 더 무게가 실려 있다.

### 추천 사용 시나리오

- 프런트엔드, 풀스택, 다중 폴더 프로젝트를 시각적으로 오가며 병렬 작업할 때
- 작업 중간중간 diff 확인, 파일 이동, 브라우저 확인을 자주 하는 사용자
- 멀티태스크와 worktree를 IDE 안에서 바로 쓰고 싶은 팀

### 비추천에 가까운 사용 시나리오

- 완전한 터미널 중심 개발 문화
- IDE 무거움이나 메모리 사용량을 특히 싫어하는 환경

### 최신 업데이트 반영 관찰

- 이번 주 Cursor의 메시지는 분명하다. “편한 AI IDE”에서 “멀티 작업 에이전트 콘솔”로 올라가고 있다.
- 다만 그만큼 안정성 엔지니어링이 제품 경쟁력의 본체가 된다.

### 현재 시점 총평

Cursor는 지금 가장 공격적으로 에이전트 IDE를 확장하고 있다. 일상 개발의 손맛과 병렬성은 강하지만, 그 성능을 유지하기 위한 안정성 공학이 계속 따라붙어야 하는 구조다.

## 3. OpenAI Codex

### 강점

- 2026-04-27 확인 기준 rate card와 플랜 문서가 모두 최근 갱신 상태라, 가격·권한·표면 구성이 빠르게 명문화되고 있다.
- OpenAI Academy의 `Working with Codex`, `Automations`가 프로젝트, 스레드, 자동화 루틴을 실무형으로 설명해 진입 장벽을 낮춘다.
- 로컬 작업과 클라우드 위임, app/CLI/IDE/web, 자동화까지 하나의 제품군으로 묶는 방향이 분명하다.

### 약점

- 이번 주 변화의 중심이 기능 발표보다 가격표, 플랜, 권한 문서라서 제품의 매력을 체감하려면 운영 문서를 함께 읽어야 한다.
- 상대 표기 위주의 Help Center 업데이트는 날짜 추적에 불편하다.
- 실제 비용 감각은 토큰 구조와 fast mode, task 크기를 함께 봐야 해서 초보자에게는 직관적이지 않다.

### 추천 사용 시나리오

- 반복 작업 자동화, 일정 기반 점검, 배치형 코드 작업
- 권한 통제, 플러그인 통제, RBAC, Compliance API 같은 관리면이 중요한 팀
- 로컬 상호작용과 클라우드 위임을 함께 쓰려는 조직

### 비추천에 가까운 사용 시나리오

- 즉흥적이고 짧은 IDE 보조만 원하는 개인 사용자
- 토큰 기반 비용 추적과 플랜 차이를 신경 쓰고 싶지 않은 팀

### 최신 업데이트 반영 관찰

- Codex는 이번 주 기능의 화려함보다 운영 경계 정리를 더 밀고 있다.
- 즉 “무엇을 할 수 있나”보다 “누가, 어떤 플랜에서, 어떤 권한으로, 얼마를 쓰며 돌리는가”가 더 중요해졌다.

### 현재 시점 총평

Codex는 가장 제품군이 넓고 자동화 친화적이지만, 그만큼 이해해야 할 운영 문서도 많다. 반복 작업과 조직 운영 면에서 강하고, 단순 체험보다 하네스화할 때 가치가 커진다.

## 추천 사용자 유형

- 코드베이스를 깊게 파고드는 단일 고난도 작업이 많다면 Claude Code가 가장 맞다.
- 눈으로 보며 병렬 작업하고, 여러 레포를 동시에 다루는 흐름이 중요하면 Cursor가 가장 즉시 생산적이다.
- 반복 점검, 주간 보고, 백그라운드 작업 위임, 플랜·권한 관리가 중요하면 Codex가 가장 구조적이다.

## 이번 주 총평

AI 코딩 도구 시장은 모델 비교에서 운영 구조 비교로 이동 중이다. Claude Code는 품질 운영, Cursor는 멀티태스크 IDE, Codex는 자동화와 관리 경계 정리가 이번 주 핵심이었다. 사용자는 이제 “어느 모델이 더 똑똑한가”보다 “내 작업을 어떤 표면에서, 어떤 비용 구조로, 얼마나 반복 가능하게 굴릴 수 있는가”를 먼저 따져야 한다.
