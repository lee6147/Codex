기사 카드

Claude Code 품질 이슈 사후 설명

official postmortem 2026-04-23 Anthropic Engineering

Anthropic은 최근 Claude Code 품질 저하가 세 가지 서로 다른 변경이 겹쳐 생긴 문제였다고 설명했다. 기본 effort 하향, 유휴 세션 reasoning 정리 버그, 과도한 간결화 프롬프트가 각각 다른 사용자 집합에 영향을 줬다는 구조다.

이 글의 가치는 변명보다 원인 분해에 있다. 어떤 종류의 “제품 최적화”가 실제 코딩 품질 저하로 이어졌는지, 또 이를 막기 위해 prompt review와 broader evals를 어떻게 강화하겠다는지까지 적혀 있다.

왜 읽을 만한가: Claude Code를 많이 쓰는 팀이라면 단순한 릴리스 노트보다 이 글이 다음 버전 해석 기준이 된다. 품질 흔들림이 모델 열화가 아니라 제품 레이어의 선택과 버그에서 나왔다는 점을 공식 확인할 수 있다.

출처 URL: https://www.anthropic.com/engineering/april-23-postmortem

Keeping the Cursor app stable

official technical post 2026-04-21 Cursor Research

Cursor는 앱 안정성을 어떻게 계측하고, 어떤 방식으로 메모리 문제를 추적했는지 공개했다. crash telemetry, memory pressure 관측, top-down과 bottom-up 디버깅, hot path 최적화, 회귀 방지용 가드레일이 핵심이다.

문서에 따르면 OOM-per-session은 late-February peak 대비 80% 줄었고, OOM-per-request는 2026-03-01 이후 73% 감소했다. 에이전트 기능을 더 붙일수록 앱을 버티게 만드는 공학이 중요해진다는 점을 정면으로 보여준다.

왜 읽을 만한가: Cursor를 “편한 IDE”로만 보면 놓치기 쉬운 운영 비용을 보여준다. 브라우저 사용, instant grep, subagents처럼 무거운 기능이 늘어날수록 안정성 자체가 제품 경쟁력임을 설명한다.

출처 URL: https://cursor.com/blog/app-stability

Working with Codex

official workflow guide 2026-04-23 OpenAI Academy

OpenAI Academy의 Working with Codex는 Codex를 처음 열었을 때 보게 되는 sidebar, threads, projects, settings를 작업 구조로 설명한다. 특히 프로젝트가 로컬 폴더와 연결되고, 작업 범위가 그 폴더 경계 안에서 움직인다는 점을 분명히 짚는다.

또 플러그인과 skills가 “더 구체적 작업”을 시키는 장치라는 점, 여러 Codex 작업을 동시에 돌리는 것이 일반적이라는 점을 초보자 수준에서 정리한다.

왜 읽을 만한가: Codex를 단순 채팅 도구가 아니라 작업 공간 단위의 실행 환경으로 이해하게 해 준다. 실제 자동화나 반복 실행을 설계할 때 전제가 되는 문서다.

출처 URL: https://openai.com/academy/working-with-codex/

Automations

official workflow guide 2026-04-23 OpenAI Academy

OpenAI Academy의 Automations는 Codex가 스케줄과 트리거로 반복 작업을 자동 실행하는 방식을 설명한다. 요점은 좋은 자동화가 “구체적이고, 반복 가능하고, 검토하기 쉬워야 한다”는 것이다.

예시도 실무형이다. 매주 리뷰 쓰기, 전날 작업 요약, 폴더의 새 파일 요약, 누락 정보 점검, 프로젝트 상태 업데이트 등 실제 운영 루틴에 가까운 작업을 기본 사례로 둔다.

왜 읽을 만한가: 이번 하네스처럼 날짜별 산출물을 만드는 자동화와 직접 연결된다. Codex를 배치형 보조 인력처럼 쓰려면 어떤 형태의 지시가 잘 맞는지 감을 잡기 좋다.

출처 URL: https://openai.com/academy/codex-automations/
