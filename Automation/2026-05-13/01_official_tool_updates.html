Cursor

Cursor in Microsoft Teams

공식 changelog2026-05-11

Cursor는 Microsoft Teams 채널에서 @Cursor를 mention해 cloud agent에게 작업을 위임하거나 Cursor 정보를 불러올 수 있다고 발표했다. Cursor는 prompt와 최근 agent 활동을 기준으로 repository와 model을 자동 선택한다고 설명한다.

개발자 의미: IDE 안에서 시작하던 agent 업무가 팀 대화 공간으로 이동한다. PR 생성까지 연결되므로 repository 선택과 권한 범위를 팀 정책으로 확인해야 한다.

발표일: 2026-05-11 | 확인 상태: 확인 완료 | 출처: Cursor Changelog

Bugbot Effort Levels

공식 changelog2026-05-11

Teams 관리자와 Individual plan 사용자는 Bugbot의 PR review effort를 Default, High, Custom으로 조정할 수 있다. Cursor는 high effort가 더 오래 걸리고 비용이 더 들지만 더 많은 버그를 찾을 수 있다고 설명한다.

개발자 의미: 모든 PR에 고강도 리뷰를 쓰기보다 위험한 변경, 보안/결제/마이그레이션 변경에 high effort를 배정하는 운영 정책이 필요하다.

발표일: 2026-05-11 | 확인 상태: 확인 완료 | 출처: Cursor Changelog

PR Review / Build in Parallel

공식 changelog2026-05-07
