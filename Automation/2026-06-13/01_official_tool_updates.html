Claude Code

항목별 날짜 확인 불가

Anthropic의 Claude Code changelog는 최신 버전 항목을 공개하고 있으나, 이번 확인 시점의 원본 파일에서는 각 버전의 게시 날짜가 노출되지 않았다. 따라서 기능 자체는 공식 changelog 기준으로 확인하고, 날짜는 확인 불가로 처리한다.

관리형 모델 제한 하위 에이전트 권한 및 백그라운드 세션
  • 2.1.175: 관리형 설정에서 허용 모델 범위를 제한하는 `enforceAvailableModels`가 추가됐다. 조직 정책이 개발자 개인 설정보다 강하게 작동하는 방향이다.
  • 2.1.172: 하위 에이전트가 다시 하위 에이전트를 생성할 수 있는 계층 구조가 추가됐다. 복잡한 분석이나 대규모 코드베이스 탐색에서 병렬 분업 폭이 넓어진다.
  • 2.1.174 전후: 백그라운드 세션 환경 변수, 모델 표시, 승인 목록, 워크플로 하위 에이전트 attribution 등 운영 안정성 수정이 이어졌다.

실무 해석: Claude Code는 개인용 자동완성보다 조직 통제, 에이전트 분업, 장기 세션 안정성 쪽으로 무게가 이동하고 있다. 팀 단위 도입에서는 모델 허용 목록과 권한 정책을 먼저 표준화해야 한다.

Cursor

공식 날짜: 2026-06-10

Cursor는 Bugbot을 평균 리뷰 시간 약 90초, 비용 약 22% 감소, 리뷰당 버그 발견 수 10% 증가로 업데이트했다고 밝혔다. 또한 `/review` 명령으로 푸시 전 코드 리뷰를 요청하고, 보안 리뷰나 Bugbot 리뷰를 선택할 수 있는 흐름을 공개했다.

Bugbot 사전 푸시 리뷰 GitHub/GitLab 연동
  • Bugbot은 GitHub/GitLab 리뷰와 Cursor 내부 리뷰 간 중복 diff 검토를 건너뛰는 방식으로 운영 비용을 줄인다.
  • 사용자는 마지막 리뷰 이후의 신규 변경분만 리뷰하도록 설정할 수 있다.
  • Composer 2.5 기반 성능 개선은 Cursor가 단순 편집기보다 리뷰 자동화 플랫폼으로 확장되고 있음을 보여준다.

실무 해석: Cursor 사용팀은 PR 리뷰 전 단계에서 `/review`를 표준 루틴으로 두면 빠른 사전 결함 탐지에 유리하다. 다만 자동 리뷰 결과를 병합 기준으로 삼기 전에 테스트와 사람 리뷰의 책임 범위를 분리해야 한다.

OpenAI Codex

공식 날짜: 2026-06-09

Codex app 26.608은 Claude Code와 Claude Cowork에서 지원 설정을 가져오는 마이그레이션 흐름, 플러그인 화면 개선, 설정 검색 확장을 추가했다. 같은 날짜의 iOS 릴리스에는 브랜치 선택, 워크트리 생성, 환경 설정 스크립트 실행, `/goal` 지원, 변경 파일 인라인 리뷰 댓글 등이 포함됐다.

마이그레이션 플러그인 관리 Goal 및 리뷰 UI
  • Claude 계열 설정 가져오기는 경쟁 도구에서 Codex로 이전하는 사용자의 초기 설정 비용을 낮춘다.
  • 플러그인 화면과 설정 검색 개선은 로컬 앱을 장기 작업 허브로 쓰는 사용성을 강화한다.
  • 모바일 `/goal`, 사이드 챗, 인라인 리뷰 댓글은 여러 작업을 병렬 관리하는 방향의 제품 신호다.

실무 해석: Codex는 긴 작업, 워크트리, 플러그인, 리뷰 흐름을 묶는 운영 콘솔 성격이 강해지고 있다. 기존 Claude/Cursor 프로젝트 지침 파일을 옮길 때는 자동 마이그레이션 후 실제 파일 경로와 테스트 명령이 최신인지 재검증해야 한다.
