Claude Code / Anthropic

Dynamic Workflows Enterprise 기본 활성화 일정

확인 완료장기 실행관리자 제어 필요

Claude Code FAQ는 Dynamic Workflows가 대형 엔지니어링 작업을 한 세션에서 끝까지 수행하는 기능이며, Enterprise 조직에서는 2026-06-08부터 기본 활성화된다고 설명한다. 관리자는 custom role, managed-settings.json의 disableWorkflows, 조직 설정 토글로 접근을 제한할 수 있다.

Claude Platform 2026-06-05 / 2026-06-02 업데이트

확인 완료API/모델 운영

Claude Platform 릴리스 노트는 2026-06-05에 Opus 4.1 API retirement 일정을 알렸고, 2026-06-02에는 advisor tool의 max_tokens와 refusal 비과금 처리를 공지했다. Claude Code 자체 기능은 2026-05-28 항목에서 Auto mode 확대, Max plan의 Opus 4.8 fast mode 기본값, Workflows research preview가 함께 언급된다.

발표일: 2026-06-05, 2026-06-02, 2026-05-28 | 출처: Claude Platform release notes | URL: https://platform.claude.com/docs/en/release-notes/overview

Cursor

Cursor 3.7: Design Mode 개선

확인 완료프론트엔드 에이전트

Cursor 3.7은 브라우저의 Design Mode에서 여러 요소를 함께 선택해 UI 변경을 지시하고, 에이전트 실행 중에도 음성으로 후속 변경을 큐잉할 수 있게 했다. 프론트엔드 작업에서 “코드 설명”보다 “시각적 대상 선택”이 더 중요한 피드백 채널로 올라오는 흐름이다.

발표일: 2026-06-05 | 출처: Cursor changelog | URL: https://cursor.com/en-US/changelog

Cursor SDK: custom tools, auto-review, JSONL store

확인 완료SDK운영화

2026-06-04 Cursor SDK 업데이트는 함수 정의만으로 local agent에 custom tools를 연결하고, local.autoReview로 도구 호출을 auto-review 경로에 태우며, SQLite 대신 JSONL 기반 에이전트/런 메타데이터 저장을 선택할 수 있게 했다. 헤드리스 에이전트 운영과 감사 가능성이 강화된 업데이트다.

발표일: 2026-06-04 | 출처: Cursor changelog | URL: https://cursor.com/en-US/changelog

Cursor 3.6: Auto-review Run Mode

확인 완료권한 관리

2026-05-29 Cursor 3.6은 Shell, MCP, Fetch 호출에 대해 allowlist·sandbox·classifier subagent를 조합하는 Auto-review Run Mode를 도입했다. 최근 Claude Code와 Codex 업데이트와 마찬가지로 “승인 프롬프트를 줄이되 위험 호출은 분리”하는 방향이다.

OpenAI Codex

Codex for every role, tool, and workflow

확인 완료플러그인Sites preview

Codex Windows Computer Use 및 Enterprise 제어
