## 도구별 평가

### 1. Claude Code

**강점**

- 터미널 기반 장시간 작업, 대형 코드베이스 탐색, skills/플러그인 기반 도메인 업무 확장이 강하다.
- 2026-05-06 릴리스는 세션 ID, resume, SIGINT, fullscreen/터미널 안정성을 다듬어 실제 운영 환경에 유리하다.

**약점**

- 강한 권한을 가진 셸/도구와 붙을수록 승인 정책과 감사 로그가 없으면 사고 범위가 커질 수 있다.
- 도메인 플러그인은 매력적이지만 조직별 검토자 승인과 데이터 접근 경계 설계가 선행돼야 한다.

**추천 사용 시나리오**

- 터미널 중심 리팩터링, 코드베이스 감사, 반복 자동화 하네스, 금융/문서/운영 업무처럼 규칙이 많은 장기 작업.

**피해야 할 사용 시나리오**

- 파괴적 운영 API 권한이 넓게 열린 환경, 백업 격리가 안 된 운영 시스템, 승인 없이 배포/삭제가 가능한 계정.

**최근 업데이트 반영 관점**

- Claude Code v2.1.132와 금융 에이전트 발표를 함께 보면, Anthropic은 "개발자 터미널 안정성"과 "업무 도메인 플러그인"을 동시에 밀고 있다.

**현재 시점 총평**

Claude Code는 고급 사용자에게 가장 강한 레버리지 도구다. 다만 강한 도구일수록 permission, backup, audit를 먼저 설계해야 한다.

### 2. Cursor

**강점**

- IDE 안에서 빠르게 수정하고 시각적으로 diff를 확인하는 흐름이 좋다.
- 2026-05-06 Context Usage Breakdown은 rules, skills, MCP, subagents가 컨텍스트를 얼마나 쓰는지 보여줘 팀 운영에 도움이 된다.

**약점**

- 에이전트가 외부 인프라 토큰과 연결될 때, IDE의 편리함이 권한 안전성을 자동으로 보장하지 않는다.
- 복잡한 장기 작업은 터미널 기반 도구나 비동기 에이전트와 역할 분리가 필요하다.

**추천 사용 시나리오**

- 일상 기능 개발, 빠른 리팩터링, 인라인 수정, 프론트엔드/앱 코드에서 사람이 계속 diff를 보며 조정하는 작업.

**피해야 할 사용 시나리오**

- 운영 DB, 클라우드 삭제 API, 광범위한 토큰이 같은 작업 컨텍스트에 노출된 환경.

**최근 업데이트 반영 관점**

- 컨텍스트 사용량 가시성은 Cursor가 단순 IDE 보조를 넘어 에이전트 운영 디버깅 표면을 강화한다는 신호다.

**현재 시점 총평**

Cursor는 여전히 IDE-first 개발자의 기본 작업면으로 강하다. 다만 고위험 작업은 권한 축소와 별도 승인 레이어가 있어야 한다.

### 3. OpenAI Codex

**강점**

- 비동기 백그라운드 작업, 잘 정의된 티켓 처리, PR 생성/리뷰 흐름에 적합하다.
- 0.129.0 릴리스는 TUI Vim composer, resume/fork picker, workspace-aware diff, plugin workspace sharing 등 로컬 개발자 경험을 빠르게 개선했다.

**약점**

- 클라우드/비동기 실행은 즉석 pair programming보다 피드백 간격이 길다.
- 좋은 결과를 내려면 테스트 가능 범위, AGENTS.md, 환경 설정, 권한 경계가 명확해야 한다.

**추천 사용 시나리오**

- 반복 유지보수, dependency update, lint/test fix, 문서화, 독립적인 백로그 티켓, 여러 작업을 병렬로 맡기는 흐름.

**피해야 할 사용 시나리오**

- 요구사항이 자주 바뀌고 중간중간 사람이 UI/디자인 결정을 계속 내려야 하는 작업.

**최근 업데이트 반영 관점**

- Codex는 앱/클라우드뿐 아니라 CLI/TUI와 플러그인 공유까지 다듬으며 조직형 자동화 하네스에 가까워지고 있다.

**현재 시점 총평**

Codex는 "맡겨두고 검토하는" 작업에 가장 잘 맞는다. 이 워크스페이스 같은 반복 보고서/검증 자동화에도 적합한 방향으로 진화 중이다.

## 추천 사용자 유형

- **터미널 중심 고급 개발자:** Claude Code를 주력으로 두고, 위험 명령 권한과 백업 정책을 먼저 정리한다.
- **IDE 중심 제품 개발자:** Cursor를 기본 작업면으로 쓰고, Context Usage Breakdown으로 rules/MCP/subagents 구성을 정리한다.
- **백로그와 반복 유지보수가 많은 팀:** Codex를 비동기 작업자처럼 쓰고, 테스트/AGENTS.md/검증 로그를 필수 계약으로 둔다.
- **혼합형 팀:** Cursor는 실시간 편집, Claude Code는 큰 리팩터/분석, Codex는 백그라운드 티켓으로 역할을 나눈다.
