## 1. Claude Code

### 강점

- 2026-04-22와 2026-04-23 이틀 연속 공개 릴리스가 나와 최근 3일 기준 공식 변화량이 가장 컸다.
- 서브에이전트, 세션 재개, MCP, 플러그인, 관측성, Windows/WSL 같은 “실제 작업 런타임” 개선이 눈에 띈다.
- 복잡한 레포를 길게 붙잡고 CLI 중심으로 해결하는 작업에서 여전히 가장 공격적인 방향성을 보인다.

### 약점

- 업데이트가 빠른 만큼 설정 표면과 운영 복잡도도 커진다.
- 기사 흐름상 Anthropic은 사용량 급증과 컴퓨트 압박의 직접적인 타격을 받고 있다.
- 보안 측면에서도 모델 성능만 믿기보다 런타임 권한 설계와 CI 환경 격리가 더 중요해졌다.

### 추천 사용 시나리오

- 대형 코드베이스 리팩터링, 장시간 디버깅, 여러 파일을 넘나드는 변경, CLI 자동화 중심 팀
- MCP/플러그인/정책 파일을 함께 관리할 수 있는 숙련된 개발 조직

### 비추천 사용 시나리오

- 토큰 예산과 rate limit 변동에 매우 민감한 조직
- IDE 시각 작업이 중심이고, 터미널 운영 지식이 약한 팀

### 최신 업데이트 반영 관찰

- 최근 릴리스는 “새 모델”보다 “세션 지속성, 플러그인 의존성, 권한/로그인 복구, Windows 최적화” 쪽에 무게가 있다.
- 즉 Claude Code는 단일 코딩 도우미보다 운영형 에이전트 런타임으로 진화하는 중이다.

### 현재 시점 총평

Claude Code는 이번 주에도 가장 강한 실행형 도구다. 다만 그 강점은 이제 모델 한 방보다는 긴 세션, 런타임 복원력, 플러그인/MCP 운용에 있다. 성능 상위권 도구인 것은 분명하지만, 운영 부담까지 감당할 팀에게 가장 잘 맞는다.

## 2. Cursor

### 강점

- 직전 공식 릴리스가 Canvases, CLI Debug Mode, Agents Window 개선처럼 시각적 작업과 다중 에이전트 조율에 초점을 맞췄다.
- IDE 안에서 에이전트를 병렬로 운용하고, 결과를 바로 비교·수정하는 흐름은 여전히 Cursor의 가장 큰 장점이다.
- GitHub Trending과 외부 기사 흐름을 함께 보면, 시장이 원하는 “오케스트레이션 레이어”에 가장 가까운 포지션을 유지하고 있다.

### 약점

- 2026-04-21~2026-04-24 기준 공식 changelog 신규 항목이 없어 이번 주 직접 신호는 약했다.
- 시장 경쟁이 거세지면서 자체 모델, 컴퓨트, 기업 기능, 가격 체계까지 동시에 방어해야 하는 부담이 크다.
- IDE 친화성이 강점이지만, 조직 차원의 자동화/백그라운드 운영에서는 Codex나 Claude Code보다 덜 직접적일 수 있다.

### 추천 사용 시나리오

- 일상 개발, UI/프론트엔드 반복 작업, 시각적 비교, 여러 에이전트를 동시에 띄우는 협업형 개발
- “수정-확인-재수정” 루프를 자주 도는 제품팀

### 비추천 사용 시나리오

- 최근 공식 릴리스 빈도를 높은 신뢰 지표로 삼는 조직
- 완전한 백그라운드 자동화나 장시간 비동기 실행을 최우선으로 두는 경우

### 최신 업데이트 반영 관찰

- 이번 주 Cursor는 새 항목보다 직전 대형 전환의 여파를 읽는 쪽이 맞다.
- 즉 “신규 기능 돌파”보다 “에이전트 창/CLI/시각 산출물” 노선을 계속 굳히는 단계로 보인다.

### 현재 시점 총평

Cursor는 여전히 가장 매끄러운 인터랙티브 작업면을 제공하는 도구에 가깝다. 이번 주 공식 신호는 약했지만, 방향성 자체는 흔들리지 않았다. 실시간 편집과 병렬 에이전트 비교가 핵심인 팀이면 아직도 가장 실전적이다.

## 3. OpenAI Codex

### 강점

- 2026-04-23~2026-04-24에 접근 플랜, RBAC, 플러그인 통제, Compliance API, rate card가 공식 문서에서 빠르게 정리됐다.
- 2026-04-16 제품 발표 이후 Codex는 컴퓨터 사용, 브라우저, 메모리, 자동화, SSH devbox, 플러그인까지 포괄하는 넓은 작업면을 갖췄다.
- 앱, CLI, IDE, 웹, 자동화까지 하나의 체계로 묶으려는 방향이 분명하다.

### 약점

- 장점이 넓은 만큼 무엇이 현재 핵심 가치인지 파악하는 데 시간이 걸린다.
- 이번 주 신호의 상당수는 기능 추가보다 플랜/과금/관리 문서 정리였고, 이는 비용 관리가 곧 제품 사용 경험이라는 뜻이기도 하다.
- 예산과 권한 체계를 같이 설계하지 않으면 “기능은 많은데 운영 기준이 없는” 상태가 되기 쉽다.

### 추천 사용 시나리오

- 백그라운드 작업, 자동화, 반복 업무, 여러 표면을 하나의 제품군으로 관리하고 싶은 팀
- 관리자 통제, 플러그인/App 권한, 조직 단위 배포 체계를 중요하게 보는 조직

### 비추천 사용 시나리오

- 아주 가벼운 코드 보조만 필요한 개인 사용자
- 토큰 과금과 플랜 차이를 계속 추적하기 어려운 작은 팀

### 최신 업데이트 반영 관찰

- 최근 신호는 “무엇을 더 할 수 있나”보다 “누가 어떤 플랜으로 어디까지 쓸 수 있나”에 가깝다.
- 이는 Codex가 기능 실험기에서 조직 배포형 제품군으로 이동 중이라는 뜻이다.

### 현재 시점 총평

Codex는 이번 주 가장 넓은 작업면을 가진 도구처럼 보인다. 다만 강점은 폭이고, 위험은 그 폭이 곧 복잡도라는 점이다. 반복 업무와 조직형 자동화까지 함께 묶고 싶다면 Codex가 가장 매력적이지만, 요금과 관리 체계를 같이 설계해야 한다.

## 추천 사용처 유형

- 코드베이스 이해와 큰 수정이 먼저면 Claude Code가 가장 유리하다.
- 하루 종일 IDE 안에서 빠르게 만들고 고치는 팀이면 Cursor가 가장 편하다.
- 자동화, 배치성 작업, 관리 통제까지 한 묶음으로 보고 싶으면 Codex가 가장 전략적이다.

## 이번 주 총평

이번 주 AI 코딩 도구 시장은 기능 경쟁만으로 읽으면 틀린다. 진짜 경쟁축은 보안 런타임, 토큰 사용량, 컴퓨트 공급, 관리자 통제, 플러그인 생태계다. Claude Code는 가장 강한 실행형, Cursor는 가장 매끄러운 인터랙티브형, Codex는 가장 넓은 운영형으로 정리된다.

## 주요 트렌드 관찰

1. 에이전트 시장은 이제 모델 경쟁에서 운영 경쟁으로 이동했다.
2. GitHub Trending 상위 신호는 컨텍스트 압축, 스킬 재사용, 학습 자료, 셸 기반 자동화처럼 주변 레이어가 강해지고 있음을 보여준다.
3. 기사 흐름은 보안과 컴퓨트 압박을 전면에 올리고 있어, 다음 분기에는 rate limit과 권한 관리가 구매 결정에 더 크게 작용할 가능성이 높다.
