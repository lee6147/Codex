Claude Code

Claude Code /goal Command: Setup Guide (2026)

실사용 가이드7일 확장

게시일: 2026-05-25 | 작성자/매체: Alex Fraser, AI Systems Lab / Medium

요약: 이 글은 Claude Code의 /goal을 단순한 "계속 진행" 버튼이 아니라 완료 조건과 evaluator loop가 붙은 장기 작업 제어 모델로 설명한다. 핵심은 evaluator가 도구를 실행하지 못하므로 완료 조건 안에 검증 출력이 대화에 남아야 한다는 점이다. PLAN.md를 먼저 만들고 acceptance criteria, turn budget, risk를 명시하라는 조언이 실무적이다.

실무 시사점: 장기 agent 작업은 "테스트가 통과했는가"보다 "테스트 명령과 결과가 transcript에 남았는가"가 중요하다. Claude Code뿐 아니라 Codex Goal mode에도 같은 검증 설계가 필요하다.

Claude Code 기사량 부족 메모

자료 부족확인 불가 항목 분리

2026-05-20부터 2026-05-26까지의 확장 창에서도 Claude Code만을 중심으로 다룬 고신뢰 신규 아티클은 충분하지 않았다. 따라서 도구별 목표 개수를 억지로 채우지 않고, 확인 가능한 /goal 운영 가이드 1건과 공식 changelog 맥락만 반영했다.

실무 시사점: Claude Code는 기사량보다 공식 changelog와 실제 workflow 문서의 품질을 우선해서 보는 편이 안전하다. 특히 장기 실행 기능은 홍보성 요약보다 완료 조건과 검증 증거 설계가 중요하다.

Cursor 3.5 brings multi-repo and no-repo Automations to the IDE's Agents Window

뉴스 기사7일 확장

게시일: 2026-05-21 | 작성자/매체: Pondero Newsdesk

요약: Cursor 3.5가 Automations를 IDE 안의 Agents Window로 가져오고, multi-repo와 no-repo 설정을 추가했다는 내용이다. 기존에는 web dashboard가 분리돼 있었지만 이제 agent work와 automation 관리가 같은 화면으로 묶인다. no-repo template는 Slack digest, product analytics, finance, customer health처럼 코드 밖 업무 신호까지 다룬다.

실무 시사점: Cursor는 editor를 넘어 업무 자동화 console을 지향한다. 팀은 repo 권한, 외부 SaaS 연결, agent run 비용을 함께 설계해야 한다.

Composer 2.5: benchmarks, prices and comparison

비교 리뷰7일 확장

게시일: 2026-05-22 업데이트 | 작성자/매체: DataCamp

요약: DataCamp는 Composer 2.5를 Cursor 안에서만 동작하는 coding-specialized model로 설명한다. 장기 코딩 세션, multi-file editing, terminal command 사용, tool use에 맞춰 훈련됐으며 CursorBench와 benchmark 수치 해석에는 한계가 있다고 짚는다. 가격은 frontier model보다 낮지만 Fast가 기본값이라 비용 체감은 설정에 따라 달라질 수 있다.

실무 시사점: Composer 2.5는 Cursor 내부 routine coding과 긴 agent session에 적합하지만, 외부 API 호출이나 독립 재현 benchmark가 필요한 조직에는 제약이 있다.

OpenAI Adds Appshots and Goal Mode to Codex

뉴스 기사7일 확장

게시일: 2026-05-22 | 작성자/매체: Andrés Martínez / aiHola

요약: aiHola는 Appshots가 macOS-only이고 Codex app build 26.519에 포함됐다고 정리한다. Goal mode가 beta를 벗어나 app, IDE extension, CLI에서 쓸 수 있게 됐고, 장기 실행 중 check-in, steer, pause가 가능하다는 점을 강조한다. 다만 multi-day run 안정성은 문서가 정량화하지 않는다고 선을 그었다.
