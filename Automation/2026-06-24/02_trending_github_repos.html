요약

이번 주 GitHub 트렌딩은 AI coding agent 본체보다 그 주변 인프라가 더 뜨거웠다. 재사용 가능한 skills, 코드베이스 memory, 넓은 웹 도달 범위, agent sandbox, skill security 검사가 동시에 상위권에 모였고, 이는 Claude Code·Cursor·Codex가 모두 최근 공식 업데이트에서 강조한 운영성 강화와 같은 방향을 가리킨다.

트렌드 해석

  • skills와 workflow packaging이 가장 강한 공통 축이었다. 이는 agent 사용이 개인 프롬프트에서 팀 단위 반복 자산으로 이동 중임을 뜻한다.
  • memory, browsing reach, sandbox가 동시에 상위권인 점은 “잘 쓰는 모델”보다 “오래 안전하게 일시키는 런타임”이 경쟁 중심이 됐다는 신호다.
  • security 계층도 강하게 떠올랐다. SkillSpector와 system prompt leak 계열 저장소는 공유 자산 자체가 공격 표면이 되었음을 보여준다.
  • Daily보다 weekly를 대표 뷰로 쓴 이유는 하루치 스파이크보다 이번 주 누적 수요가 개발 생산성 인프라 방향을 더 안정적으로 보여줬기 때문이다.
