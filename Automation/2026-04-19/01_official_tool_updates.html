Claude Code

v2.1.113 / v2.1.114: 실행 안정성, 보안 규칙, 리뷰 속도 개선

확인 완료 공식 changelog 2026-04-17~2026-04-18

2026-04-17 공개된 v2.1.113에서는 CLI가 번들 자바스크립트 대신 네이티브 Claude Code 바이너리를 호출하도록 바뀌었고, 특정 도메인을 막는 sandbox.network.deniedDomains 설정이 추가됐다. 같은 릴리스에는 /ultrareview 병렬화, PowerShell 및 Bash 동작 보정, 여러 보안 규칙 강화도 포함됐다. 2026-04-18 v2.1.114는 권한 대화상자 크래시 1건을 즉시 수정했다.

업무 시사점

Claude Code는 새 기능을 더하는 것만이 아니라, 긴 세션과 병렬 작업에서 생기는 실제 마찰을 빠르게 줄이는 방향으로 움직이고 있다. 팀 사용 관점에서는 성능보다 “허용 규칙, 리뷰 흐름, 샌드박스 정책”이 점점 더 핵심 경쟁력이 되고 있다는 뜻이다.

확인 완료 공식 changelog 2026-04-16

2026-04-16 공개된 v2.1.111은 Claude Opus 4.7 xhigh, Max 구독자용 Auto mode, /less-permission-prompts, /ultrareview를 도입했다. Windows용 PowerShell 도구 롤아웃과 읽기 전용 Bash 허용 범위 완화도 같은 릴리스에 포함됐다. 직후 v2.1.112는 Auto mode에서 발생하던 가용성 문제를 1줄 수정으로 바로 막았다.

업무 시사점

이번 묶음 업데이트는 “더 똑똑한 모델”보다 “허용 프롬프트 감소, 코드 리뷰 자동화, 고노력 모드” 같은 운영 레버를 전면에 올렸다. Claude Code가 강한 것은 이제 단순 생성 품질보다 장기 실행과 검증 루프의 밀도라는 점이 더 분명해졌다.

최신 공개 기준선: Canvases

확인 완료 공식 changelog 2026-04-15

2026-04-15 공개된 Canvases는 Cursor가 에이전트 응답을 표·박스·다이어그램·차트·diff·to-do가 섞인 인터랙티브 산출물로 내놓게 만든 기능이다. 이 산출물은 Agents Window의 터미널·브라우저·소스컨트롤 옆에서 지속적으로 남는다. 2026-04-13의 3.1 업데이트와 결합하면 Cursor의 최근 방향은 “병렬 에이전트 관리 + 시각적 산출물”이다.

업무 시사점

새 공식 글이 조사 구간 안에는 없었지만, 최신 기준선만 봐도 Cursor의 초점은 코드 편집보다 “보여주고 비교하고 조율하는 작업면”에 있다. 즉, Cursor는 여전히 오케스트레이션 계층에서 해석하는 편이 맞다.

OpenAI Codex

Codex for (almost) everything

확인 완료 공식 제품 글 2026-04-16

2026-04-16 OpenAI는 Codex의 대규모 제품 업데이트를 발표했다. 핵심은 background computer use, 더 많은 앱/도구 연동, 이미지 생성, 메모리, 반복 작업과 장기 작업 자동화다. 동시에 PR 리뷰, 여러 파일·터미널 보기, 원격 devbox SSH 연결, 인앱 브라우저도 함께 강화됐다.

업무 시사점

Codex의 무게중심은 더 이상 코드 작성 보조 하나가 아니다. “컴퓨터 위에서 실제 작업을 이어가는 에이전트”라는 넓은 개념으로 재정의되고 있으며, 이는 Claude Code와 Cursor 사이의 경계를 의도적으로 허무는 움직임으로 읽힌다.

도움말 문서 갱신: 플랜 범위, 앱, 자동 코드 리뷰 정리

확인 완료 공식 도움말 2026-04-19 갱신 확인

OpenAI Help Center의 “Using Codex with your ChatGPT plan” 문서는 2026-04-19 기준 최근 수 시간 안에 갱신된 상태로 확인됐다. 이 문서에는 Codex 앱, 클라우드 위임, GitHub 자동 코드 리뷰, Free/Go 한시 포함, 2x rate limits, 로컬 도구와 Codex Cloud의 관계가 명시되어 있다.
