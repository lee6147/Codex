## 1. Claude Code

### 강점

- 2026-04-16~2026-04-18 구간에 확인 가능한 공식 업데이트가 가장 촘촘했다. Opus 4.7 xhigh, Auto mode, `/less-permission-prompts`, `/ultrareview`, 네이티브 바이너리 전환, 보안 규칙 강화가 짧은 간격으로 붙었다.
- 권한, 리뷰, 장기 세션, 허용 규칙 같은 실제 운영 마찰을 줄이는 방향이 분명하다.
- CLI 중심이라 팀 규칙 파일, 메모리, 검증 루프를 직접 통제하기 좋다.
- 실전 사용기 기준으로 리팩터링, 테스트 스캐폴딩, 스택 트레이스 해석, 문서화 같은 텍스트 중심 작업에서 강점이 선명하다.

### 약점

- GUI 기반 시각 작업, 실기기 검증, 브라우저/앱 상호작용까지 넓게 다루려면 별도 도구와 조합해야 하는 경우가 많다.
- 기능이 changelog, 문서, 데스크톱 앱, CLI 흐름으로 분산돼 있어 따라가는 비용이 있다.
- 강력한 대신 사용자가 프로세스를 직접 설계해야 하므로 초반 진입 장벽이 낮지는 않다.

### 추천 사용 시나리오

- 리포 규칙과 검증 루프를 강하게 걸고 싶은 팀
- 리팩터링, 테스트, 코드 리뷰, 반복 실행을 터미널 중심으로 운영하는 환경
- 장기 작업을 잘게 나눠 재개하고, 허용 규칙을 점진적으로 다듬어야 하는 조직

### 쇠퇴하거나 덜 맞는 시나리오

- 화면 위에서 바로 비교하고 설명하는 시각적 협업이 가장 중요한 팀
- IDE 안에서 모든 것을 끝내고 싶은 사용자

### 최신 업데이트 반영 관찰

- 이번 주 Claude Code의 메시지는 “더 좋은 모델”보다 “더 적은 승인 마찰 + 더 강한 검증”이었다.
- `/ultrareview`와 `/less-permission-prompts`는 생산성보다 운영 통제에 더 가까운 기능이며, 이 점이 곧 차별점이다.

### 현재 시점 총평

Claude Code는 여전히 가장 강한 실행 엔진에 가깝다. 특히 팀 기준, 검증, 반복 실행, 리뷰 자동화를 직접 설계하고 싶은 사용자에게는 가장 선명한 선택지다.

## 2. Cursor

### 강점

- Agents Window, Tiled layout, Canvases까지 이어지는 흐름이 명확하다. 여러 에이전트를 띄워 보고 비교하고 정리하는 작업면으로 가장 적극적으로 움직인다.
- InfoQ 분석이 짚은 대로, 제품의 정체성이 “IDE에 AI를 붙인 것”에서 “에이전트 관리 화면”으로 바뀌고 있다.
- 결과물 시각화와 팀 공유 측면에서 강하다.

### 약점

- 2026-04-16 ~ 2026-04-19 구간 안에 신규 공식 항목이 확인되지 않아 이번 주 공식 변화량은 상대적으로 얕다.
- IDE 정체성을 좋아했던 사용자에게는 agent-first 전환이 피로하게 느껴질 수 있다.
- 모델 경쟁력과 비용 구조는 여전히 하부 인프라와 공급망에 크게 좌우된다. xAI GPU 기사도 그 점을 보여준다.

### 추천 사용 시나리오

- 여러 에이전트 결과를 한 화면에서 비교하고 조율해야 하는 제품/프론트엔드 팀
- 브라우저, diff, to-do, 도식화 같은 시각적 산출물을 자주 활용하는 조직
- “직접 코딩”보다 “에이전트 매니지먼트”가 중요해진 사용자

### 쇠퇴하거나 덜 맞는 시나리오

- 강한 로컬 규칙 파일과 텍스트 기반 제어만으로 모든 것을 관리하려는 사용자
- UI 변화를 부담스러워하고 전통 IDE 감각을 더 중요하게 보는 팀

### 최신 업데이트 반영 관찰

- 새 공식 글이 없는 구간에서도 Cursor의 방향은 바뀌지 않았다. 최신 기준선은 여전히 Canvases와 병렬 에이전트 관리다.
- 즉, Cursor는 실행 품질 단일 승부보다 오케스트레이션 UX에서 점유율을 확보하려는 쪽으로 해석하는 것이 맞다.

### 현재 시점 총평

Cursor는 지금 가장 “작업면 지향적”인 도구다. 에이전트를 시각적으로 운영하고 비교하는 경험이 중요하면 강력하지만, 텍스트 기반 제어만으로 밀어붙이려는 사용자에게는 덜 매력적일 수 있다.

## 3. OpenAI Codex

### 강점

- 2026-04-16 업데이트는 범위 확장이 매우 컸다. 컴퓨터 사용, 메모리, 자동화, 브라우저, 이미지 생성, PR 리뷰 대응, 여러 터미널과 파일 미리보기까지 묶였다.
- 2026-04-19 기준 Help Center 문서도 빠르게 갱신돼 앱, 클라우드 위임, GitHub 코드 리뷰, 플랜 포함 범위가 정리돼 있다.
- 로컬 도구, IDE, 앱, 클라우드, GitHub 리뷰를 한 서비스 레이어로 묶는 전략이 분명하다.
- OpenAI Agents SDK와 같은 공식 오픈소스 인프라도 빠르게 두꺼워지고 있다.

### 약점

- 기능 폭이 넓어진 만큼 권한 경계와 작업 분류를 잘못 잡으면 복잡성이 빨리 커진다.
- “무엇이 어디에서 동작하는지”를 파악해야 할 표면이 많다. 앱, 클라우드, 로컬, Help Center, 제품 글, SDK를 함께 봐야 한다.
- CLI 하나만으로 단순하고 엄격한 워크플로를 원하는 사용자에게는 과하게 넓게 느껴질 수 있다.

### 추천 사용 시나리오

- 로컬 작업과 클라우드 위임을 함께 쓰고 싶은 팀
- 프론트엔드 반복, 앱 테스트, 리뷰 대응, 문서/디자인 산출물까지 한 흐름으로 묶고 싶은 조직
- 제품, 디자인, 개발, 운영을 하나의 에이전트 작업면으로 연결하려는 환경

### 쇠퇴하거나 덜 맞는 시나리오

- 넓은 제품면보다 엄격한 텍스트 중심 로컬 실행이 최우선인 사용자
- 복잡한 권한/기능 조합을 관리할 여력이 적은 작은 팀

### 최신 업데이트 반영 관찰

- Codex는 이번 주에 “코드를 대신 써주는 모델”에서 “컴퓨터 위에서 장기 작업을 이어가는 작업 시스템”으로 설명 범위를 넓혔다.
- 이는 Claude Code와 Cursor가 각각 강하던 실행 계층과 작업면 계층을 동시에 넘보는 움직임이다.

### 현재 시점 총평

Codex는 가장 빠르게 넓어지고 있다. 특히 앱·클라우드·로컬·리뷰·자동화를 한데 묶어 운영하려는 사용자라면 매력이 크다. 다만 넓어진 표면을 제대로 제어할 수 있는 팀에게 더 잘 맞는다.

## 이번 주 총평

이번 주 시장은 단일 승자 게임처럼 보이지 않았다. Cursor는 에이전트 오케스트레이션 화면을 밀고, Claude Code는 실행과 검증을 다듬고, Codex는 작업면 자체를 넓혔다. 세 도구가 점점 더 같은 문제를 향해 움직이지만, 강한 위치는 아직 다르다.

GitHub Trending도 같은 신호를 보냈다. 상위권에 오른 것은 모델 래퍼보다 스킬 체계, 멀티에이전트 조직도, 장기 실행 메모리, 운영 자동화였다. 지금 중요한 질문은 “어느 모델이 더 똑똑한가”보다 “에이전트를 어떤 절차와 작업면으로 운영할 것인가”에 가깝다.

## 주요 트렌드 관찰

1. 스킬 팩과 작업 하네스가 모델 자체 못지않게 중요한 차별화 포인트가 되고 있다.
2. 메모리와 자동화는 부가 기능이 아니라 장기 실행 에이전트의 기본 조건으로 올라왔다.
3. 실행 엔진과 오케스트레이션 화면이 분리되면서, 하나의 도구보다 “스택 조합”이 더 현실적인 전략이 되고 있다.
4. 운영 자동화와 SRE 같은 후속 시장이 빠르게 커지고 있어, AI 코딩 툴의 경쟁 범위가 개발 단계를 넘어가고 있다.
