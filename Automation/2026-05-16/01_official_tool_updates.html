Claude Code

확인 불가 포함공식 changelog

v2.1.142 계열: background agent 구성과 안정성 보강

무엇이 바뀌었나
최신 공식 changelog는 claude agents에 --add-dir, --settings, --mcp-config, --plugin-dir, --permission-mode, --model, --effort 등 background session 구성 플래그가 추가됐다고 적고 있다.
왜 중요한가
Claude Code가 단일 터미널 대화형 도구에서 여러 백그라운드 세션을 운영하는 에이전트 런타임으로 이동하고 있음을 보여준다.

기능 내용은 공식 changelog에서 확인 완료. 다만 공식 changelog/Docs 페이지가 개별 배포일을 본문에 노출하지 않아 발표일은 확인 불가로 기록한다.

Cursor

확인 완료2026-05-13

Development environments for cloud agents

무엇이 바뀌었나
Cloud agents와 automations가 multi-repo environments를 지원하며, 하나의 환경에 작업에 필요한 여러 저장소를 구성할 수 있다.
환경 구성
Dockerfile 기반 환경 정의가 개선됐고 build secrets와 layer caching이 보강됐다. Cursor는 캐시 적중 빌드가 70% 더 빠르다고 설명한다.
보안/운영
환경별 version history, rollback 권한 제한, audit log, environment-level egress/secrets scoping이 추가됐다.

실사용 의미: Cursor Cloud Agent는 이제 단일 레포 자동수정보다 팀의 실제 개발환경을 복제한 장시간 실행 작업에 더 적합해졌다.

발표일: 2026-05-13 · 출처: Cursor Changelog · 상태: 확인 완료

OpenAI Codex

확인 완료2026-05-14

Codex in ChatGPT mobile app preview

무엇이 바뀌었나
OpenAI는 Codex가 ChatGPT 모바일 앱 preview로 들어와 iOS/Android에서 active thread, approvals, diffs, terminal output, screenshots, test results를 확인할 수 있다고 발표했다.
원격 실행 구조
파일, credentials, permissions, local setup은 Codex가 실행되는 machine에 남고, 모바일은 secure relay layer를 통해 상태와 승인 흐름을 동기화한다.
운영 기능
Remote SSH는 일반 제공, Hooks는 전 플랜에서 사용 가능, programmatic access tokens는 Enterprise/Business에서 제공된다고 명시됐다.

실사용 의미: Codex는 데스크톱/CLI 중심의 로컬 에이전트에서 모바일 승인과 원격 환경을 포함한 장시간 작업 운영 도구로 확장 중이다.
