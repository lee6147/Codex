## 1. Claude Code

### 강점

- 최신 공식 changelog 최상단이 2.1.179로 올라가며 연결 끊김, 원격 세션, WSL2 스크롤 같은 실제 마찰을 줄였다.
- 권한 규칙과 세션 지속성 개선이 이어져 대규모 CLI 운영에 유리하다.
- Disney, Paramount 사례에서 기업 도입 생산성 신호가 계속 확인된다.

### 약점

- raw changelog에 버전별 게시일이 명시되지 않아 날짜 추적성이 약하다.
- 도입이 넓어질수록 spend tracking, usage analytics, release quality gate가 함께 붙는다.
- "많이 쓰는 것" 자체보다 "어디까지 허용할 것인가"가 더 큰 관리 문제로 떠오른다.

### 추천 사용 시나리오

- 터미널 중심 작업, 대규모 코드베이스 탐색, 권한 규칙과 작업 절차를 세밀하게 설계해야 하는 팀.
- 한 번의 답변보다 멀티스텝 실행과 검증 루프가 중요한 작업.

### 현재 시점 총평

Claude Code는 여전히 가장 강한 CLI 감각을 주는 축에 있다. 다만 2026-06-18 시점의 경쟁력은 "생성 품질" 하나가 아니라 권한, 세션, 예산 통제까지 포함한 운영 완성도에서 나온다.

## 2. Cursor

### 강점

- 2026-06-10 공식 changelog 기준 Bugbot 리뷰 시간이 약 90초까지 내려갔고, 비용은 약 22% 줄고, 발견 버그 수는 10% 늘었다.
- IDE 안에서 생성, 수정, 리뷰를 짧은 루프로 돌리는 경험이 여전히 가장 분명하다.
- SpaceX의 600억 달러 인수 뉴스로 시장 영향력과 자본력 기대가 크게 높아졌다.

### 약점

- 최근 3일 기준 새 공식 changelog 항목은 없어서, 공식 레이어의 최신도는 다소 정체돼 보인다.
- 대기업 도입 기사에서 token usage 통제와 실패한 AI-coded 제품 감소 요구가 함께 등장한다.
- IDE 강점은 분명하지만, headless 운영성과 통제 측면에서는 비교 대상보다 설명이 더 필요하다.

### 추천 사용 시나리오

- IDE 안에서 빠른 반복, 리뷰 자동화, UI/프로덕트 개발 속도 개선이 중요한 팀.
- PR 전 단계에서 빠르게 결함을 잡고 싶은 조직.

### 현재 시점 총평

Cursor는 "잘 쓰는 코드 생성기"보다 "빠른 편집-리뷰 시스템"으로 이해하는 편이 맞다. 이번 주에는 기능 기사보다 시장 포지셔닝 기사에서 더 크게 존재감을 보였다.

## 3. OpenAI Codex

### 강점

- 2026-06-16 공식 changelog에서 Computer Use, Chrome extension, Memories, Chronicle 접근이 넓어졌다.
- 2026-06-11 app 26.609은 Browser Developer mode와 Windows per-app access controls를 추가해 브라우저/앱 디버깅 면을 강화했다.
- Ona 인수 기사와 공식 changelog가 모두 "장기 실행 가능한 에이전트 환경"이라는 같은 방향을 가리킨다.

### 약점

- 최근 7일 창 안의 제3자 기사량이 Claude/ Cursor보다 적어 외부 담론 밀도는 낮았다.
- 실행 표면이 넓어질수록 권한 관리와 감사 가능한 로그 체계의 중요성이 함께 커진다.
- 코드 생성 성능을 직접 비교해 주는 최신 외부 검증 기사는 이번 주 범위에서 충분하지 않았다.

### 추천 사용 시나리오

- 브라우저 작업, 데스크톱 조작, 장기 실행, 기억 유지까지 묶은 agent workflow를 설계하려는 팀.
- 코드 작성 이후 검증, 디버깅, 운영 자동화를 한 도구 안에서 연결하고 싶은 경우.

### 현재 시점 총평

Codex의 이번 주 메시지는 "코드를 더 잘 쓰는가"보다 "더 오래, 더 넓은 실행 환경에서 일할 수 있는가"에 가깝다. 장기 작업 인프라가 강해질수록 차별성이 커질 가능성이 있다.
