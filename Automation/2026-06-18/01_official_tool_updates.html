확인 방식

이번 공식 레이어의 공통점은 "더 똑똑한 모델"보다 검토 루프, 권한 제어, 브라우저/데스크톱 실행, 장기 작업 지속성 쪽에 무게가 실린다는 점이다.

요약

2026-06-18 기준으로 새 공식 신호가 가장 뚜렷한 쪽은 OpenAI Codex다. 2026-06-16에는 EEA, 영국, 스위스 대상으로 Computer Use, Chrome extension, Memories, Chronicle 접근성이 넓어졌고, 2026-06-11에는 Browser Developer mode와 Windows용 Computer Use 제어가 추가됐다. Claude Code는 최신 changelog 최상단이 2.1.179로 올라가며 연결 끊김, WSL2 스크롤, 원격 세션 플러그인 로딩 같은 마찰 감소에 집중했다. Cursor는 최근 3일 안의 새 공식 공지는 없었지만, 가장 최신 공식 항목인 2026-06-10 Bugbot 개선과 2026-06-04 Canvas Design Mode가 여전히 제품 방향을 대표한다.

주의 사항

Claude Code raw changelog는 버전별 게시일을 분리해서 주지 않는다. 따라서 이 보고서에서는 "2026-06-18에 최신 항목 2.1.179를 확인했다"는 표현만 쓰고, 항목 날짜는 확인 불가로 둔다. Cursor는 2026-06-10 이후 새 공식 changelog 항목이 확인되지 않았다.

Claude Code

최신 공식 항목은 2.1.179, 초점은 세션 마찰 감소

공식 changelog 2026-06-18 확인 항목 게시일 확인 불가

Anthropic의 raw changelog 최상단은 2.1.179다. 이 항목은 mid-stream connection drop 시 partial responses 보존, WSL2 on Windows Terminal/VS Code 스크롤 회귀 수정, 대규모 allowRead/denyRead glob로 세션이 무거워지는 문제, 원격 세션 background task가 멈춘 것처럼 보이는 문제, 원격 세션 plugin loading 성능 등을 고쳤다.

새로운 대표 기능보다 "이미 돌고 있는 세션을 덜 깨지게 만드는" 성격이 강하다. CLI 중심 사용자에게는 체감 품질 개선이 곧 생산성이다.

직전 2.1.178의 메시지는 권한 규칙 정교화

권한 규칙 운영 안전성 연속 업데이트

같은 changelog의 직전 항목 2.1.178은 Tool(param:value) 문법 기반의 permission rule 정교화가 핵심이다. 즉, Claude Code의 최근 공식 레이어는 새 모델 홍보보다 권한 제어, 세션 지속성, 플러그인/원격 실행 안정화에 계속 투자하고 있다.

기업 배치에서는 이 방향이 중요하다. 실무 채택이 늘어날수록 "뭘 할 수 있나"보다 "어떻게 통제하고 재현하나"가 더 큰 질문이 되기 때문이다.

Cursor

2026-06-10 최신 공식 항목: Bugbot이 더 빠르고 더 싸고 더 잘 잡는다

공식 changelog 2026-06-10 리뷰 자동화

Cursor changelog 최신 항목은 2026-06-10이다. Bugbot 평균 리뷰 시간은 약 5분에서 약 90초로 줄었고, 리뷰당 발견 버그 수는 0.56에서 0.62로 늘었으며, 실행 비용은 약 22% 낮아졌다고 밝혔다. 여기에 /review, /review-bugbot, /review-security를 push 전에 돌릴 수 있고, GitHub/GitLab PR과 같은 diff를 중복 검토하지 않도록 동기화한다.

Cursor의 공식 메시지는 여전히 "IDE 안의 생성"보다 "배포 전 리뷰 자동화의 ROI"에 가깝다.

2026-06-04 보강점: Canvas Design Mode와 context usage report

Canvas Design Mode Context 가시화

2026-06-04 항목은 Canvas Design Mode와 context usage report를 밀었다. 사용자는 canvas에서 UI 요소를 직접 지목하며 수정 지시를 줄 수 있고, context explorer로 시스템 프롬프트, rules, skills, tool definitions가 토큰을 어떻게 쓰는지 시각적으로 본다.

Cursor는 "모델이 답을 잘 내는가"보다 "편집과 리뷰 과정이 얼마나 빨리 반복되나"를 계속 제품화하고 있다.

OpenAI Codex

2026-06-16: EEA, 영국, 스위스에서도 앱 기능이 넓어졌다

공식 changelog 2026-06-16 지역 가용성 확대

OpenAI Developers의 Codex changelog에 따르면 2026-06-16 항목에서 EEA, 영국, 스위스 사용자에게 Computer Use, Codex Chrome extension, Memories, Chronicle 접근이 확대됐다. Memories는 선호 설정과 반복 워크플로를 기억하고, Chronicle은 최근 화면 맥락에서 기억을 빌드하는 opt-in preview다.

즉 Codex는 단순 코드 생성기를 넘어, 브라우저와 데스크톱 문맥을 오래 끌고 가는 작업 환경 쪽으로 제품 범위를 계속 넓히고 있다.

2026-06-11 app 26.609: Browser Developer mode와 Windows 제어 강화

공식 changelog 2026-06-11 브라우저 디버깅

2026-06-11 app 26.609 항목은 Browser use용 Developer mode, /init, Enterprise용 Computer Use, Windows의 per-app access controls를 추가했다. Developer mode는 CDP 접근을 통해 네트워크, 콘솔, 런타임 오류, 페이지 상태를 더 깊게 디버깅하게 한다.

Codex의 공식 방향은 브라우저 작업과 데스크톱 제어를 연결해 "코드 작성 이후 검증"까지 한 도구 안으로 묶는 데 있다.
