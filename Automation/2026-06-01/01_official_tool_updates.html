Claude Code 공식 변경

2.1.159 - 2026-05-31

공식 changelog에는 내부 인프라 개선으로 표시되어 있으며 사용자 노출 기능 변화는 없다고 정리되어 있다.

2.1.158 - 2026-05-30

Auto mode가 Bedrock, Vertex, Foundry 환경의 Opus 4.7 및 Opus 4.8에서도 사용 가능해졌다. 활성화 조건은 CLAUDE_CODE_ENABLE_AUTO_MODE=1 설정이다.

2.1.157 - 2026-05-29

  • .claude/skills 아래 플러그인이 marketplace 없이 자동 로드된다.
  • claude plugin init <name> 명령으로 로컬 skill 플러그인을 스캐폴딩할 수 있다.
  • OTEL 세부 로깅이 켜진 경우 tool decision telemetry에 bash 명령, MCP/skill 이름 같은 tool parameter가 포함된다.
  • Windows 11 WSL 이미지 붙여넣기, 스크린샷 붙여넣기, Windows Explorer 드래그 이미지 처리가 개선됐다.

해석: Claude Code는 단순 기능 추가보다 skill/plugin 배포와 장시간 세션 복구, Windows 터미널 안정성 개선에 초점이 있다.

OpenAI Codex 공식 변경

Codex app 26.527 - 2026-05-29

  • Computer Use가 Windows에서 동작한다. Codex가 foreground Windows 데스크톱 앱을 보고 클릭하고 입력할 수 있다.
  • Remote control이 Windows 기기를 지원한다. iOS/Android의 ChatGPT 또는 Mac의 Codex에서 Windows 기기 작업을 시작하고 진행 상황을 확인할 수 있다.
  • Profile 섹션에 프로필 상세, 사용량, token activity가 표시된다.
  • 로컬 프로젝트와 worktree의 thread coordination, 과거 thread 검색 확장이 추가됐다.

Codex CLI 0.135.0 - 2026-05-28

  • codex doctor가 환경, Git, 터미널, app-server, thread inventory 진단 정보를 더 풍부하게 보고한다.
  • /status가 remote transport 연결 정보를 표시한다.
  • Vim mode text-object 편집, permission profile 표시, Python SDK sandbox preset이 추가됐다.

해석: Codex는 Windows 사용자와 원격 작업자를 직접 겨냥했다. 로컬 자동화, 원격 handoff, 진단 가능성이 한 번에 커졌다.

Cursor 공식 변경

Cursor 3.6 - 2026-05-29

Auto-review Run Mode가 추가됐다. Cursor가 더 오래 작업하면서 승인 프롬프트를 줄이고, Shell/MCP/Fetch tool call을 위험도에 따라 자동 허용, sandbox 실행, classifier subagent 검토, 사용자 승인 요청으로 분기한다.

  • 설정 경로: Settings > Cursor Settings > Agents > Run Mode.
  • classifier agent에 사용자 지정 지침을 줄 수 있다.
  • 목표는 YOLO식 무제한 실행이 아니라 승인 빈도와 안전성의 균형이다.

해석: Cursor는 IDE 중심 agent 경험에서 승인 정책을 제품화하고 있다. 반복적 local edit/test 작업에 특히 유리하다.
