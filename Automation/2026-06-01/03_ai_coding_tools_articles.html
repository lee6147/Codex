Claude Code

공식Claude Code Changelog - 2026-05-31에는 사용자 노출 변화 없는 내부 인프라 개선, 2026-05-30에는 Bedrock/Vertex/Foundry의 Auto mode 지원 확대, 2026-05-29에는 skill/plugin 자동 로드와 Windows/WSL 개선이 확인됐다.

최근 해설Claude Media: Claude Code v2.1.158 Auto mode provider support - 2026-05-30 릴리스를 Auto mode 관점에서 설명한다. 공식 1차 출처는 아니므로 세부 사실은 공식 changelog와 대조해야 한다.

OpenAI Codex

최근 해설AI Tools Recap: OpenAI Codex Windows Update - Windows computer use와 mobile remote control의 의미를 제품 해설 형태로 정리했다. 보조 출처로만 활용한다.

Cursor

공식Cursor Changelog - 2026-05-29 Cursor 3.6에서 Auto-review Run Mode가 추가됐다. Shell, MCP, Fetch tool call을 allowlist, sandbox, classifier subagent, 사용자 승인으로 분기한다.

최근 해설Totalum: Cursor Auto-review Run Mode - Auto-review를 YOLO, Claude Code, Codex CLI, Cline과 비교한다. 비교 관점은 유용하지만 최종 사실 확인은 Cursor 공식 문서가 기준이다.

비교 읽기

비교Built In: Claude Code vs. Codex vs. Cursor vs. GitHub Copilot - 2026-05-13 글이지만 최근 비교 맥락을 제공한다. 기사 결론은 도구별 장단점보다 “AI 도구가 과잉 설계할 수 있으므로 전문가 검토가 필요하다”는 위험 신호로 읽는 편이 적절하다.

비교Kingy AI: Codex vs Claude Code vs Cursor - 2026-05-27 글로, Cursor는 IDE 기본값, Claude Code는 terminal-first agent, Codex는 OpenAI/ChatGPT ecosystem 통합 흐름으로 구분한다.

실무 시사점

  • 안전한 장시간 실행이 경쟁축이다. Cursor Auto-review, Claude Auto mode, Codex permission/diagnostic 개선은 같은 문제를 다른 UI에서 푼다.
  • Windows 자동화가 다시 중요해졌다. Codex 26.527과 Claude Code WSL 개선은 Windows를 주변 플랫폼이 아니라 주요 agent 실행 환경으로 다룬다.
  • 기사보다 공식 changelog를 우선해야 한다. 최근 2일 기사량이 부족했고, 일부 보조 글은 제품 해설 성격이 강해 공식 출처와 대조가 필요하다.
  • 다음 주에는 “승인 정책이 실제로 덜 묻고 덜 위험한가”를 로컬 테스트로 확인하는 것이 가장 생산적이다.
