Claude Code 품질 저하 보고에 대한 공식 포스트모템

official postmortem 2026-04-23 Anthropic Engineering

Anthropic는 최근 Claude Code 품질 저하 보고가 세 가지 다른 변경의 결합 효과였다고 설명했다. 기본 reasoning effort를 낮춘 결정, idle session 사고 이력 정리 버그, 과도한 길이 제한 프롬프트가 각각 다른 시점에 다른 사용자에게 영향을 줬다고 정리한다.

핵심은 단순 사과문이 아니라 구체적 원인과 수정 날짜가 붙어 있다는 점이다. 2026-04-20까지 문제를 되돌렸고, 더 넓은 eval, prompt 변경 통제, 공개 빌드 기준 내부 사용 강화, Code Review 맥락 개선까지 후속 조치를 적었다.

왜 읽을 만한가: Claude Code를 실무에 쓰는 팀이라면 “모델이 나빠졌다”는 인상 뒤에 실제로 어떤 제품 레이어 문제가 있었는지 파악하는 데 가장 직접적인 자료다.

Keeping the Cursor app stable

official technical post 2026-04-21 Cursor Research

Cursor는 앱이 메모리 부족으로 죽는 문제를 어떻게 줄였는지 공개했다. crash telemetry, crash watcher, heap snapshot, allocation profiling, feature flag 기반 실험, Bugbot rules, 자동 롤백까지 안정성 개선 파이프라인을 자세히 설명한다.

문서에서 late-February 대비 OOM-per-session 80% 감소, 2026-03-01 이후 OOM-per-request 73% 감소 수치를 제시한 점이 특히 강하다. 에이전트 창, browser use, instant grep 같은 무거운 기능을 계속 넣으면서도 데스크톱 앱이 버텨야 한다는 현실을 보여준다.

왜 읽을 만한가: Cursor를 쓰는 팀뿐 아니라, 에이전트형 코딩 앱을 만드는 쪽에도 “기능 확대와 안정성 관리가 어떻게 묶이는가”를 보여주는 실전 사례다.

OpenAI Academy: Working with Codex / Automations

official workflow guide 2026-04-23 OpenAI Academy

OpenAI Academy는 2026-04-23에 Codex 사용 흐름을 설명하는 자료 묶음을 올렸다. Working with Codex는 threads, projects, settings, steer, 로컬 작업 제약을 설명하고, Automations는 schedule과 trigger 기반 반복 작업 예시를 보여준다.

내용 자체는 초보자용이지만, Codex가 어디에 강한지 매우 분명하게 드러난다. 즉, 대화형 보조보다 파일과 반복 작업을 넘겨주는 구조, 그리고 로컬 프로젝트 폴더와 권한 경계 안에서 굴리는 운영 방식이다.

왜 읽을 만한가: Codex를 “하나 더 있는 코딩 챗봇”이 아니라 백그라운드 작업자와 자동화 도구로 이해할 때 가장 짧게 감을 잡을 수 있는 공식 자료다.
