Claude Code 관련 글

Anthropic releases its first Mythos-class model Claude Fable

뉴스 기사2026-06-09The Verge

Anthropic이 Claude Fable 5와 제한된 Mythos 5를 공개했다는 보도다. Fable 5는 소프트웨어 엔지니어링, 지식 업무, 시각 작업에서 강점을 보이며, 위험 영역에는 안전 제약과 fallback이 붙는 구조로 설명된다.

실무 시사점: Claude Code 사용자는 모델 성능 상승만 볼 것이 아니라 refusal, fallback, 안전 분류로 인한 워크플로 중단 가능성을 함께 봐야 한다. 보안/바이오/역공학성 요청이 섞인 자동화는 사전 설계가 필요하다.

Anthropic says Claude now writes more than 80 percent of its merged code

뉴스 기사2026-06-06Tom's Hardware

Anthropic 내부에서 Claude가 병합 코드의 80% 이상을 작성한다는 주장과 recursive self-improvement 우려를 다룬 기사다. 내부 수치와 경고성 메시지가 섞여 있어 독립 검증은 필요하지만, AI 코딩 비용과 통제 논의를 키우는 기사다.

실무 시사점: Claude Code를 대규모로 쓰는 조직은 생산성 수치만 추적하면 부족하다. 인간 리뷰 비율, rollback, 테스트 실패율, 비용 한도를 같이 측정해야 한다.

Cursor 관련 글

Design Mode Improvements

공식 글2026-06-05Cursor

Cursor 공식 changelog는 브라우저 Design Mode에서 multi-select와 voice input을 추가했다고 설명한다. 에이전트가 선택한 요소의 코드, 주변 레이아웃, 시각 관계를 함께 이해하게 한다는 점이 핵심이다.

실무 시사점: Cursor는 최근 기사보다 공식 업데이트가 더 정보량이 높다. UI 작업자는 텍스트 프롬프트보다 화면 위 selection과 음성 피드백을 결합하는 작업법을 실험할 가치가 있다.

Custom stores, custom tools, and auto-review for the Cursor SDK

공식 글2026-06-04Cursor

Cursor SDK가 custom tools, auto-review, JSONL/custom store, nested subagents를 지원한다는 공식 글이다. IDE 안의 상호작용형 에이전트와 별개로, headless agent를 CI나 내부 스크립트에 넣는 방향이 강화됐다.

실무 시사점: Cursor를 도입한 팀은 편집기 UI만 평가하지 말고 SDK 런 기록, permission rule, JSONL 로그 보관 방식을 함께 검토해야 한다.

OpenAI Codex 관련 글

Codex for every role, tool, and workflow

공식 글2026-06-02OpenAI

OpenAI가 Codex를 소프트웨어 개발자 전용 도구에서 비개발 지식 업무까지 확장하는 공식 발표다. role-specific plugins, Sites, annotations, 62개 앱, 110개 skills, 주간 사용자 500만 명 이상이 핵심 수치로 제시됐다.

실무 시사점: Codex 도입 판단은 “코딩 성능”만으로는 부족하다. 조직은 어떤 앱 권한, plugin, skill, Sites 공유 범위를 허용할지 먼저 정해야 한다.
