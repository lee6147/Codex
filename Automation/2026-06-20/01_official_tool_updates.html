Claude Code 2.1.183: auto mode 안전장치 강화

발표일: 확인 불가 (공식 raw changelog에 항목별 게시일 미노출)

  • auto mode에서 파괴적 git 명령과 Terraform destroy 류 명령을 더 강하게 차단한다.
  • subagent WebSearch 빈 결과, Windows Terminal 전체화면 손상, 세션 attribution 옵션 같은 운영성 이슈를 함께 손봤다.
  • 의미: 대규모 자동 실행을 허용하되, 실수 한 번으로 작업트리나 인프라를 망가뜨리지 않도록 보호층을 두껍게 하는 흐름이다.

Claude Code 2.1.181: 설정·스트리밍·네트워크 드라이브 안정화

발표일: 확인 불가 (공식 raw changelog에 항목별 게시일 미노출)

  • 특히 네트워크 드라이브와 클라우드 동기화 폴더에서 0바이트/잘린 파일이 생기던 문제를 수정했다.
  • 의미: 개인 해커톤용 도구가 아니라 실제 팀 저장소와 원격 파일시스템에서 돌릴 수 있는 CLI로 성숙시키는 방향이다.

2. Cursor

공식 2026-06-18

Cursor 3.8: Automations 전면 강화

  • /automate 스킬로 로컬 세션에서 곧바로 자동화를 만들 수 있다.
  • Slack 이모지 트리거, GitHub 리뷰/워크플로 이벤트 트리거 5종, automation용 computer use가 추가되었다.
  • 의미: IDE 안 보조 기능을 넘어, 이슈 코멘트 대응과 실패한 GitHub Actions 정리까지 자동화하는 운영 레이어로 이동 중이다.
공식 2026-06-17

Cursor 3.7: 클라우드 환경 설정과 /in-cloud 서브에이전트

  • 클라우드 에이전트가 10분 이내 개발환경을 세팅하고, 결과를 재사용 가능한 스냅샷으로 남길 수 있다.
  • /in-cloud로 별도 VM과 브랜치에서 병렬 작업을 돌리고, /babysit으로 PR을 백그라운드에서 관리할 수 있다.
  • 의미: Cursor의 차별점이 모델 그 자체보다 로컬-클라우드 오케스트레이션 경험으로 이동하고 있음을 보여준다.
공식 2026-06-10

Bugbot 성능 개선: 더 빠른 리뷰, 더 낮은 비용

  • Bugbot 평균 리뷰 시간이 약 5분에서 약 90초로 줄었고, 비용은 약 22% 낮아졌으며, 버그 탐지량은 10% 늘었다.
  • /review 경로로 push 전 리뷰를 로컬에서 먼저 돌릴 수 있게 했다.
  • 의미: Cursor는 코드 생성만이 아니라 리뷰 자동화 ROI까지 숫자로 설득하려 한다.

3. OpenAI Codex

공식 2026-06-18

Codex App 26.616: Record & Replay와 자동화 히스토리 일괄 작업

  • 에이전트 작업을 기록하고 재생하는 Record & Replay가 추가되었다.

  • 의미: 재현 가능한 데모와 작업 회고가 중요해지는 팀 도입 단계에 맞춘 변화다.
공식 2026-06-18

Codex CLI 0.141.0: 원격 실행 채널과 MCP 연결성 강화

  • 원격 executor와의 통신에 인증된 암호화 Noise relay 채널을 도입했다.
  • 플랫폼별 native cwd/shell 보존, 선택된 executor plugin의 stdio MCP 활성화, Windows sandbox credential 복구가 포함되었다.
  • 의미: Codex는 단일 로컬 도구보다 원격 실행과 플러그인 연계를 포함한 에이전트 런타임 쪽으로 무게중심을 옮기고 있다.
공식 2026-06-11

Codex App 26.609: Browser use 개발자 모드와 Windows 권한 제어

  • CDP 기반 Browser use developer mode와 /init가 추가되었다.
  • Windows에서 앱별 접근 제어를 제공하고, Enterprise용 Computer Use 제공 범위를 확대했다.
  • 의미: 코딩 에이전트가 브라우저와 데스크톱을 다루는 범위가 넓어질수록 권한 제어 UI가 제품 핵심으로 올라오고 있다.
