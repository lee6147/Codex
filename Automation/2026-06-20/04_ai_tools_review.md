## 1. Claude Code

### 강점

- 최신 공식 changelog가 안전장치, 파일 안정성, 세션 복구, auto-retry 쪽을 집중 보강하고 있다.
- CLI 기반 장기 작업과 세밀한 권한 제어 시나리오에서 여전히 가장 실무적인 인상을 준다.
- 스킬, 하위 에이전트, 셸 실행을 묶어 복잡한 개발 루프를 견고하게 유지하는 방향이 분명하다.

### 약점

- 공식 changelog 항목별 날짜가 노출되지 않아 최신 변경 추적의 투명성이 떨어진다.
- 보안 연구 논문에서 반복적으로 사례 대상이 되고 있어, 강력한 실행기가 가진 공격면도 함께 커지고 있다.
- 모델 성능 경쟁보다 운영 안정성·보안 검증 부담이 사용자 책임으로 남는 구간이 있다.

### 추천 사용 시나리오

- 로컬 저장소와 셸 중심으로 깊게 작업하는 고급 개발자.
- 코드 수정, 테스트, 반복 검증을 하나의 CLI 루프로 길게 돌리고 싶은 팀.
- 권한 제어를 세밀하게 다루면서도 자동 실행 강도를 높이고 싶은 환경.

### 피해야 할 경우

- 실행 권한 모델을 잘 모르는 초보 팀이 바로 전사 도구로 확대할 때.
- 스킬 공급망과 보안 통제를 아직 설계하지 않은 조직.

### 현재 판단

Claude Code는 이번 주에도 가장 “숙련자 친화적”이다. 다만 강점의 반대편에는 보안·운영 책임이 함께 따라붙는다. 즉, 가장 강한 도구이지만 가장 가볍게 도입할 도구는 아니다.

## 2. Cursor

### 강점

- 공식 업데이트가 `automations`, `in-cloud`, `review`처럼 실행 가능한 워크플로 기능을 빠르게 늘리고 있다.
- IDE UX와 클라우드 병렬 작업을 한 제품 안에서 연결하는 능력이 강하다.
- GitHub/Slack 이벤트를 바로 자동화로 연결하는 제품 포지션이 명확하다.

### 약점

- 이번 주 외부 기사 상당수가 기능보다 인수합병, 기업가치, 컴퓨트 확보 서사에 몰려 있다.
- 자체 모델 전략이 강화되는 만큼, 향후 가격·정책·생태계 방향성 변동성도 커질 수 있다.
- “빠른 팀 도입”은 강하지만, 장기적으로 어떤 모델/인프라에 잠기는지 아직 불확실한 면이 있다.

### 추천 사용 시나리오

- IDE 안에서 빠르게 생성하고 리뷰하고 다시 수정하는 루프가 핵심인 팀.
- 클라우드 서브에이전트를 이용해 CI 수정, PR babysit, 이벤트 기반 자동화를 병렬로 돌리고 싶은 조직.
- 비개발자와 개발자 협업까지 고려해 상대적으로 쉬운 진입점을 원할 때.

### 피해야 할 경우

- 특정 벤더 종속과 가격 정책 변화에 매우 민감한 조직.
- 기능보다 장기적인 모델 독립성과 인프라 통제권이 더 중요한 환경.

### 현재 판단

Cursor는 이번 주 가장 공격적으로 외연을 넓힌 도구다. 단순 코딩 보조를 넘어 자동화 운영 플랫폼으로 이동 중이며, 시장도 그 점을 높게 평가하고 있다. 반면 제품 가치가 컴퓨트와 자본 서사에 더 강하게 묶이기 시작했다는 점은 리스크다.

## 3. OpenAI Codex

### 강점

- Record & Replay, 원격 executor 채널, Browser use, Computer Use 등 제품 범위를 빠르게 넓히고 있다.
- 코딩만이 아니라 브라우저·데스크톱 작업까지 포함한 범용 에이전트 워크플로에 가장 가깝다.
- MCP와 원격 실행을 연결하는 구조 덕분에 도구 확장성이 높다.

### 약점

- 외부 기사량이 아직 많지 않고, 이번 주 대표 기사도 용량 이슈 복구 쪽에 초점이 맞춰졌다.
- 기능이 빠르게 늘어나는 만큼 제품 경계와 권한 관리 복잡도도 함께 올라간다.
- “무엇까지 Codex에게 맡길 것인가”에 대한 팀 차원의 운영 규칙이 없으면 산만해질 수 있다.

### 추천 사용 시나리오

- 브라우저, 셸, 파일, 자동화 기록을 하나의 제품 안에서 묶어 다루고 싶은 사용자.
- 원격 실행과 로컬 작업을 섞어 쓰는 팀.
- 코딩 보조보다 작업 수행형 에이전트에 가까운 경험을 원하는 경우.

### 피해야 할 경우

- 브라우저/컴퓨터 사용 기능이 불필요하고, 단순 IDE 코드 생성만 원하는 팀.
- 장애 민감도가 높은 환경에서 가용성 이슈를 매우 보수적으로 보는 조직.

### 현재 판단

Codex는 가장 범용적인 방향으로 가고 있다. 이번 주만 보면 아직 시장 서사가 완전히 굳지는 않았지만, 기능 방향성은 분명하다. 코딩 도구라기보다 업무 실행형 에이전트 플랫폼으로 수렴 중이다.

## 추천 사용자 유형

- `깊은 로컬 제어`가 최우선이면 Claude Code.
- `빠른 팀 생산성과 병렬 자동화`가 최우선이면 Cursor.
- `브라우저/컴퓨터 사용까지 한 제품에서 묶는 범용 에이전트`가 필요하면 Codex.

## 이번 주 총평

이번 주 AI 코딩 도구 시장은 “누가 더 똑똑한가”보다 “누가 더 안전하게 오래 일하게 만들 수 있는가”를 겨루고 있다. Claude Code는 안정성과 통제력, Cursor는 자동화 운영성, Codex는 범용 실행 범위를 앞세우고 있다. 세 도구의 차이는 모델 품질보다 운영 철학에서 더 선명해졌다.

## 주요 트렌드 관찰

1. 자동화와 원격 실행이 이제 부가 기능이 아니라 핵심 경쟁축이다.
2. 메모리, MCP, 리뷰, 재현성 같은 운영 요소가 모델 성능만큼 중요해졌다.
3. 보안 연구와 공급망 리스크가 코딩 에이전트 시장의 본격 변수로 올라왔다.
