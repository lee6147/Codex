1. Claude Code

2026-06-14

CmdNeedle: 명령 denylist 우회 가능성 계량화

CmdNeedle 논문은 Claude Code에 적용된 denylist도 우회 가능한 명령 조합을 놓칠 수 있다고 지적한다. 이는 코딩 에이전트의 안전장치가 “명령 목록 차단” 하나로 충분하지 않다는 경고다.

2026-06-15

Dynamic Malicious Skills: 스킬 문서 자체가 공격면

Dynamic Malicious Skills 논문은 SKILL.md 같은 자연어 지침에 악성 내용을 숨겨 런타임에 악성 로직을 주입할 수 있음을 보였다. Claude Code 같은 스킬 기반 에이전트 생태계의 공급망 리스크가 기사 바깥 학술 영역에서도 빠르게 부상 중이라는 뜻이다.

2026-06-17

PhantomSkill: 보조 리소스에 숨는 악성 스킬

PhantomSkill은 노골적 지시가 아니라 보조 스크립트와 취약점처럼 보이는 코드에 공격을 숨기는 방식을 보여준다. 즉, 향후 보안 경쟁은 프롬프트 검열보다 스킬 패키지와 실행 격리 정책으로 이동할 가능성이 크다.

2. Cursor

2026-06-16

Business Insider: SpaceX의 Cursor 600억 달러 인수 발표

Business Insider 보도에 따르면 SpaceX는 Cursor를 600억 달러에 인수하기로 발표했다. 기사 초점은 코딩 도구 자체의 기능보다, AI 코딩 시장에서 컴퓨트와 자본이 얼마나 전략적 우위가 되었는가에 있다.

2026-06-16

Business Insider: Michael Truell 프로필과 자체 모델 전환

같은 매체의 프로필 기사에서는 Cursor가 Anthropic 의존도를 줄이고 Composer 자체 모델 전략을 강화하는 흐름을 조명했다. 이는 Cursor가 단순 UX 툴이 아니라 모델 주권까지 가져가려 한다는 신호다.

2026-06-19

MarketWatch: Cursor 회복 서사와 SpaceX 컴퓨트 레버리지

MarketWatch는 “Cursor is dead” 담론 이후 SpaceX 인수/컴퓨트 지원이 회복 서사로 전환되고 있다고 보도했다. 기사 흐름상 Cursor의 최신 경쟁력은 IDE 감성보다 GPU 접근성과 자체 모델 스케일링 기대감에 더 크게 묶인다.

  • Claude Code는 기능 홍보보다 보안 연구의 실험 대상이 되면서 “강력하지만 위험한 실행기” 프레임이 커지고 있다.
