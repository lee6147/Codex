Claude Code

v2.1.110: /tui 전체화면, 푸시 알림, /focus 분리

공식 글신규 기능

2026-04-15 릴리즈에서 Claude Code는 `/tui fullscreen`, 푸시 알림 도구, `/focus` 전용 토글, 자동 스크롤 제어 설정을 추가했다. 장시간 세션을 유지하는 사용자를 직접 겨냥한 업데이트다.

발표일: 2026-04-15 | 왜 중요한가: 장기 실행 세션에서 UI 안정성과 재진입 알림이 좋아져 실제 운영성이 올라간다. | 출처: Claude Code Changelog

v2.1.108: 1시간 프롬프트 캐시, /recap, Skill 기반 slash command 발견

공식 글운영성 개선

2026-04-14 릴리즈는 1시간 프롬프트 캐시 TTL, 세션 복귀용 `/recap`, built-in slash command 발견, `/undo` 별칭, 메모리 사용량 감소를 묶어 공개했다. 새 모델 추가보다 세션 복귀와 비용 최적화에 더 초점이 있다.

발표일: 2026-04-14 | 왜 중요한가: 긴 대화와 반복 세션을 쓰는 팀에서 비용·컨텍스트 관리가 더 쉬워진다. | 출처: Claude Code Changelog

v2.1.105: 기존 worktree 진입, PreCompact hook, plugin monitors

공식 글에이전트 워크플로

2026-04-13 릴리즈에서는 기존 worktree로 들어가는 `path` 파라미터, compaction을 차단할 수 있는 PreCompact hook, 플러그인용 background monitor support가 추가됐다. 자동화 하네스와 병렬 worktree 운영에 바로 연결되는 변화다.

발표일: 2026-04-13 | 왜 중요한가: 멀티 worktree, hook, background monitoring을 쓰는 고급 워크플로에 직접 도움이 된다. | 출처: Claude Code Changelog

Cursor

Canvases: 에이전트가 인터랙티브 결과물을 직접 만든다

공식 글신규 기능

2026-04-15 Cursor는 Agents Window와 에디터에서 인터랙티브 캔버스를 생성하는 기능을 공개했다. 표, 박스, 다이어그램, 차트, diff, to-do list를 포함한 시각 결과물을 에이전트 응답으로 남길 수 있고, 그 결과는 사이드 패널에서 지속 아티팩트로 유지된다.

발표일: 2026-04-15 | 왜 중요한가: 텍스트 응답만 받던 에이전트 결과를 검토 가능한 UI 아티팩트로 바꾸기 때문에 설계·요약·리뷰 단계가 훨씬 읽기 쉬워진다. | 출처: Cursor Changelog

Cursor 3.1: 타일 레이아웃과 향상된 음성 입력

공식 글멀티에이전트 UX

2026-04-13 공개된 Cursor 3.1은 Agents Window를 타일 레이아웃으로 분할해 여러 에이전트를 병렬 관리하게 했고, 음성 입력 안정성, 브랜치 선택, diff에서 파일 이동, 검색 필터 등도 함께 개선했다.

발표일: 2026-04-13 | 왜 중요한가: 여러 에이전트 출력을 비교하고 worktree/branch를 오가며 검토하는 실무 흐름의 전환 비용을 줄인다. | 출처: Cursor Changelog
