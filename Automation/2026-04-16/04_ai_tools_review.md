## 1. Claude Code

### 강점

- 최근 공개 changelog만 봐도 /tui, /focus, /recap, prompt caching, plugin monitors처럼 장기 세션 운영에 직결되는 기능이 빠르게 늘고 있다.
- worktree, hooks, background monitoring까지 공식 기능으로 다듬고 있어 자동화 하네스와의 궁합이 좋다.
- 주간 노트와 changelog를 같이 보면 CLI 기반 에이전트 운영체계로서의 방향이 분명하다.

### 약점

- 공개 정보가 changelog, 주간 노트, 개별 문서로 분산돼 있어 추적 난도가 높다.
- GUI 중심 사용자가 바로 이해하기에는 제품 표면이 여전히 텍스트·명령 중심이다.
- 업데이트 속도가 빨라서 팀 규칙과 내부 운영 문서를 자주 따라가야 한다.

### 추천 사용 시나리오

- CLI 중심 개발팀이 장기 작업, worktree, hooks, background monitoring을 같이 굴릴 때.
- AGENTS.md, rules, memory, automation 같은 운영 레이어를 적극적으로 설계하는 팀.
- 사람 승인과 에이전트 자율 실행의 균형을 세밀하게 맞추고 싶은 경우.

### 현재 시점 총평

Claude Code는 강한 코딩 모델이라기보다 운영 가능한 CLI 에이전트 환경으로 빠르게 진화하고 있다. 최신 공개 변화도 대부분 실행 루틴, 세션 복귀, 모니터링, 권한 제어 같은 운영성 강화에 몰려 있다.

## 2. Cursor

### 강점

- 최근 3일 안에서도 Canvases와 3.1 같은 변화가 나와, 멀티에이전트 결과를 시각적으로 검토하고 비교하는 UX가 계속 강해지고 있다.
- 공식 블로그의 multi-agent kernels 연구는 Cursor가 단순 IDE가 아니라 에이전트 오케스트레이션을 핵심 경쟁축으로 보고 있음을 보여준다.
- 타일 레이아웃, branch selection, diff navigation처럼 실제 팀 작업 흐름을 바로 줄여 주는 개선이 많다.

### 약점

- 제품 표면이 빠르게 변해 익숙해질 즈음 다시 학습이 필요한 경우가 많다.
- IDE 중심성이 강해, 셸 우선 자동화나 로컬 하네스 문화에는 Claude Code보다 덜 직접적일 수 있다.
- 강점이 UI와 오케스트레이션에 몰려 있어, 규칙 기반 통제는 별도 체계가 필요하다.

### 추천 사용 시나리오

- 여러 에이전트 결과를 동시에 비교하고 검토해야 하는 제품·플랫폼 팀.
- 설계안, diff, 대시보드 같은 시각 결과물을 빠르게 공유해야 하는 협업 환경.
- 병렬 에이전트 워크플로를 IDE 안에서 자연스럽게 운영하고 싶은 경우.

### 현재 시점 총평

Cursor는 현재 가장 제품화된 멀티에이전트 워크스페이스에 가깝다. 최근 공개 신호도 모델 자체보다 시각화된 결과물, 병렬 관리, 연구 하네스 쪽에 집중돼 있다.

## 3. OpenAI Codex

### 강점

- 최근 공개 문서들은 Codex를 개인 개발자 도구보다 조직형 업무 에이전트로 확장하는 방향을 분명히 보여준다.
- ChatGPT Release Notes, Academy 배포 가이드, 고객 사례를 함께 보면 과금, 권한, 롤아웃, 실사용 사례가 비교적 선명하게 연결된다.
- 설계 검토, 코드 리뷰, 문서화, 비개발 부서 인접 업무까지 확장 가능한 포지셔닝이 강하다.

### 약점

- 최근 3일 창에서는 눈에 띄는 신규 공식 업데이트가 없어 제품 변화 추적의 즉시성이 약했다.
- 최근 공개 신호가 기능 자체보다 사용량 정책, 배포 가이드, 사례 소개에 더 몰려 있다.
- 공식 정보가 Help Center, Academy, OpenAI 블로그로 나뉘어 있어 탐색 동선이 길다.

### 추천 사용 시나리오

- 조직 차원의 도입, 권한 관리, 배포 정책 정리가 중요한 환경.
- 코드 생성뿐 아니라 설계 검토, 문서 정리, 워크플로 표준화까지 함께 보려는 팀.
- ChatGPT 중심 계정·업무 체계를 이미 쓰고 있는 조직.

### 현재 시점 총평

Codex는 이번 주 기준으로 새 기능이 매일 쏟아지는 제품보다는 조직 안에 안정적으로 배치하는 제품 쪽에 더 무게가 실려 있었다. 최신 공개 신호도 기능 홍보보다 배포·정책·고객 활용 사례에 가깝다.

## 이번 주 총평

이번 주 시장은 한 번 더 분명해졌다. 승부처는 더 똑똑한 한 번의 답변이 아니라, 에이전트를 오래 돌리고, 결과를 읽기 좋게 만들고, 기억을 붙이고, 비용과 권한을 통제하는 운영 계층이다. GitHub Trending에서도 가장 크게 뜬 저장소들이 바로 그 레이어를 다루고 있다.

즉 지금 팀이 먼저 해야 할 일은 어느 모델이 제일 세냐보다 우리 팀의 규칙 파일, 메모리, worktree, 요약 포맷, 문서 정규화 계층을 어떻게 만들 것인가를 정하는 것이다. 모델보다 운영체제가 더 중요해진 주간이다.
