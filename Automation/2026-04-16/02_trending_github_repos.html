요약

이번 주 트렌딩은 더 좋은 모델보다 더 오래 일하는 에이전트 운영 계층이 강하게 뜬 주간이다. 스킬 파일, 장기 메모리, managed agents, harness builder, 문서 정규화 도구가 함께 올라왔다.

선정 저장소

forrestchang/andrej-karpathy-skills

SkillsAgent Rules

Andrej Karpathy의 LLM 코딩 실패 패턴 관찰을 바탕으로 Claude Code 행동을 교정하는 단일 CLAUDE.md 룰셋이다.

  • Star 수: 43,460
  • 지금 주목할 이유: 모델 교체보다 규칙 파일이 체감 품질을 좌우한다는 흐름을 상징한다.
  • 연결점: Claude Code, Codex, Cursor 모두 비슷한 규칙층으로 옮겨 쓰기 좋다.

thedotmack/claude-mem

MemoryClaude Code

Claude Code 세션 동안 나온 맥락을 자동 캡처·압축하고, 이후 세션에 다시 주입하는 메모리 플러그인이다.

  • Star 수: 57,924
  • 지금 주목할 이유: 장기 작업에서 가장 큰 병목인 컨텍스트 단절을 메우는 레이어다.
  • 연결점: Codex와 Cursor 사용자도 외부 메모리 계층 설계 아이디어를 그대로 가져갈 수 있다.

NousResearch/hermes-agent

Agent RuntimePlatform

스킬, 스케줄링, 다중 채널 인터페이스, 서브에이전트 운영을 묶은 범용 자율 에이전트 플랫폼이다.

  • Star 수: 89,875
  • 지금 주목할 이유: 오픈소스 진영이 단일 에이전트에서 운영 가능한 에이전트 런타임으로 이동하고 있음을 보여준다.
  • 연결점: 장기 실행, 스케줄, 메모리 계층을 별도로 설계할 때 비교 대상이 된다.

multica-ai/multica

Managed AgentsWorkflow

코딩 에이전트를 실제 팀원처럼 배정하고 추적하는 오픈소스 managed agents 플랫폼이다.

  • Star 수: 13,667
  • 지금 주목할 이유: 작업 할당, 진행 추적, 스킬 누적을 제품 표면으로 끌어올린다.
  • 연결점: 여러 에이전트에게 독립 작업을 나눠 맡기는 팀 운영 흐름과 직접 맞닿아 있다.

coleam00/Archon

HarnessDeterministic

AI 코딩을 더 결정적이고 반복 가능하게 만들기 위한 오픈소스 harness builder다.

  • Star 수: 18,239
  • 지금 주목할 이유: 좋은 모델보다 좋은 하네스가 재현성을 만든다는 흐름을 보여준다.
  • 연결점: 자동화 하네스, 검증 루프, 규칙 파일 중심 개발을 하는 팀에 특히 잘 맞는다.

microsoft/markitdown

DocsProductivity

PDF, Office, HTML, 오디오 등 다양한 입력을 LLM 친화적인 Markdown으로 정규화하는 도구다.

  • Star 수: 109,562
  • 지금 주목할 이유: 에이전트 성능이 아니라 입력 정리 계층이 생산성에 얼마나 중요한지 보여준다.
  • 연결점: Claude Code, Cursor, Codex 모두 긴 자료를 그대로 먹이기보다 Markdown 정규화와 같이 쓸 때 효율이 오른다.
