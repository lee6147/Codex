이 문서는 [[01_official_tool_updates]], [[02_trending_github_repos]], [[03_ai_coding_tools_articles]]를 종합한 Obsidian용 평가 노트다. 2026-05-15 기준 최신 공식 업데이트, GitHub 트렌딩 스냅샷, 최근 기사/릴리스 해설을 함께 반영했다.

## 개요

- 공식 업데이트의 중심은 새 모델 발표가 아니라 에이전트 실행 환경, 세션 안정성, 원격 제어, 권한/감사 로그 같은 운영 표면이다.
- GitHub 트렌딩에서는 스킬, 메모리, 터미널 에이전트, 개발 방법론처럼 Claude Code/Codex/Cursor를 보조하는 주변 도구가 강했다.
- 기사 수집은 최근 2일만으로 도구별 2~3개를 채우기 어려워 7일(2026-05-09 ~ 2026-05-15)로 확장했다.

## 도구별 평가

### Claude Code

**강점**
- CLI 기반 장시간 작업에서 훅, 세션, 백그라운드 에이전트, 플러그인 운영 기능이 빠르게 보강되고 있다.
- GitHub 릴리스가 세부 수정 사항을 공개해 Windows, MCP, 권한 모드, 상태라인 이슈를 추적하기 좋다.

**약점**
- 기능 표면이 넓어질수록 hooks, plugins, background agents, managed settings 조합에서 회귀 가능성을 계속 봐야 한다.
- 팀 사용자는 로컬 지침과 스킬 품질이 낮으면 모델 성능을 제대로 살리기 어렵다.

**추천 사용 시나리오**
- 터미널 중심 개발, 큰 코드베이스 탐색, 멀티 파일 계획, 명확한 로컬 작업 계약이 있는 리팩터링.

**피해야 할 사용 시나리오**
- 테스트와 리뷰 없이 큰 기능을 한 번에 맡기는 방식.

**최근 업데이트 반영 관점**
- 2026-05-13 v2.1.141은 운영 안정성 릴리스로 해석하는 것이 적절하다.

**현재 시점 총평**
- Claude Code는 여전히 “로컬 작업 지침과 검증 루프를 잘 갖춘 숙련자”에게 가장 강하다.

### Cursor

**강점**
- 2026-05-13 업데이트로 클라우드 에이전트 환경을 멀티 레포, Dockerfile, build secret, 감사 로그, 환경별 권한 범위까지 끌어올렸다.
- IDE, Teams, Bugbot, cloud agents를 묶어 팀 워크플로 안으로 들어가는 속도가 빠르다.

**약점**
- 기능이 IDE/클라우드/팀 채널로 넓어지면서 설정과 거버넌스 복잡도가 커질 수 있다.
- 소규모 개인 작업에서는 Claude Code나 Codex CLI보다 UI/운영 레이어가 무겁게 느껴질 수 있다.

**추천 사용 시나리오**
- 여러 저장소와 내부 빌드 환경을 다루는 팀, IDE 안에서 모델/에이전트/PR 리뷰를 함께 운영하려는 조직.

**피해야 할 사용 시나리오**
- 단순 터미널 자동화나 최소 UI를 원하는 개인 워크플로.

**최근 업데이트 반영 관점**
- Cursor의 경쟁 포인트는 “AI 편집기”에서 “통제 가능한 클라우드 에이전트 운영 환경”으로 이동 중이다.

**현재 시점 총평**
- Cursor는 팀형 에이전트 운영에서는 가장 공격적으로 제품 표면을 넓히고 있다.

### OpenAI Codex

**강점**
- 0.130.0 이후 remote-control, app-server, multi-environment, Windows sandbox, apply_patch diff 정확도 같은 자동화 친화 기능이 보인다.
- 샌드박스/원격 실행형 작업에 강해 “정의가 분명한 큰 작업”을 분리해 맡기기 좋다.

**약점**
- 0.131 alpha 계열은 릴리스 간격이 촘촘해 안정판 도입보다 변경 추적 대상으로 보는 편이 안전하다.
- 공개 릴리스 노트가 짧은 pre-release는 실제 영향 범위를 GitHub PR이나 2차 변경 분석으로 보완해야 한다.

**추천 사용 시나리오**
- 테스트가 있는 리팩터링, 원격 제어형 자동화, 다중 환경에서의 CLI 작업, 반복 보고서/진단 하네스.

**피해야 할 사용 시나리오**
- 완료 조건이 불명확한 탐색형 작업을 긴 시간 방치하는 방식.

**최근 업데이트 반영 관점**
- Codex는 모델보다 앱 서버/원격 제어/다중 환경 실행 계층이 핵심 관찰 포인트다.

**현재 시점 총평**
- Codex는 로컬 CLI와 원격 작업을 이어 붙이는 자동화형 개발자에게 매력적이다.

## 추천 사용자 유형

- **터미널 중심 개인 개발자:** Claude Code 또는 Codex. 로컬 지침, 테스트, 작은 작업 단위가 있으면 효과가 크다.
- **IDE 안에서 계속 작업하는 팀:** Cursor. 특히 멀티 레포와 클라우드 에이전트 환경이 필요한 팀에 맞다.
- **반복 자동화와 리포트 하네스 운영자:** Codex. remote-control, app-server, 다중 환경 변경 흐름을 계속 추적할 가치가 있다.
- **초보자:** Cursor가 접근성이 좋지만, AI 결과물 검증법을 먼저 익혀야 한다.
