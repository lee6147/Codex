트렌드 해석

오늘의 트렌딩은 모델 래퍼보다 “에이전트가 덜 망가지게 일하게 만드는 보조 구조” 쪽으로 기울어 있다. 스킬 디렉터리, 메모리, 터미널 에이전트, 개발 방법론, 브라우저 자동화처럼 Codex, Claude Code, Cursor를 더 오래 안정적으로 굴리기 위한 주변 도구가 눈에 띈다.

선정 저장소

1

mattpocock/skills

81.5k stars+174 todayShell

“Real Engineers”를 위한 스킬 모음으로, Claude 계열 에이전트가 반복적으로 같은 품질 규칙을 따르도록 만드는 프롬프트/스킬 자산이다. Codex나 Cursor 사용자에게도 프로젝트별 작업 계약을 구조화하는 참고점이 된다.

한줄 총평: 에이전트 품질은 모델보다 반복 가능한 작업 규칙에서 먼저 갈린다는 신호.

2

rohitg00/agentmemory

7.3k stars+96 todayTypeScript

AI 코딩 에이전트를 위한 지속 메모리 계층이다. 장기 작업에서 이전 결정, 코드베이스 규칙, 사용자 선호가 사라지는 문제를 줄이는 방향이라 Claude Code/Codex의 장시간 세션 운영과 직접 연결된다.

한줄 총평: “컨텍스트를 다시 설명하는 비용”을 줄이는 도구는 계속 주목할 만하다.

3

Hmbown/DeepSeek-TUI

28.8k stars+99 todayRust

DeepSeek 모델용 터미널 코딩 에이전트다. 특정 모델 계열을 CLI 워크플로에 밀착시키는 접근으로, Claude Code와 Codex CLI가 만든 “터미널 에이전트” 시장의 확장을 보여준다.

한줄 총평: AI 코딩 도구 경쟁이 IDE뿐 아니라 TUI/CLI 레이어로 넓어지고 있다.

4

obra/superpowers

189.7k stars+100 todayShell

에이전트 스킬 프레임워크와 소프트웨어 개발 방법론을 결합한 프로젝트다. 단순 프롬프트보다 작업 분해, 품질 기준, 협업 절차를 표준화하려는 흐름과 맞닿아 있다.

한줄 총평: 팀이 에이전트를 쓰려면 “사용법”보다 “운영 방식”이 필요하다는 신호.

5

multica-ai/andrej-karpathy-skills

129.3k stars+94 todayClaude Code

LLM 코딩의 함정과 코딩 표준을 CLAUDE.md 형태로 묶은 단일 파일 자산이다. Claude Code용으로 보이지만, AGENTS.md나 Codex 로컬 지침 설계에도 바로 참고할 수 있다.

한줄 총평: 좋은 에이전트 성능은 좋은 로컬 지침 파일에서 시작한다.

6

NousResearch/hermes-agent

149.4k stars+89 todayPython

일반 AI agent 프레임워크지만 ChatGPT/LLM/agent 태그가 붙어 있고, 도구 호출과 자기 확장형 에이전트 실험에 가깝다. 코딩 전용은 아니므로 생산 도입보다 구조 관찰 대상으로 보는 편이 적절하다.

한줄 총평: 넓은 agent 프레임워크는 아이디어는 풍부하지만 코드 작업 특화성은 별도 검증이 필요하다.

7

CloakHQ/CloakBrowser

9.9k stars+70 todayPython

Playwright 대체를 표방하는 stealth Chromium 프로젝트다. AI 코딩 도구 자체는 아니지만, 브라우저 자동화와 테스트/스크래핑 기반 에이전트 작업의 인프라로 연결된다. 사용 시 정책/윤리/사이트 약관 리스크를 반드시 점검해야 한다.

한줄 총평: 브라우저 자동화 인프라는 강력하지만 오용 리스크가 크다.

8

tinyhumansai/openhuman

5.9k stars+168 todayRust

개인 AI 슈퍼인텔리전스를 표방하는 Rust 프로젝트다. 코딩 생산성과의 직접성은 약하지만, 로컬/개인 에이전트 런타임에 대한 관심이 커지고 있음을 보여준다.

한줄 총평: 직접 도입보다 개인 에이전트 런타임 흐름을 관찰할 후보.
