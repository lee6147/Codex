## 도구별 평가

## 1. Claude Code

### 강점

- 로컬 터미널 및 IDE 중심의 조향형 작업에 강하다.
- 스킬, CLAUDE.md, 훅, 세션 운영 지식이 GitHub 트렌딩에서 계속 파생되고 있다.
- 대규모 코드베이스, 복잡한 리팩터링, 사람이 중간에 방향을 조정해야 하는 작업에서 여전히 강한 선택지로 평가된다.

### 약점

- 이번 엄격 3일 공식 업데이트 창에서는 새 기능 발표가 확인되지 않았다.
- 기사 흐름은 기능 개선보다 사용량 제한, 크레딧, Codex와의 경쟁 구도에 치우쳤다.
- 2차 보도 기반의 한도 증가 정보는 공식 원문 확인 전까지 보수적으로 다뤄야 한다.

### 추천 사용 시나리오

- 사람이 계속 코드 맥락을 보며 에이전트를 조향해야 하는 디버깅과 리팩터링.
- 로컬 개발 환경, MCP, 터미널 도구, 사내 스크립트가 작업 품질에 중요한 프로젝트.
- 스킬과 작업 계약서를 정교하게 만들어 반복 업무를 표준화하려는 팀.

### 피해야 할 사용 시나리오

- 비용과 사용량 예측이 가장 중요한 대량 자동화.
- 공식 업데이트 확인 없이 2차 보도나 커뮤니티 글만 보고 운영 정책을 바꾸는 결정.
- 사람이 중간 승인 없이 장시간 병렬 작업을 대량 위임해야 하는 경우.

### 최근 업데이트 반영 관점

이번 공식 업데이트 레인에서는 Claude Code 자체의 새 릴리즈보다 주변 생태계가 더 활발했다. `academic-research-skills`, `skills`, `andrej-karpathy-skills` 같은 저장소가 오른 것은 Claude Code 사용자가 도구 성능보다 작업 규칙과 스킬 설계를 더 중요하게 보기 시작했다는 신호다.

### 현재 시점 총평

Claude Code는 여전히 깊은 작업 조향과 로컬 맥락 활용에 강하지만, 이번 주의 뉴스 주도권은 Cursor와 Codex에 있다. Claude Code 사용자는 새 기능보다 사용량 정책과 스킬 운영 품질을 점검하는 주간으로 보는 편이 맞다.

## 2. Cursor

### 강점

- 2026-05-18 Composer 2.5 공개로 모델 품질과 비용 대비 성능을 전면에 세웠다.
- 2026-05-13 클라우드 에이전트 개발 환경 업데이트와 연결하면, Cursor는 IDE를 넘어 에이전트 실행 환경을 통제하려는 방향이 분명하다.
- Composer 2.5 출시 첫 주 사용량 확대는 실제 코드베이스 평가에 유리하다.

### 약점

- Composer 2.5는 Cursor 생태계 안에서만 의미가 크다.
- 공개 벤치마크 수치는 Cursor 자체 평가와 2차 해석이 섞여 있어 독립 재현 전까지 과신하면 안 된다.
- terminal-heavy, DevOps-first 작업에서는 Codex 계열이 더 적합하다는 비교 리뷰가 있다.

### 추천 사용 시나리오

- Cursor를 주 개발 IDE로 쓰고 있으며 다중 파일 수정, 계획 기반 구현, PR 리뷰를 자주 수행하는 팀.
- 에이전트가 실제 의존성, 시크릿, 멀티레포 환경에서 테스트까지 수행해야 하는 조직.
- 모델 API 비용보다 IDE 내 포함 사용량과 예측 가능한 워크플로가 중요한 사용자.

### 피해야 할 사용 시나리오

- 통합 모델 라우터나 외부 API로 모든 에이전트 추론을 관리해야 하는 조직.
- Cursor 외부 CLI나 서버 자동화에서 동일 모델을 직접 호출해야 하는 경우.
- 독립 검증 전 벤치마크 수치만 보고 기존 검증 절차를 줄이는 결정.

### 최근 업데이트 반영 관점

Cursor의 2026-05-18 Composer 2.5와 2026-05-13 클라우드 환경 업데이트는 같은 방향을 가리킨다. 좋은 모델을 IDE 안에 넣는 수준을 넘어, 에이전트가 실행되는 환경, 비용, 리뷰, 병렬 작업까지 묶어 관리하려는 전략이다.

### 현재 시점 총평

Cursor는 이번 주 가장 강한 공식 업데이트를 낸 도구다. 이미 Cursor를 쓰는 팀이라면 Composer 2.5를 실제 업무 브랜치에서 테스트할 이유가 충분하다.

## 3. OpenAI Codex

### 강점

- 2026-05-18 Dell 파트너십으로 하이브리드 및 온프레미스 기업 환경 배포 이야기가 강화됐다.
- 2026-05-14 ChatGPT 모바일 앱 원격 접근은 장시간 작업의 승인과 조향 표면을 넓힌다.
- Codex 관련 기사들은 모바일, 원격 승인, 기업 데이터 근접 배포를 반복해서 다룬다.

### 약점

- 모바일 원격 승인은 작은 화면에서 위험한 승인 결정을 만들 수 있어 팀 규칙이 필요하다.
- Dell 파트너십은 방향성은 명확하지만 실제 고객 배포, 가격, 운영 제약은 아직 별도 확인이 필요하다.
- Claude Code나 Cursor처럼 로컬 IDE 내부의 세밀한 조향을 선호하는 사용자에게는 작업 방식이 맞지 않을 수 있다.

### 추천 사용 시나리오

- 장시간 작업을 백그라운드로 돌리고 모바일 또는 원격 표면에서 승인과 방향 전환을 관리해야 하는 사용자.
- 엔터프라이즈 데이터 위치, 내부 시스템 접근, 감사 가능성이 중요한 조직의 파일럿.
- 터미널 중심 작업, 자동 테스트, 명확한 스펙 기반 구현을 병렬로 위임하는 워크플로.

### 피해야 할 사용 시나리오

- 승인 기준이 모호한 상태에서 모바일로 위험한 파일 변경이나 배포 명령을 승인하는 작업.
- 내부 데이터 위치와 보안 요구사항을 확인하지 않은 클라우드 에이전트 도입.
- 로컬 미커밋 변경이 많은 작업을 원격 에이전트에 그대로 맡기는 방식.

### 최근 업데이트 반영 관점

Codex는 이번 주 개인 생산성 도구에서 기업 운영 표면으로 확장되는 모습이 강하다. 모바일 원격 접근은 사용자 승인 표면을 넓히고, Dell 파트너십은 배포 위치와 거버넌스의 답을 찾는 움직임이다.

### 현재 시점 총평

OpenAI Codex는 "어디서든 에이전트 작업을 조향하고, 기업 데이터 가까이에서 실행한다"는 메시지를 강화하고 있다. 개인 개발자에게는 모바일 원격 관리, 기업에는 온프레미스 및 하이브리드 검토가 핵심이다.
