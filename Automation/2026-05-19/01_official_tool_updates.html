2026-05-15 CLI changelog: background agents, permission, terminal fixes

최근 3일 기준 신규 공지는 없지만, 직전 공식 changelog에서는 background session 실행 인자, Fast mode 기본 모델, plugin skill 노출, remote control 인증 회복, Windows/terminal 관련 오류 수정이 묶여 있었다.

  • 사용자 영향: 긴 작업 중 subagent/background session, 권한 요청, 원격 제어, Windows 터미널 안정성에 직접 연결된다.
  • 실무 메모: 팀/자동화 하네스는 background agent 실행 플래그와 권한 모드가 바뀐 지점을 점검해야 한다.
  • 출처: Claude Code CLI changelog
확인 완료

2026-05-12 Desktop changelog: Quick Entry와 scheduled task 수정

Desktop 앱은 자동 업데이트, corrupt bundled binary 재다운로드, credential helper 중복 탭, CJK 입력 누락, scheduled task 삭제 옵션을 수정했다.

  • 사용자 영향: 한국어/일본어/중국어 입력과 scheduled task 파일 삭제 동작이 더 예측 가능해졌다.
  • 실무 메모: Desktop 기반 자동화 사용자는 bundled CLI binary 캐시 문제를 먼저 의심할 수 있다.
  • 출처: Claude Code Desktop changelog

Cursor 공식 변경

확인 완료

2026-05-13 Cursor 3.4: cloud agent 개발 환경

Cursor는 cloud agents와 automations가 multi-repo 환경을 사용할 수 있게 했고, Dockerfile 기반 환경 정의, build secret, layer cache 개선, 환경 version history, rollback, egress/secret scoping을 발표했다.

  • 사용자 영향: 단일 repo toy task보다 실제 조직의 다중 repo 업무에 cloud agent를 붙이기 쉬워졌다.
  • 실무 메모: agent가 실패했을 때 어떤 환경 버전에서 돌았는지 확인할 수 있어 감사와 재현성이 좋아진다.
  • 출처: Cursor changelog
확인 완료

2026-05-11 Teams 연동과 Bugbot effort levels

Cursor는 Microsoft Teams에서 @Cursor를 호출해 cloud agent에 작업을 위임하는 흐름과 Bugbot의 review effort level 설정을 공개했다.

  • 사용자 영향: 개발자 개인 IDE를 넘어 팀 채널과 PR review 운영 흐름으로 확장되는 신호다.
  • 실무 메모: usage-based billing 조건과 고 effort 설정의 비용 증가를 같이 봐야 한다.
  • 출처: Cursor changelog

OpenAI Codex 공식 변경

확인 완료

2026-05-14 Codex mobile remote access preview

OpenAI는 Codex를 ChatGPT 모바일 앱에서 preview로 사용할 수 있게 했다. 모바일에서 active thread, approval, plugin, project context를 이어 보고, Mac host 또는 원격 환경에서 계속 실행 중인 Codex 작업에 개입할 수 있다.

  • 사용자 영향: 장시간 agent 작업의 병목이던 질문 응답, 방향 전환, 승인 확인을 이동 중에도 처리할 수 있다.
  • 제약: iOS/Android preview이며 현재 phone-to-Windows Codex app 연결은 추후 지원으로 공지됐다.
  • 출처: OpenAI - Work with Codex from anywhere
확인 완료

2026-05-14 Enterprise: access tokens, hooks, HIPAA local use

Enterprise/Edu release note는 Codex access token, remote access governance, workspace setting, Remote connections, hooks, local environment HIPAA support를 함께 명시했다.

  • 사용자 영향: CI, release workflow, 내부 automation이 browser sign-in 없이 workspace identity를 사용할 여지가 생겼다.
  • 실무 메모: remote control은 기본 off이며 admin/owner enablement가 필요하므로 보안 운영 체크포인트로 봐야 한다.
