트렌드 해석

1. Skill과 memory가 agent 운영 단위로 부상

academic-research-skills, agent-skills, skills, agentmemory는 모두 agent의 행동을 단발 prompt보다 오래 유지하는 방향이다. 팀에서는 prompt보다 versioned skill package와 memory policy가 더 중요해진다.

2. 검증 도구가 별도 시장이 됨

react-doctor처럼 agent가 만든 코드를 다시 검사하는 도구가 올라오는 것은 생성 자체보다 검증이 병목이라는 신호다. AI 코딩 도구 도입 KPI는 생성량보다 defect escape와 review time으로 잡아야 한다.

3. Agent가 실행 환경을 요구

codegraph, CLI-Anything, UI-TARS-desktop은 agent가 단순 text assistant가 아니라 repo, CLI, browser, desktop을 다루는 실행자라는 방향을 보여준다.
