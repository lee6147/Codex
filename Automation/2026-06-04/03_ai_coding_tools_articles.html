OpenAI Codex

Codex for every role, tool, and workflow

공식 글OpenAI2026-06-02

OpenAI는 Codex를 개발자 중심 도구에서 역할별 지식 업무 도구로 확장한다고 발표했다. 핵심은 역할별 플러그인, 공유 가능한 Sites preview, 생성물의 특정 부분을 지정해 수정하는 annotations다.

실무적으로는 Codex가 코드베이스 수정뿐 아니라 보고서, 대시보드, 피치 자료, 디자인 프로토타입까지 다루는 에이전트 작업면으로 확장되고 있음을 뜻한다.

실무 시사점: 개발팀은 사내 플러그인·스킬·문서 권한 모델을 미리 정리해야 한다.

Codex is becoming a productivity tool for everyone

공식 글OpenAI2026-06-02

OpenAI는 Codex가 5백만 명 이상의 weekly active users를 기록했고, 지식 노동자가 전체 사용자 중 약 20%를 차지한다고 밝혔다. 빠르게 늘어나는 사용 사례는 데이터 분석, 리서치, 지식 산출물 제작이다.

개발 생산성 관점에서는 코딩 도구가 점점 문서, 스프레드시트, 프레젠테이션 생성과 결합된다는 점이 중요하다.

실무 시사점: 엔지니어링 팀의 자동화 기준이 비개발 팀 산출물에도 확산될 가능성이 크다.

Claude Code

Claude Code 2.1.160 turns some files into approval points

실사용 후기The Forge2026-06-02

이 글은 Claude Code 2.1.160의 핵심을 위험 파일 쓰기 승인 지점으로 요약한다. shell startup 파일과 빌드 도구 설정 파일은 작은 변경만으로도 임의 코드 실행 경로가 될 수 있어 자동 편집에서 별도 승인이 필요하다는 설명이다.

Claude Code의 방향은 단순히 더 많이 자동 실행하는 것이 아니라, 위험도가 높은 쓰기 경로를 구분해 에이전트의 장시간 실행을 안전하게 만드는 쪽이다.

실무 시사점: 저장소 정책에서 .npmrc, .pre-commit-config.yaml, devcontainer 같은 파일을 고위험 변경으로 분류할 필요가 있다.

Coding Beyond Your Training: Claude Code and the Technological Frontier

비평 리뷰arXiv2026-05-28

Claude Code 도입 이후 GitHub 개발자 패널을 분석한 연구로, Claude-co-authored commit을 처리 시점으로 보고 월간 commit, 기여 repo, 사용 언어 수 등의 변화를 측정했다.

실무 시사점: 도구 도입 효과를 체감이 아니라 repo-level 지표로 추적하는 평가 설계가 필요하다.

Cursor

Auto-review Run Mode

공식 글Cursor Forum2026-05-29

Cursor는 Auto-review Run Mode를 공식 포럼에서 설명했다. Shell, MCP, Fetch 호출은 allowlist, sandbox, classifier 순서로 판단되며, classifier는 현재 요청과 사용자 지침을 함께 보고 허용·대안·승인 요청을 결정한다.

이는 Cursor가 장시간 에이전트 실행의 마찰을 줄이면서도 완전 무승인 실행으로 바로 가지 않는 중간 단계를 만들었다는 의미다.

실무 시사점: 팀은 allowlist와 custom instructions를 별도 운영 자산으로 관리해야 한다.

원문 링크

Cursor Auto-Review Mode Ends the Agentic Interrupt Loop

비평 리뷰Vibe Coder Blog2026-06-01

이 글은 Auto-review를 "계속 승인하기"와 "무제한 실행" 사이의 운영 모드로 해석한다. 도구 호출이 반복되는 agentic session에서 사람이 매번 멈춰 승인하지 않아도 되도록 classifier subagent가 중간 판단을 맡는다는 점을 강조한다.

실무 시사점: Cursor를 장시간 자동화에 쓰는 팀은 sandbox 가능한 호출과 classifier로 넘길 호출의 경계를 테스트해야 한다.
