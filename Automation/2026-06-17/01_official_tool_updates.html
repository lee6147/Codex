요약

2026-06-17 기준 공식 업데이트 흐름은 세 갈래다. Claude Code는 2.1.178까지 올라오며 권한 규칙, nested skills, subagent 자동 분류, `/doctor`와 remote control 안정성 보강이 이어졌고, Cursor는 2026-06-10 Bugbot 개선으로 리뷰 자동화의 ROI를 수치로 밀어붙였다. OpenAI Codex는 2026-06-16과 2026-06-11 공식 changelog에서 브라우저·컴퓨터 사용·메모리·자동화 승인 모드 같은 실행 환경 개선을 가장 강하게 드러냈다.

Claude Code

최신 공식 항목은 2.1.178, 안정판 latest도 2.1.178

공식 changelog npm latest 확인 항목별 게시일 확인 불가

공식 changelog 최상단 최신 항목은 2.1.178이다. 이번 버전대에서는 permission rule에 `Tool(param:value)` 문법을 추가했고, nested `.claude/skills` 디렉터리 로드 우선순위를 정리했으며, auto mode의 subagent spawn 분류와 `/doctor`, remote control 오류 처리, MCP 관련 안정성을 다듬었다.

npm registry 기준 `latest`와 `version`은 2.1.178, `next`는 2.1.179다. 즉 2026-06-17 현재 안정판과 다음 채널이 분리되어 있으며, 운영 환경에서는 `next` 기능 유입 여부보다 stable과 권한 정책 변화를 먼저 보는 편이 안전하다.

권한 규칙 subagent remote control

이번 흐름은 모델 자체보다 운영 제어면이 중심이다. nested skills 우선순위, 도구별 permission 표현식, background와 remote control 안정화는 팀 단위 배치에서 바로 체감되는 변화다. Claude Code는 빠르게 움직이는 대신 변경량이 많아서, 관리자 입장에서는 version pinning과 권한 규칙 검증이 필수다.

Cursor

2026-06-10 Bugbot 업데이트가 아직 최신 공식 기준점

공식 changelog 2026-06-10 리뷰 자동화

Cursor changelog의 최신 주요 항목은 2026-06-10 Bugbot 개선이다. Cursor는 리뷰 시간이 약 5분에서 90초 수준으로 줄었고, 리뷰당 버그 탐지량은 0.56에서 0.62로 늘었으며, 비용은 22% 낮아졌다고 발표했다.

`/review`, `/review-bugbot`, `/review-security`와 GitHub·GitLab 동기화는 Cursor가 IDE 보조를 넘어 PR 검증 루프를 제품의 중심면으로 끌어올렸다는 뜻이다. CLI 지원은 예고만 있고 아직 핵심 강점은 IDE 안의 리뷰 경험에 남아 있다.

OpenAI Codex
