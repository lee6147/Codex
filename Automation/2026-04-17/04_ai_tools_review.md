## 1. Claude Code

### 강점

- 2026-04-16 기준 Opus 4.7 xhigh, Auto mode 확대, `/less-permission-prompts`, `/ultrareview` 까지 들어오며 CLI 에이전트 운영의 밀도가 더 높아졌다.
- `/tui fullscreen`, 푸시 알림, `/focus`, `/recap`, 1시간 프롬프트 캐시처럼 장기 세션과 복귀 흐름을 직접 개선하는 기능이 계속 쌓이고 있다.
- worktree, hooks, monitoring, cloud review 같은 요소가 한 제품 안에서 자연스럽게 이어져 자동화 하네스와 궁합이 좋다.

### 약점

- 기능과 운영 문서가 changelog, weekly note, reference docs로 분산돼 있어 추적 난도가 높다.
- 여전히 텍스트·명령 중심이라 GUI 친화적 협업 환경에서는 초기 적응이 필요하다.
- 업데이트 속도가 빠르기 때문에 팀 규칙 파일과 내부 운영 문서를 자주 따라가야 한다.

### 추천 사용 시나리오

- 터미널 중심 개발팀이 장기 작업, 리뷰 자동화, worktree, hooks, background monitoring을 함께 굴릴 때.
- 규칙 파일, 메모리, permissions, 검증 루프를 직접 설계하는 엔지니어링 조직.
- 사람 승인과 에이전트 자율 실행의 경계를 세밀하게 제어하고 싶은 경우.

### 현재 시점 총평

Claude Code는 여전히 가장 강한 CLI 운영형 코딩 에이전트에 가깝다. 최신 변화도 모델 교체보다 권한 마찰 축소, 세션 복귀, 클라우드 리뷰, 장기 실행 같은 운영성 강화에 집중돼 있다.

## 2. Cursor

### 강점

- Canvases로 에이전트 응답을 시각 결과물로 남기고, Cursor 3.1로 여러 에이전트를 타일 형태로 비교할 수 있어 리뷰와 협업 해상도가 높다.
- 공식 연구 글이 보여주듯 Cursor는 제품 UI만이 아니라 장기 멀티에이전트 오케스트레이션 자체를 핵심 경쟁축으로 밀고 있다.
- branch selection, diff navigation, voice input, file search filters처럼 실무 편집 흐름을 줄여 주는 개선이 많다.

### 약점

- 제품 표면이 빠르게 변해 익숙해질 즈음 다시 학습 비용이 든다.
- IDE 중심성이 강해 셸 우선 자동화 문화에는 Claude Code보다 덜 직접적일 수 있다.
- 강점이 UI와 오케스트레이션 쪽에 몰려 있어, 세밀한 규칙 제어는 별도 체계가 필요하다.

### 추천 사용 시나리오

- 여러 에이전트 결과를 눈으로 비교하고 검토해야 하는 제품·플랫폼 팀.
- 설계안, diff, 대시보드, 프로토타입 같은 시각 결과물을 빠르게 공유해야 하는 협업 환경.
- 멀티에이전트 흐름을 IDE 안에서 자연스럽게 운영하고 싶은 팀.

### 현재 시점 총평

Cursor는 지금 가장 제품화된 멀티에이전트 워크스페이스에 가깝다. 최근 공개 신호도 모델 자체보다 시각화된 산출물, 병렬 비교, 장기 멀티에이전트 시스템으로 수렴하고 있다.

## 3. OpenAI Codex

### 강점

- 2026-04-16 업데이트로 background computer use, 인앱 브라우저, 이미지 생성, 90개 이상 추가 플러그인, 리뷰 코멘트 처리, SSH devbox, 자동화 재개, 메모리 프리뷰까지 들어오며 제품 폭이 크게 넓어졌다.
- 최신 도움말 기준으로 앱, CLI, IDE extension, 웹이 더 명확히 한 제품군으로 정리되고 있다.
- 기능뿐 아니라 크레딧, 플랜, Usage Dashboard 같은 운영·과금 표면도 빠르게 정리되고 있어 조직 도입 판단이 쉬워지고 있다.

### 약점

- 공개 정보가 제품 블로그, Help Center, Academy로 나뉘어 있어 탐색 동선이 길다.
- 최근 변화가 매우 넓은 범위를 다루기 때문에 실제 팀에 필요한 부분만 추려 적용해야 한다.
- 로컬 중심 깊은 제어보다는 넓은 표면 확장에 초점이 있어, 세밀한 운영 규칙은 아직 팀별 보완이 필요하다.

### 추천 사용 시나리오

- 코드 작성뿐 아니라 리뷰, 브라우저 확인, 문서 연결, 플러그인, 장기 자동화를 한 허브에서 다루고 싶은 팀.
- ChatGPT 기반 계정 체계와 플랜 관리가 이미 자리 잡은 조직.
- 개발자뿐 아니라 비개발 부서까지 같은 에이전트 제품을 확장 도입하려는 환경.

### 현재 시점 총평

Codex는 이번 주를 기점으로 가장 빠르게 표면을 넓히는 제품으로 다시 올라왔다. 이제 비교 포인트는 "코드를 잘 쓰느냐" 하나가 아니라, 개발 수명주기 전반을 한 작업 공간에서 얼마나 자연스럽게 이어 주느냐에 가깝다.

## 이번 주 총평

이번 주 시장은 한 단계 더 분명해졌다. 승부처는 더 똑똑한 한 번의 답변이 아니라, 에이전트를 계속 돌리고, 기억을 붙이고, 결과를 읽기 좋게 만들고, 브라우저와 앱과 PR과 문서를 하나의 흐름으로 엮는 운영 계층이다.

GitHub Trending도 같은 방향을 가리킨다. 가장 많이 뜬 저장소들이 모델 래퍼보다 규칙 파일, 스킬, 장기 메모리, 관리형 에이전트, 하네스, 문서 정규화 계층을 다루고 있었다. 지금 팀이 먼저 정해야 할 것은 "어느 모델이 제일 세냐"보다 "우리 팀의 규칙, 메모리, 검증, 산출물 포맷을 어떻게 운영체제로 만들 것인가" 이다.

## 주요 트렌드 관찰

1. 규칙 파일과 스킬 묶음이 모델 성능 못지않게 중요한 차별화 포인트가 되고 있다.
2. 메모리와 장기 자동화가 에이전트 도구의 기본 기능으로 빠르게 이동하고 있다.
3. Cursor와 Codex는 시각 결과물, 브라우저, 앱 제어처럼 에이전트가 닿는 표면을 넓히고 있다.
4. Claude Code는 여전히 가장 정교한 CLI 운영형 에이전트로, 권한과 복귀 흐름을 깊게 다듬고 있다.
5. 문서 정규화와 하네스 설계 같은 입력/운영 계층이 생산성 체감치를 크게 좌우하고 있다.
