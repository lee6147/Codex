# Presentation Style

이 문서는 AI 트렌드 보고서의 가독성 기준을 정의한다.
정보가 맞더라도 읽기 불편하면 실패로 간주한다.

## 목표

- 첫 화면 10초 안에 보고서 성격과 핵심 변화가 보여야 한다.
- 표를 끝까지 읽지 않아도 상단 요약만으로 이번 흐름이 파악돼야 한다.
- 각 항목은 카드 또는 명확한 블록으로 끊어져 스캔 가능해야 한다.
- 긴 문단보다 짧은 문장, 배지, 요약 불릿, 메타 라인을 우선한다.

## HTML 스타일 규칙

- 상단에 강한 헤더를 둔다.
- 헤더에는 보고서 제목, 생성일, 조사 기간, 기간 확장 여부를 담는다.
- 헤더 바로 아래에 요약 박스 또는 핵심 통계 박스를 둔다.
- 목차는 별도 카드로 분리한다.
- 본문은 가능하면 카드형 섹션으로 작성한다.
- 거대한 단일 표는 피한다.
- 태그나 배지를 적극적으로 사용한다.
- 각 카드에서 무엇이 바뀌었는지, 왜 중요한지, 출처가 무엇인지가 빠르게 보여야 한다.

## Markdown 스타일 규칙

- 제목에 날짜를 함께 적어 구분되게 한다.
- 상단에 해시태그 또는 태그 묶음을 배치해 Obsidian 탐색성을 높인다.
- 본문 첫 부분에 어떤 HTML 산출물을 종합한 문서인지 명시한다.
- 도구별 평가는 같은 소제목 순서를 반복해 리듬을 만든다.
- 도구 간 비교는 가능하면 비교 표 1개로 압축한다.
- 마지막에는 이번 주 핵심 트렌드와 다음 체크포인트를 따로 둔다.

## 금지 패턴

- 모든 정보를 한 개의 큰 표로 밀어 넣는 형태
- 스캐폴드 문구만 지우고 구조는 그대로 두는 형태
- 제목, 요약, 출처가 한 덩어리로 붙어 시선 분리가 안 되는 형태
- 카드 구분 없이 긴 문단을 연속 배치하는 형태

## 최소 통과선

- HTML 3개 모두 헤더, 요약 영역, 목차, 본문 카드, 출처 블록을 가져야 한다.
- Markdown은 제목, 태그, 소스 연결, 도구별 평가, 비교 표, 트렌드 요약을 가져야 한다.
- 사용자가 스크롤 없이도 상단에서 이번 보고서의 핵심을 읽을 수 있어야 한다.
