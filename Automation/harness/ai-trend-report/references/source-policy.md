# Source Policy

## 핵심 원칙

- 가능하면 1차 또는 공식 출처를 우선한다.
- 확인된 사실만 쓴다.
- 추측하지 않는다.
- 확인이 불가능한 정보는 `확인 불가`로 적는다.
- 실제 게시일을 `YYYY-MM-DD`로 적는다.
- 같은 사건의 중복 보도는 대표 출처 1개에서 2개 정도로 줄인다.
- 한국어로 작성한다.
- 중요한 주장에는 링크를 남긴다.

## 공식 업데이트 규칙

- 공식 출처가 있으면 2차 보도는 제외한다.
- 공식 출처는 없지만 여러 신뢰 가능한 매체가 동일 사실을 보도했다면 `공식 확인 불가` 구역으로 분리해 적을 수 있다.

## 아티클 선별 규칙

- 실사용 경험, 비교, 워크플로 개선, 구체적 구현 내용이 있는 글을 우선한다.
- 정보량이 낮은 요약 재탕 글은 제외한다.
- 홍보성이 강하면 `광고성 강함`으로 표시한다.

## 트렌딩 해석 규칙

- GitHub Trending에 인위적으로 최근 3일 규칙을 씌우지 않는다.
- 조사 시점에 보이는 Daily 또는 Weekly 스냅샷을 최신 흐름으로 해석한다.
- AI 코딩 또는 개발 생산성과의 연결성이 약한 저장소는 제외한다.

## 완료 체크리스트

완료 전에 아래를 모두 확인한다.

- HTML 3개와 Markdown 1개가 모두 존재하는가
- 폴더명이 오늘 날짜와 일치하는가
- 공식 업데이트 3일, 아티클 2일 우선 규칙이 지켜졌는가
- 7일 확장이 있었다면 분명히 표시했는가
- 주요 주장에 출처 링크가 있는가
- 추측성, 중복성, 광고성 항목이 줄었거나 표시되었는가
- 최종 Markdown이 실제 HTML 내용을 반영하는가

가능하면 위 점검 전에 `scripts/verify_outputs.py`를 실행해 기본 형식 오류와 스캐폴드 잔존 여부를 먼저 확인한다.
