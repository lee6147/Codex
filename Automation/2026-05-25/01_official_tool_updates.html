Claude Code

확인 완료 공식 changelog + GitHub release

v2.1.150 · 2026-05-23

  • 공식 changelog는 이 릴리스를 내부 인프라 개선으로 분류하며 사용자-facing 변경은 없다고 적는다.
  • GitHub 릴리스도 v2.1.150을 최신 릴리스로 표시한다. 공개 릴리스 노트 본문은 제한적이므로 기능 변화로 과장하지 않는다.

직전 운영 맥락: v2.1.149 · 2026-05-22

  • /usage가 skills, subagents, plugins, MCP 서버별 사용량 원인을 나눠 보여 주도록 바뀌었다.
  • PowerShell cd 계열 함수가 작업 디렉터리를 몰래 바꿔 workspace 밖 읽기로 이어질 수 있던 권한 분석 문제가 수정됐다.
  • git worktree에서 sandbox write allowlist가 main repo 전체로 넓어지던 문제를 공유 .git 범위 중심으로 좁혔다.
  • PowerShell prefix/wildcard allow 규칙과 PWD/OLDPWD 변수 추적 문제도 함께 정리됐다.

실무 의미: 2026-05-25 현재 Claude Code의 새 strict-window 신호는 안정화 성격이다. Windows/PowerShell, worktree, sandbox 기반 자동화에서는 2.1.149 이상을 기준으로 보는 것이 안전하다.

Cursor

strict window 신규 공식 항목 미확인 근접 공식 맥락
  • 2026-05-20 공식 changelog는 Cursor Automations를 Agents Window 안으로 넣고, 여러 repo를 붙이거나 repo 없이 실행하는 automation 구성을 추가했다.
  • no-repo automation 템플릿은 Slack digest, product analytics, product FAQ, finance, customer health agent처럼 코드 밖 업무 신호를 감시하는 쪽으로 확장됐다.
  • 2026-05-19 공식 changelog는 Jira work item을 Cursor cloud agent에 할당하거나 @Cursor mention으로 시작하는 통합을 발표했다.
  • 2026-05-18 Composer 2.5 발표는 장기 작업, 복잡한 지시 이행, effort calibration, targeted RL, 25배 많은 synthetic task 훈련을 핵심으로 설명한다.

실무 의미: Cursor의 최신 방향은 IDE 안의 agent console에서 업무 시스템과 repo를 동시에 다루는 것이다. 다만 이번 strict window에는 신규 공식 업데이트가 없어 날짜를 근접 맥락으로 분리해야 한다.

OpenAI Codex

확인 완료 GitHub release + OpenAI Help Center

Codex CLI 0.134.0-alpha.1 · 2026-05-22

  • GitHub 릴리스 페이지는 0.134.0-alpha.1을 2026-05-22 pre-release로 표시한다.
  • 릴리스 페이지에서 사용자-facing 상세 항목은 제한적이므로 안정 채널 기능으로 간주하지 않는다.

안정 채널 기준점: 0.133.0 및 Codex 26.519 · 2026-05-21

  • 0.133.0은 Goals 기본 활성화, 전용 저장소와 진행 추적, codex remote-control 구조 변경, permission profile API/상속, marketplace-aware plugin discovery를 포함한다.
  • OpenAI Help Center의 2026-05-21 Codex 릴리스 노트는 Appshots, Goal mode GA, in-app browser annotation, locked computer use, browser use 개선을 공지했다.
  • Appshots는 macOS Codex app에서 앱 창의 screenshot과 available text를 thread에 붙여 컨텍스트 설명 비용을 줄이는 기능으로 설명된다.

실무 의미: 0.134.0-alpha.1은 추적 대상이지만, 팀 운영 기준으로는 0.133.0 + Codex 26.519가 더 확실한 기준선이다. Goal mode가 app, IDE extension, CLI 전반에서 일반 사용 가능해진 점이 가장 큰 운영 변화다.
