Claude Code 기사

Claude Code v2.1.149 changelog 해설

changelogs.directory2026-05-23 확인

v2.1.149가 usage breakdown, diff keyboard scrolling, PowerShell permission bypass, sandbox write allowlist 문제를 정리했다는 점을 요약한다. 공식 changelog를 재구성한 페이지라 독립 분석은 제한적이지만, 운영자가 확인해야 할 항목을 한 화면에서 보기 좋다.

실무 시사점: Windows PowerShell 또는 git worktree를 쓰는 팀은 v2.1.149 이상으로 업데이트했는지 확인해야 한다. 특히 "workspace 밖 읽기"와 "main repo 전체 write allowlist"는 자동화 에이전트의 권한 경계 문제라 단순 UX 개선보다 중요하다.

Claude Code v2.1.147 보안/운영 맥락

ClaudeKit2026-05-22

v2.1.147의 managed-settings 우회 수정, REPL/Workflow sandbox hardening, Windows/PowerShell 관련 수정 흐름을 설명한다. v2.1.149 기사와 함께 보면 2026-05-21~22 릴리스가 기능 추가보다 경계 보강에 집중했음을 알 수 있다.

실무 시사점: Claude Code를 조직 정책 아래에서 쓰는 경우 managed settings와 third-party/API-key session 경계가 실제로 enforce되는지 점검해야 한다.

Cursor 기사

Composer 2.5 benchmark와 가격 분석

DataCamp2026-05-22

Composer 2.5가 Cursor IDE 안의 장기 coding session, tool use, multi-file editing에 맞춰 훈련됐고, targeted textual feedback과 25배 많은 synthetic task가 핵심이라고 설명한다. benchmark는 유용하지만 CursorBench가 내부 benchmark라는 한계도 분명히 짚는다.

실무 시사점: Cursor 안에서 일상적인 refactor/debugging을 많이 한다면 Composer 2.5 Standard의 비용 이점이 크다. 다만 외부 API가 없고, frontier model과 benchmark 조건이 완전히 같지 않다는 점을 비용 비교에 반영해야 한다.

Cursor Composer 2.5 비용 중심 해석

Digital Applied2026-05-18

Composer 2.5를 "agent coding at lower cost" 관점에서 다루며 Cursor의 공식 발표와 가격 포지셔닝을 해석한다. 공식 benchmark를 받아쓰는 성격이 강하므로, 독립 검증보다는 시장 반응 자료로 보는 편이 맞다.

실무 시사점: Cursor가 editor wrapper에서 자체 coding model stack으로 이동하고 있다는 신호다. 구매 결정에는 실제 조직 코드베이스에서의 오류 복구, 테스트 실행, PR 품질을 별도 측정해야 한다.

OpenAI Codex 기사

OpenAI Codex Appshots와 Goal mode

AIhola2026-05-22

Codex의 Appshots를 macOS에서 현재 보고 있는 앱 창을 thread에 붙이는 기능으로 설명하고, Goal mode가 app, IDE extension, CLI 전반에서 GA가 됐다는 점을 정리한다. 공식 changelog와 기능 UX를 짧게 연결한 기사다.

실무 시사점: 화면/앱 컨텍스트를 설명하는 시간이 줄어들 수 있지만, Screen Recording과 Accessibility 권한을 요구하므로 보안 정책 검토가 필요하다.

Codex for Mac Appshots 보도

9to5Mac2026-05-21

Mac 사용자가 Command 키 조합으로 app window를 Codex thread에 붙일 수 있다는 점을 사용자 관점에서 설명한다. OpenAI Developers의 공개 메시지를 근거로 Appshots, upgraded goal mode, faster in-app browser, shared plugins를 함께 다룬다.

실무 시사점: frontend, 디자인 리뷰, 문서/브라우저 비교처럼 화면 상태가 중요한 작업에서 prompt 작성 부담을 줄인다. Windows 사용자는 해당 기능 범위를 macOS-only로 구분해야 한다.
