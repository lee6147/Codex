Claude Code

확인 완료 공식 changelog + GitHub release

2.1.150 · 2026-05-23

  • 공식 changelog와 GitHub release 모두 “내부 인프라 개선, 사용자-facing 변경 없음”으로 정리된다.
  • 실무 영향: 즉시 워크플로를 바꿀 기능보다는 릴리스 안정화 성격이다.

2.1.149 · 2026-05-22

  • /usage가 skills, subagents, plugins, MCP 서버별 비용 등 한도 사용 원인을 더 세분화해 보여준다.
  • /diff detail view 키보드 스크롤과 GFM task-list 체크박스 렌더링이 추가됐다.
  • PowerShell cd 계열 함수로 작업 디렉터리가 몰래 바뀌며 workspace 밖 읽기로 이어질 수 있던 권한 우회가 수정됐다.
  • git worktree에서 sandbox write allowlist가 main repository root 전체로 넓어지던 문제가 수정됐다.

관찰: Windows/PowerShell 환경의 Claude Code 사용자는 2.1.149 이상을 우선 확인하는 편이 좋다. 이번 항목은 생산성 기능보다 권한 모델의 신뢰도에 더 가깝다.

Cursor

strict window 신규 공식 항목 없음 근접 공식 맥락

가장 가까운 공식 업데이트

  • 2026-05-20: Automations가 Agents Window 안으로 들어오고, multi-repo/no-repo automation 구성이 추가됐다.
  • 2026-05-19: Jira에서 work item을 Cursor cloud agent에 할당하거나 @Cursor mention으로 작업을 시작할 수 있는 통합이 공개됐다.
  • 2026-05-18: Composer 2.5가 Cursor에 제공되며 장기 작업 지속성, 복잡한 지시 준수, 협업 경험 개선을 강조했다.

관찰: Cursor의 방향은 IDE 안의 코딩 보조에서 “업무 시스템과 연결된 상시 agent 운영”으로 확장 중이다. 다만 오늘 strict window 안에서 새 공식 릴리스는 확인 불가다.

OpenAI Codex

확인 완료 GitHub release + OpenAI Help Center

Codex CLI 0.134.0-alpha.1~3 · 2026-05-22~23

  • GitHub releases에 0.134.0-alpha.1, alpha.2, alpha.3 프리릴리스가 확인된다.
  • 릴리스 페이지에는 상세 기능 설명이 제한적으로 노출되어 있어, 사용자-facing 변화는 확인 불가로 둔다.

Codex CLI 0.133.0 · 2026-05-21, 근접 공식 맥락

  • Goals가 기본 활성화되고, dedicated storage와 active turn progress tracking을 갖췄다.
  • codex remote-control의 readiness/status 동작과 foreground command 성격이 강화됐다.
  • permission profiles, plugin discovery, extension lifecycle event 관찰성이 넓어졌다.

Codex app / ChatGPT release notes · 2026-05-21, 근접 공식 맥락

  • Appshots는 macOS Codex app에서 다른 앱 창의 screenshot과 available text를 thread에 붙이는 기능이다.
  • Goal mode는 Codex app, IDE extension, CLI 전반에서 generally available로 표시됐다.
  • Browser annotations, locked computer use, browser-use 개선도 함께 공지됐다.

관찰: Codex는 “장기 목표 실행 + 외부 앱/브라우저 문맥 주입”을 제품 전면으로 밀고 있다. 0.134 alpha는 추적 대상이지만 운영 적용은 stable 릴리스 노트를 기다리는 편이 안전하다.
