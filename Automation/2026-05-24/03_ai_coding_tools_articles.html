Claude Code

Claude Code v2.1.149 분석

게시일: 2026-05-23 매체: Claude Media 비공식 리뷰

v2.1.147 기능 릴리스 이후 v2.1.148 hotfix와 v2.1.149 권한/샌드박스 정비가 이어졌다는 흐름을 설명한다. Windows PowerShell 사용자, git worktree 사용자, managed settings 운영자에게 영향이 크다고 정리한다. 공식 changelog와 맞물려 보아도 2.1.149의 핵심은 새 기능보다 권한 분석 보강이다.

실무 시사점: Windows 팀은 Claude Code 버전 고정과 업데이트 점검을 automation checklist에 넣는 것이 좋다.

Claude Code 공식 changelog

게시일: 2026-05-22/23 작성자: Anthropic 공식 글

2.1.149는 usage breakdown, diff keyboard scrolling, GFM checkbox rendering 같은 작업 편의 개선과 PowerShell permission bypass, worktree sandbox allowlist, PowerShell prefix/wildcard allow rule 수정 등을 포함한다. 2.1.150은 사용자-facing 변화가 없는 내부 인프라 개선으로 표시된다.

실무 시사점: 기능보다 security boundary 회복이 더 중요하다. Windows/PowerShell과 worktree 조합에서 특히 업데이트 확인이 필요하다.

Cursor

Improvements to Cursor Automations

게시일: 2026-05-20 작성자: Cursor 공식 글

Cursor Automations가 Agents Window 안으로 들어가고, multi-repo automations와 no-repo automations가 추가됐다. Slack digest, product analytics, FAQ, finance, customer health 같은 no-repo template도 함께 제시됐다. 할인 공지까지 포함되어 adoption 촉진 의도가 뚜렷하다.

실무 시사점: Cursor는 “IDE 안 coding agent”를 넘어 monitoring/reporting/ops automation까지 확장한다.

Cursor Automations Agents Window 기사

게시일: 2026-05-20 매체: AI อะไรเนี่ย AI drafted, human reviewed

공식 changelog의 핵심을 기사형으로 요약하면서 Agents Window 편입, multi-repo, no-repo template을 강조한다. 독립 정보량은 크지 않지만 제품 변화의 사용자-facing 의미를 빠르게 파악하는 데는 유용하다.

실무 시사점: Jira/Slack/analytics 연동형 agent 운영을 Cursor 안에서 묶으려는 팀은 공식 문서를 먼저 확인해야 한다.

OpenAI Codex

Codex updates: richer context, goal mode, browser improvements

게시일: 2026-05-21 작성자: OpenAI 공식 글

OpenAI는 Codex app의 Appshots, Goal mode GA, in-app browser annotations, locked computer use, browser-use improvements를 한 번에 공지했다. Appshots는 Mac app window의 screenshot과 available text를 thread에 붙이고, Goal mode는 app/IDE/CLI across surfaces로 일반 제공된다.

실무 시사점: Codex는 “더 오래 맡기기”와 “현재 화면 맥락을 덜 설명하기”를 동시에 줄이는 방향으로 이동 중이다.

Codex for Mac Appshots 보도

게시일: 2026-05-21 매체: 9to5Mac 뉴스 기사

Mac 사용자가 Command-Command로 frontmost app window를 Codex thread에 붙이는 Appshots 경험을 중심으로 설명한다. OpenAI Developers의 공지와 연결하며, Appshots가 screenshot뿐 아니라 available text까지 포함한다고 보도한다.

실무 시사점: 디자인, 브라우저, IDE, 문서 등 다른 앱 상태를 설명하느라 생기던 prompt friction을 줄일 수 있다.

OpenAI Adds Appshots and Goal Mode to Codex

게시일: 2026-05-22 매체: AIhola 비공식 기사

Appshots와 Goal mode GA를 짧게 정리하고, Screen Recording/Accessibility permission이 필요하다는 운영상 포인트를 짚는다. 공식 release notes를 보완하는 사용자 관점 설명으로 쓸 만하지만, 상세 정책은 OpenAI 문서로 재확인해야 한다.

실무 시사점: enterprise rollout에는 macOS 권한 설정, 보안 안내, 녹화/접근성 승인 절차가 필요하다.
