핵심 신호
Claude Code는 최신 패키지/체인지로그 흐름이 가장 활발하고, Cursor와 Codex는 직전 공식 릴리스의 운영 기능이 이번 주 기준선이다.

권한, 모델 제한, 브라우저 디버깅, 리뷰 자동화처럼 에이전트 운영 안정성 쪽 업데이트가 주류다.

요약

최근 3일 안에 Cursor와 Codex의 새 공식 changelog 항목은 확인되지 않았다. 대신 Cursor의 2026-06-10 Bugbot 업데이트와 OpenAI Codex app 26.609(2026-06-11)가 이번 주 사용 판단에 가장 직접적이다. Claude Code는 공식 GitHub changelog 최신 항목이 2.1.176까지 보이며, npm 레지스트리는 2026-06-15 수정 시각과 더 최신 패키지 타임라인을 노출하지만 GitHub changelog 원문은 항목별 게시일을 제공하지 않는다.

확인 불가 또는 주의

Claude Code 공식 changelog에는 버전별 날짜가 없으므로 날짜는 npm 레지스트리 보조 신호와 구분해 기록했다. npm 조회에서는 `version` 필드와 `time` 목록 사이에 2.1.177/2.1.178 차이가 보여, 보고서 본문에는 GitHub changelog 기준 최신 공개 내용과 npm 수정 시각을 분리했다.

Claude Code

2.1.176 changelog: 세션, 권한, Remote Control, background agent 안정화

공식 changelog 확인게시일 확인 불가

최신 GitHub changelog 상단은 2.1.176이다. 핵심은 대화 언어 기반 세션 제목, footer link badge 설정, Bedrock credential 캐싱 개선, `availableModels` 제한 우회 방지, Fable 5 auto mode fallback, Read/Edit/Write hook 경로 매칭 수정이다.

실무 관점에서는 조직 관리 설정이 모델 선택과 권한을 더 강하게 통제하고, tmux/SSH/Windows/Remote Control/background agent의 끊김과 오표시를 줄이는 방향이다. 대규모 작업을 여러 세션으로 돌리는 팀에는 기능 추가보다 운영 신뢰성 개선으로 보는 편이 맞다.

발표일: 공식 changelog 원문 기준 확인 불가 | 보조 확인: npm registry modified 2026-06-15T19:03:37Z | 확인 상태: 확인 완료/일부 날짜 확인 불가

최근 버전 흐름: background, model allowlist, sandbox, agent view

운영 안정성엔터프라이즈 설정

2.1.172~2.1.176 구간은 nested subagents, Bedrock region 처리, plugin 검색, model allowlist enforcement, background-session attach/respawn, Windows agent view 수정이 반복된다. 새 모델 발표보다 에이전트 실행면을 안정화하는 업데이트가 많다.

Claude Code를 업무 표준으로 쓰는 팀은 모델 allowlist, managed settings, background daemon 상태, Remote Control 접속 정책을 먼저 점검해야 한다.

Cursor

Bugbot: 3배 이상 빠른 리뷰, 22% 비용 감소, 10% 더 많은 버그 발견

공식 changelog2026-06-10

Cursor의 최신 공식 항목은 2026-06-10 Bugbot 개선이다. 평균 리뷰 시간이 약 5분에서 약 90초로 줄고, 리뷰당 평균 버그 발견 수는 0.56에서 0.62로 올라갔으며, 실행 비용은 약 22% 낮아졌다고 발표했다.

`/review`, `/review-bugbot`, `/review-security`로 push 전 검토를 실행하고, 같은 diff가 PR로 올라오면 GitHub/GitLab Bugbot 리뷰와 동기화하는 흐름도 추가됐다. CLI 지원은 “coming soon”으로 표시되어 현재는 Cursor 3.7+ 및 cursor.com/agents 중심이다.

발표일: 2026-06-10 | 출처: Cursor changelog | 확인 상태: 확인 완료

SDK 및 Design Mode 직전 업데이트도 이번 주 판단에 유효

SDKDesign Mode

2026-06-04~05 업데이트에서는 Design Mode multi-select/voice input, SDK custom tools, local auto-review, JSONL/custom stores, nested subagents, requestId, Windows bundled ripgrep 개선이 발표됐다. Cursor는 IDE 내부 작업 지시와 리뷰 자동화를 한 화면에서 묶는 쪽으로 강하게 간다.

OpenAI Codex

Codex app 26.609: Browser Developer mode, reset banking, Computer Use, 자동화 수정

공식 changelog2026-06-11

OpenAI 개발자 문서의 Codex changelog는 2026-06-11 Codex app 26.609를 최신 주요 항목으로 제시한다. 새 기능은 Plus/Pro rate-limit reset banking, referral invitations, Browser Developer mode, `/init`, macOS Dock 아이콘, Enterprise Computer Use, Windows per-app Computer Use controls, Unread chats section이다.

성능/수정 항목에서는 CDP와 DOM snapshot 최적화로 Browser use가 최대 2배 빨라졌다고 설명한다. scheduled automations가 선택한 approval mode를 따르도록 수정된 점은 이 자동화 워크스페이스와 직접 관련 있다.

발표일: 2026-06-11 | 출처: OpenAI Developers Codex changelog | 확인 상태: 확인 완료

실무 의미: Codex는 로컬 앱, 브라우저, 자동화, 엔터프라이즈 권한을 묶는 방향

브라우저 디버깅자동화권한 관리

이번 업데이트는 코드 생성 품질보다 실행 환경 관리에 초점이 있다. Browser Developer mode와 Computer Use 정책, automations approval mode 수정은 Codex가 “작업 실행면”을 넓히는 과정에서 필요한 안전장치로 해석된다.
