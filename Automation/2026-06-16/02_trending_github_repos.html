요약

GitHub 쪽 신호는 “코딩 에이전트 자체”에서 “에이전트를 오래 굴리기 위한 주변 시스템”으로 이동 중이다. 상위 후보에는 system prompt 아카이브, Claude/Codex/Cursor 공통 메모리 계층, long-horizon harness, local code graph, skill library, agent security scanner가 반복적으로 잡힌다.

조사 방식 메모

GitHub Trending Daily/Weekly 페이지는 이번 실행에서도 repository card를 안정적으로 추출하지 못했다. 따라서 `pushed:>2026-06-09` 조건의 GitHub Search API 결과를 사용했고, 별점은 조회 시점의 누적 star 수다. 이는 “Trending 순위”가 아니라 최신 push가 있는 관련 저장소의 fallback metadata다.

트렌드 해석

1. Context와 memory가 제품 차별점이 됨

`claude-mem`, code graph 계열 저장소는 agent가 한 번의 세션을 넘어 지식을 유지하도록 돕는다. Claude Code, Codex, Cursor 사용자는 이제 모델 성능보다 “어떻게 같은 프로젝트 맥락을 반복 주입하지 않을 것인가”에 관심이 커지고 있다.

2. Skill/command library가 새로운 플러그인 표면

`agent-skills`, `antigravity-awesome-skills`, `taste-skill` 같은 저장소는 에이전트 프롬프트와 작업 규약을 재사용 가능한 패키지로 다룬다. 팀 표준화와 품질 통제에 가까운 흐름이다.

3. 보안과 검증 도구가 뒤따라옴

`hol-guard`, MCP devtools, review/graph 도구는 agent에게 더 많은 권한을 줄수록 실행 전 검사와 산출물 검증이 필요하다는 수요를 반영한다.
