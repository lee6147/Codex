요약

이번 조사에서 가장 직접적인 업데이트는 Cursor와 Codex다. Cursor는 Bugbot 성능과 사전 PR 리뷰 흐름을 개선했고, OpenAI는 Codex 앱 26.608 및 iOS Codex 기능을 공개했다. Claude Code는 2026-06-08 릴리스에서 안전 모드, 작업 디렉터리 이동, Windows 지연 문제 수정 등 운영 안정성 중심 개선을 냈다.

실무 관점에서는 "AI가 코드를 쓰는가"보다 "어디에서 안전하게 실행하고 검토하는가"가 더 중요해졌다. 세 도구 모두 리뷰, 권한, 세션, 플러그인, 작업 환경 전환에 기능을 집중하고 있다.

도구별 공식 업데이트

Claude Code

공식 GitHub 릴리스 2026-06-08

업데이트: v2.1.169 릴리스에서 --safe-mode, /cd, bundled skills 비활성화 설정, MCP 정책 enforcement 수정, Windows에서 claude -p가 느리거나 멈추는 회귀 수정이 포함됐다.

중요성: Claude Code 사용자가 플러그인, 스킬, hooks, MCP 서버를 많이 쓰는 환경에서 문제 원인을 빠르게 좁힐 수 있다. Windows와 원격 세션 관련 수정도 많아 팀 운영 안정성에 직접 연결된다.

Cursor

공식 changelog 2026-06-10

업데이트: Bugbot이 평균 리뷰 시간을 약 5분에서 약 90초로 줄였고, 리뷰당 버그 발견 수가 0.56에서 0.62로 증가했으며, 실행 비용이 약 22% 낮아졌다고 공지했다.

추가 흐름: /review, /review-bugbot, /review-security로 push 전에 Bugbot과 Security Review를 실행할 수 있고, 동일 diff가 PR로 올라오면 GitHub/GitLab 리뷰와 중복 실행을 줄인다.

OpenAI Codex

공식 Codex changelog GitHub release

업데이트: 2026-06-09 Codex app 26.608은 Claude Code와 Claude Cowork 설정 가져오기, 플러그인 화면 개편, Settings 검색 확대, diff 정렬 및 Windows 10 렌더링 개선을 포함했다.

CLI 쪽 신호: openai/codex 0.138.0은 /app 명령을 통한 CLI thread의 Codex Desktop handoff, 로컬 이미지 파일 경로 노출, reasoning effort 선택 개선, 플러그인 JSON 출력 확장을 포함했다.

실무 영향 비교

도구이번 주 핵심 변화개발자에게 중요한 이유주의점
Claude Code 안전 모드, 세션 디렉터리 이동, MCP/Windows 안정성 수정 커스터마이징이 많은 프로젝트에서 재현과 장애 격리가 쉬워진다. 대부분 bug fix와 운영 안정화라 기능 체감은 환경에 따라 다르다.
Cursor Bugbot 성능, 비용, pre-push 리뷰 흐름 개선 PR 이전에 리뷰 신호를 얻어 피드백 왕복을 줄일 수 있다. Cursor 3.7 이상 및 cursor.com/agents 중심이며 CLI 지원은 곧 제공 예정으로 표시됐다.
OpenAI Codex 앱 온보딩, 플러그인, Desktop handoff, 이미지 경로 처리 개선 CLI, 데스크톱, 모바일, 플러그인 사용 흐름이 더 연결된다. 일부 기능은 앱 버전, 플랫폼, 플랜에 따라 적용 범위가 다를 수 있다.
