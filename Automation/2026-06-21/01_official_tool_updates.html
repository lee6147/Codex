Claude Code

Claude Code 2.1.183: destructive command 차단과 Windows 안정화

공식 changelog 안전성 Windows
  • Auto-accept 모드에서 destructive command 차단이 추가돼 에이전트 자동 실행의 하한 안전선이 높아졌다.
  • WebSearch empty result 이슈와 Windows Terminal fullscreen corruption 문제가 함께 수정됐다.
  • 운영 관점에서는 새 기능보다 사고 방지와 실제 사용 중 마찰 감소에 초점을 둔 릴리스다.

Claude Code 2.1.181: /config 단순화와 네트워크 환경 보강

공식 changelog 설정성
  • /config key=value 문법이 추가돼 운영자 입장에서 설정 변경 경로가 단순해졌다.
  • Bun 1.4, 스트리밍 개선, auto-retry 보강이 포함됐고 네트워크 드라이브 truncated-file 문제도 수정됐다.
  • 개별 화제성은 낮지만, 장시간 자동화나 조직 내부 파일 환경에서는 체감 가치가 큰 종류의 업데이트다.

Cursor

Cursor 3.8 (2026-06-18): Automations를 외부 트리거와 연결

공식 changelog Automations 연동
  • /automate 진입점이 강화됐고 Slack emoji trigger, GitHub 신규 트리거 5종, computer use 연동이 추가됐다.
  • 미완료 automation 저장, PR tool 기본화, memory file 삭제 지원까지 묶이며 자동화 제품군이 한 단계 더 운영형 툴로 이동했다.
  • Cursor의 차별점은 모델 품질 경쟁보다 워크플로 연결성에서 더 선명해지고 있다.

OpenAI Codex

Codex App 26.616 (2026-06-18): Record & Replay와 실행 이력 관리

공식 changelog 재현성 앱

Codex CLI 0.141.0 (2026-06-18): 암호화된 원격 executor와 thread 단위 MCP

공식 changelog 원격 실행 CLI
  • 인증된 encrypted Noise relay channel이 원격 executor 연결에 도입됐다.
  • executor-native cwd/shell 유지, thread별 stdio MCP 활성화, realtime append 제어, TUI auto-resolve 개선이 함께 포함됐다.
  • 이는 단순 CLI 편의성보다 분산 실행과 운영 제어를 전제로 한 변화다.

종합 관찰

  • 세 도구 모두 새 모델 성능 과시보다 자동화 안전장치, 외부 시스템 연결, 원격 운영성 개선에 집중했다.
  • Claude는 안전한 자동 실행, Cursor는 트리거 연결성, Codex는 재현성과 원격 실행 인프라가 핵심 축이다.
  • 이번 주 공식 업데이트만 놓고 보면 승부처는 코드 생성 자체보다 운영 가능한 에이전트 시스템 구축 역량이다.
