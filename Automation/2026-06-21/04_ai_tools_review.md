## 1. Claude Code

### 강점

- destructive command 차단, WebSearch 보정, Windows 안정화처럼 실제 운영 사고를 줄이는 업데이트가 누적되고 있다.
- CLI 중심 사용성과 자동화 친화성이 계속 성숙하고 있어 고빈도 실무 루프에 잘 맞는다.
- 생태계 존재감이 커서 주변 자료, 패턴, skill, 운영 노하우가 빠르게 축적된다.

### 약점

- 공식 changelog에서 버전별 게시 날짜가 직접 보이지 않아 외부 비교 시 시간 정렬이 불편하다.
- 이번 주 외부 기사 레이어는 기능 찬사보다 공유 자산 악용, 악성 skill, 권한 위험 같은 보안 담론이 강했다.
- 생태계가 커질수록 서드파티 skill과 자동 승인 흐름이 supply-chain 리스크를 같이 끌고 온다.

### 추천 사용 시나리오

- 로컬 CLI 중심 자동화, 반복 코딩 작업, 긴 세션형 리팩터링.
- 안전 가드레일이 중요하지만 속도도 포기하기 싫은 개발팀.

### 피해야 할 경우

- 검증되지 않은 skill이나 공유 프롬프트를 광범위하게 유통해야 하는 환경.
- 권한 경계가 모호한 조직 자동화.

### 현재 시점 총평

Claude Code는 여전히 가장 실전적인 CLI형 에이전트 중 하나다. 다만 지금의 경쟁우위는 "똑똑함" 자체보다 "안전하게 오래 굴릴 수 있느냐"에서 증명되고 있다.

## 2. Cursor

### 강점

- Automations가 Slack, GitHub, computer use와 결합되며 IDE를 넘어 실행 허브로 확장되고 있다.
- 제품 UX와 진입 장벽이 낮아 개인 개발자부터 팀 단위 자동화까지 연결하기 쉽다.
- SpaceX 인수 서사는 장기적으로 컴퓨트, 훈련, 배포 인프라 지원 기대를 키운다.

### 약점

- 이번 주 외부 내러티브는 제품 완성도보다 인수와 전략 서사가 지배해 기술 본체 평가가 흐려졌다.
- 강한 자본 서사는 동시에 lock-in과 장기 전략 종속 리스크를 같이 만든다.
- multi-model 정체성과 제품 포지션이 더 넓어질수록 핵심 사용층에 대한 초점이 흐려질 수 있다.

### 추천 사용 시나리오

- IDE 안에서 바로 자동화 트리거를 연결하고 싶은 팀.
- Slack/GitHub 기반 협업 루프를 빠르게 붙이고 싶은 조직.

### 피해야 할 경우

- 전략적 독립성과 인프라 소유 구조를 매우 민감하게 보는 조직.
- 장기적으로 특정 플랫폼 종속을 최소화해야 하는 환경.

### 현재 시점 총평

Cursor의 무기는 여전히 좋은 개발 경험이지만, 이번 주에는 그 위에 "워크플로 운영 플랫폼"으로 변신하는 속도가 더 중요하게 보였다. 반대로 외부 평판은 시장 서사 영향도 크게 받는다.

## 3. OpenAI Codex

### 강점

- Record & Replay, automation history bulk actions, thread handoff로 운영 재현성과 시연 가능성이 강해졌다.
- encrypted remote executor, executor-native cwd/shell 유지, per-thread MCP 활성화 등 실행 계층이 빠르게 두꺼워지고 있다.
- 앱과 CLI가 함께 움직이며 단일 도구보다 런타임 체계로 확장되는 그림이 선명하다.

### 약점

- 이번 주 외부 기사량은 얇았고, 주요 노출도 장애/용량 이슈 기사 쪽에 치우쳤다.
- 강한 기능만큼 서비스 안정성과 capacity 운영 품질이 곧바로 평판 리스크가 된다.
- 기능 폭이 넓어질수록 운영 복잡도와 학습 비용도 함께 올라간다.

### 추천 사용 시나리오

- 데스크톱 자동화, 원격 executor, 재현 가능한 작업 로그가 중요한 팀.
- 단순 코드 생성보다 장기 작업 오케스트레이션을 중시하는 환경.

### 피해야 할 경우

- 서비스 가용성 민감도가 매우 높고 단순한 단일 IDE 경험만 원하는 사용자.
- 런타임 복잡도보다 즉시성만 필요한 작은 개인 워크플로.

### 현재 시점 총평

Codex는 기사량보다 제품 방향성이 더 중요한 케이스다. 외부 보도는 조용하지만 내부 기능 로그를 보면 "운영 가능한 에이전트 런타임"으로 가장 직접적으로 전진하고 있다.

## 추천 사용자 유형

- 개인 개발자의 즉시 생산성은 Cursor가 가장 빠르게 체감될 수 있다.
- 로컬 자동화와 반복 작업의 실전 안정성은 Claude Code가 여전히 강하다.
- 재현 가능한 실행 이력, 원격 executor, 복합 도구 연결이 중요하면 Codex가 가장 흥미롭다.

## 이번 주 총평

현 시점 경쟁은 더 이상 "누가 더 잘 코드를 쓰는가"만의 싸움이 아니다. 진짜 승부는 더 긴 작업을 더 안전하게, 더 연결성 있게, 더 재현 가능하게 굴리는 운영 능력에 있다.

## 주요 트렌드 관찰

1. MCP, skill, memory, compression, security scanning이 새로운 보조 인프라 층으로 급부상하고 있다.
2. 공식 릴리스와 GitHub 현장 수요, 외부 기사 서사가 모두 운영성과 안전성 중심으로 수렴하고 있다.
3. 기사량이 적은 제품일수록 언론 보도보다 공식 changelog를 더 세밀하게 읽어야 실제 방향을 놓치지 않는다.
