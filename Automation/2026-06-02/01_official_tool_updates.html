공식 AI 코딩 도구 업데이트

Codex, Claude Code, Cursor의 공식 발표만 우선 확인했다. 상대 날짜가 아니라 확인된 공개 날짜를 기준으로 정리했다.

핵심 요약

  • OpenAI Codex: 2026-06-01에 Codex CLI 0.136.0이 공식 공개됐다. TUI 링크/표 표시, 세션 아카이브, app-server, 원격 실행, Windows 샌드박스, 인증/보안 수정까지 포함된 비교적 큰 CLI 릴리스다.
  • Claude Code: 2026-05-31에 2.1.159가 올라왔지만, 공식 설명은 내부 인프라 개선이며 사용자 노출 기능은 없다고 명시했다.
  • Cursor: 엄격 창 안의 새 공식 업데이트는 확인하지 못했다. 가장 가까운 공식 릴리스는 2026-05-29 Cursor 3.6 Auto-review Run Mode다.

공식 확인

OpenAI Codex: CLI 0.136.0

공개일: 2026-06-01 · 출처: OpenAI Developers Codex changelog

이번 릴리스는 단순 패치보다 운영면 개선 폭이 크다. 터미널 UI의 Markdown 링크가 OSC 8 메타데이터를 유지하고, 좁은 표는 링크 대상을 잃지 않는 key/value 형태로 전환된다. 세션은 TUI의 /archive 또는 CLI의 codex archive/codex unarchive로 보관할 수 있으며, 보관된 세션은 복원 전까지 resume/fork에서 보호된다.

개발자 경험

링크 가능한 TUI 출력, 세션 아카이브, app-server의 초기 턴 페이지 재개와 MCP 서버 상태 노출이 개선됐다. 반복 리포트/검증 세션을 많이 다루는 팀에는 세션 보관 기능이 특히 유용하다.

보안/운영

ChatGPT 인증 갱신, 명령 안전성, 샌드박스 명령 정리, Windows 네트워크 거부 처리, PowerShell 실행 방어, browser-origin websocket 핸드셰이크 거부가 포함됐다.

확장성

원격 실행 등록에 CODEX_API_KEY를 쓸 수 있고, remote-control websocket은 ChatGPT access token 대신 단기 서버 토큰을 사용한다. Windows 관리자용 샌드박스 설치 경로도 alpha로 추가됐다.

공식 확인

Claude Code: 2.1.159

공개일: 2026-05-31 · 출처: Claude Code Docs changelog

공식 변경 내역은 “내부 인프라 개선, 사용자 노출 변경 없음”이다. 기능 채택 판단에는 큰 영향을 주지 않지만, 직전 2.1.158과 2.1.157에서 Auto mode, plugin/skills 로딩, agents, worktree, tmux, resume 관련 개선이 이어졌으므로 운영 안정화 흐름은 계속되고 있다.

엄격 창 밖

Cursor: 3.6 Auto-review Run Mode

공개일: 2026-05-29 · 출처: Cursor changelog

2026-05-31~2026-06-02 안에서 새 공식 Cursor 업데이트는 확인하지 못했다. 다만 가장 가까운 공식 항목인 Cursor 3.6은 Auto-review Run Mode를 추가했다. Shell, MCP, Fetch 호출을 대상으로 allowlist 실행, sandbox 실행, classifier subagent 판단을 조합해 승인 프롬프트를 줄이면서 장시간 작업을 더 안전하게 실행하도록 설계됐다.

운영 관찰 포인트

도구오늘의 의미팀 적용 판단
CodexCLI/앱 서버/보안/Windows 운영면이 동시에 강화됐다.터미널 기반 자동화, Windows 샌드박스, 원격 실행을 쓰는 팀은 업데이트 후보.
Claude Code오늘 창에서는 제품 기능보다 안정화 릴리스 성격이다.새 기능 검토보다 기존 워크플로 회귀 여부 확인에 초점.
Cursor엄격 창 밖이지만 승인 감소와 classifier 기반 실행 정책이 중요하다.장시간 agent run을 자주 돌리는 IDE 팀은 3.6 정책을 별도 검토.
