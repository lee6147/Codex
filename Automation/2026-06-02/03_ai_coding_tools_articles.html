Salt Security: Salt Code 발표

Claude Code, Cursor, GitHub Copilot, Windsurf, Kiro, Codex, Gemini CLI, Antigravity를 대상으로 AI-generated code 보안 정책을 개발 생명주기 안에서 적용하겠다는 발표다. 독립 기사라기보다 벤더 발표지만, 시장이 “agent 보안 정책” 레이어로 이동 중이라는 신호다.

반복되는 논점

논점관찰실무 의미
도구 수렴Claude Code, Cursor, Codex, Antigravity가 모두 planning, approval, tool use, background execution, parallelism 쪽으로 수렴한다.도구 선택은 모델 성능보다 팀의 승인 정책, repo 접근 방식, 비용 통제 방식으로 결정될 가능성이 커졌다.
검토 부담AI가 PR과 bug report 양을 늘리고, maintainer가 이를 검증하는 비용이 증가한다.agent 도입은 리뷰/테스트 자동화와 함께 설계해야 한다.
비용/품질 곡선Cursor 보고서는 모델별 요청 비용, accepted-line 비용, benchmark score 차이를 수치화한다.고성능 모델을 항상 쓰기보다 task routing과 cache/context 전략이 중요하다.
보안 정책Salt Code 같은 제품은 AI 코딩 assistant 내부에서 정책을 강제하려 한다.엔터프라이즈 도입은 “어떤 assistant인가”보다 “어떤 정책이 생성·리뷰·배포 단계에 적용되는가”가 중요해진다.

도구별 해석

  • Claude Code: 긴 terminal 세션, agent orchestration, 승인 기반 작업에 강한 이미지를 유지한다. 기사/후기에서는 자율성과 self-correction이 장점으로 반복된다.
  • Codex: 배경 작업, 대량 변경, ChatGPT/Codex 생태계 배포력, CLI/desktop/app-server 운영 개선이 강점이다. 오늘 공식 릴리스도 운영면 강화에 가깝다.
  • Cursor: IDE 안에서 multi-model, review, cloud agents, 자동화, 데이터 기반 cost-quality 관찰을 결합하는 방향이다. 3.6의 Auto-review는 승인 프롬프트를 줄이는 방향을 보여준다.
