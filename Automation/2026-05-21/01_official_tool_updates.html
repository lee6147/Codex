Claude Code v2.1.145

날짜 제한공식 내용 확인

공식 GitHub raw changelog 기준으로 v2.1.145에는 claude agents --json, OTEL span의 agent 식별자, GitHub repo/PR 정보를 포함하는 status line JSON, 플러그인 탐색 화면 개선, hook 입력 확장, Windows PowerShell 5.1 resume 힌트 수정 등이 포함됐다.

  • 중요한 이유: 팀 단위 Claude Code 운용에서 세션 목록 자동화, 관측성, 플러그인 설치 전 검토, Windows 호환성이 모두 운영 품질에 직접 연결된다.
  • 발표일: 공식 raw changelog에는 날짜 미표기. 비공식 추적 페이지는 2026-05-19로 표시하나, 본 보고서에서는 날짜를 확정하지 않는다.
  • 확인 상태: 기능 내용 확인 완료, 공식 발표일 확인 불가

출처: Anthropic Claude Code 공식 changelog raw, 날짜 참고: Claude Updates v2.1.145

Cursor 3.5: Automations 개선

확인 완료2026-05-20

Cursor Automations가 Agents Window 안으로 들어오고, 여러 저장소를 붙인 자동화와 저장소 없는 자동화가 가능해졌다. Cursor는 Slack digest, product analytics, FAQ, finance, customer health 같은 no-repo automation 템플릿도 함께 제시했다.

  • 중요한 이유: Cursor가 단순 IDE 보조에서 반복 업무를 맡는 에이전트 운영판으로 이동하고 있다.
  • 실무 영향: 멀티레포 제품, 데이터/지원/재무 알림처럼 코드 외부 신호를 함께 읽는 자동화에 유리하다.
  • 확인 상태: 확인 완료

출처: Cursor Changelog

Cursor in Jira

확인 완료2026-05-19

Jira work item에서 Cursor를 할당하거나 댓글로 호출해 cloud agent를 시작할 수 있다. 작업 완료 후 Jira에는 진행 상태와 PR 링크가 표시된다.

  • 중요한 이유: 기획 티켓에서 구현 PR까지 이어지는 흐름을 Cursor가 직접 연결한다.
  • 제약: Cursor admin access와 Jira Commercial Cloud with Rovo가 필요하다.
  • 확인 상태: 확인 완료

출처: Cursor Changelog

Cursor Composer 2.5

확인 완료2026-05-18, 엄격 기간 직전

Composer 2.5는 Cursor 내에서 사용 가능한 최신 Composer 모델로, 장기 작업 지속성, 복잡한 지시 준수, 협업 감각 개선을 강조한다. 표준 가격은 $0.50/M input, $2.50/M output으로 공개됐다.

  • 중요한 이유: Cursor는 IDE 경험뿐 아니라 자체 모델 품질과 가격 경쟁력으로 Claude Code/Codex와 직접 경쟁하고 있다.
  • 확인 상태: 확인 완료, 엄격 3일 창에서는 하루 앞선 참고 항목

출처: Cursor Changelog, Cursor Forum announcement

GPT-5.3-Codex: Copilot Business/Enterprise 기본 모델

2026-05-17, 엄격 기간 밖공식성 높음

GitHub는 Copilot Business와 Enterprise 조직의 기본 모델을 GPT-4.1에서 GPT-5.3-Codex로 전환했다. 이 모델은 OpenAI와의 파트너십에서 GitHub의 첫 LTS 모델로 지정됐고, 2027-02-04까지 사용 가능하다고 공지됐다.

  • 중요한 이유: Codex 계열 모델이 엔터프라이즈 Copilot의 기본값이 되면서 조직 단위 코드 생성·리뷰 품질의 기준선이 바뀐다.
  • 주의: 2026-05-19~2026-05-21 엄격 창 안의 OpenAI 1차 Codex 업데이트는 확인하지 못했다.
  • 확인 상태: GitHub 공식 확인 완료, 엄격 기간 내 OpenAI 1차 업데이트는 확인 불가

출처: GitHub Changelog
