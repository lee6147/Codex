## 1. Claude Code

### 강점

- destructive command 차단, WebSearch 보정, Windows 안정화처럼 실제 운영 사고를 줄이는 업데이트가 누적되고 있다.
- CLI 중심 사용성과 장기 루프 친화성이 계속 성숙하고 있어 고빈도 실무 자동화에 잘 맞는다.
- 외부 기사에서도 Claude는 "한 번 잘 답하는 모델"보다 "오래 굴리는 루프를 잘 설계하는 도구"로 인식되기 시작했다.

### 약점

- 공식 changelog에서 버전별 게시 날짜가 직접 보이지 않아 외부 비교 시 시간 정렬이 불편하다.
- 이번 주 외부 기사 레이어는 기능 찬사보다 Shared Claude Chats 악용, 악성 skill, 권한 위험 같은 보안 담론이 강했다.
- 생태계가 커질수록 공유 자산과 자동 승인 흐름이 supply-chain 리스크를 같이 끌고 온다.

### 추천 사용 시나리오

- 로컬 CLI 중심 자동화, 반복 코딩 작업, 긴 세션형 리팩터링.
- 안전 가드레일이 중요하지만 속도도 포기하기 싫은 개발팀.

### 피해야 할 경우

- 검증되지 않은 skill이나 공유 프롬프트를 광범위하게 유통해야 하는 환경.
- 권한 경계가 모호한 조직 자동화.

### 최근 업데이트 반영 관점

Claude Code의 이번 주 핵심은 더 똑똑해졌다는 주장보다, auto mode를 더 안전하게 만들고 긴 세션 마찰을 줄이는 데 있다. 이는 실제 사용 시간 총합을 늘리는 방향의 개선이다.

### 현재 시점 총평

Claude Code는 여전히 가장 실전적인 CLI형 에이전트 중 하나다. 현재 경쟁우위는 순수 모델 감탄보다 "안전하게 오래 굴릴 수 있느냐"에서 더 강하게 증명되고 있다.

## 2. Cursor

### 강점

- Automations가 Slack, GitHub, computer use와 결합되며 IDE를 넘어 실행 허브로 확장되고 있다.
- 제품 UX와 진입 장벽이 낮아 개인 개발자부터 팀 단위 자동화까지 연결하기 쉽다.
- cloud subagent와 local/cloud handoff 흐름은 실제 운영 확장성 면에서 매우 실용적이다.

### 약점

- 이번 주 외부 내러티브는 제품 완성도보다 SpaceX 인수와 전략 서사가 지배해 기술 본체 평가가 흐려졌다.
- 강한 자본 서사는 동시에 vendor lock-in과 장기 전략 종속 리스크를 같이 만든다.
- 실행형 플랫폼으로 넓어질수록 제품 범위와 학습 곡선도 함께 커질 수 있다.

### 추천 사용 시나리오

- IDE 안에서 바로 자동화 트리거를 연결하고 싶은 팀.
- Slack/GitHub 기반 협업 루프를 빠르게 붙이고 싶은 조직.

### 피해야 할 경우

- 전략적 독립성과 인프라 소유 구조를 매우 민감하게 보는 조직.
- 장기적으로 특정 플랫폼 종속을 최소화해야 하는 환경.

### 최근 업데이트 반영 관점

Cursor의 무기는 여전히 좋은 개발 경험이지만, 3.7과 3.8을 합쳐 보면 핵심 변화는 이제 IDE가 아니라 workflow operating layer에 있다. 사용자 입장에서는 "잘 도와주는 편집기"보다 "외부 시스템을 실행까지 묶는 제품"으로 봐야 한다.

### 현재 시점 총평

Cursor는 여전히 가장 빠르게 체감되는 개발 경험을 주는 축에 속한다. 다만 이번 주만 놓고 보면 제품 평판은 기술 자체보다 시장 서사와 소유 구조 변화의 영향을 크게 받는다.

## 3. OpenAI Codex

### 강점

- Record & Replay, automation history bulk actions, thread handoff로 운영 재현성과 시연 가능성이 강해졌다.
- 앱과 CLI가 함께 움직이며 단일 도구보다 런타임 체계로 확장되는 그림이 선명하다.
- 외부 기사에서도 Codex는 단발성 코드 생성기보다 장기 목표 수행과 orchestration 도구로 언급되기 시작했다.

### 약점

- 이번 주 외부 기사량은 얇았고, 주요 노출도 장애와 capacity 이슈 기사 쪽에 치우쳤다.
- 강한 기능만큼 서비스 안정성과 운영 품질이 곧바로 평판 리스크가 된다.
- 기능 폭이 넓어질수록 운영 복잡도와 학습 비용도 함께 올라간다.

### 추천 사용 시나리오

- 데스크톱 자동화, 원격 executor, 재현 가능한 작업 로그가 중요한 팀.
- 단순 코드 생성보다 장기 작업 오케스트레이션을 중시하는 환경.

### 피해야 할 경우

- 서비스 가용성 민감도가 매우 높고 단순한 단일 IDE 경험만 원하는 사용자.
- 런타임 복잡도보다 즉시성만 필요한 작은 개인 워크플로.

### 최근 업데이트 반영 관점

Codex는 기사량보다 제품 방향성이 더 중요한 케이스다. 공식 changelog를 보면 분명히 실행 이력, handoff, state persistence처럼 운영 계층을 밀고 있는데, 외부에서는 아직 안정성 기사로 먼저 소비된다.

### 현재 시점 총평

Codex는 "운영 가능한 에이전트 런타임"이라는 방향성이 가장 직접적인 도구다. 반면 그만큼 안정성, capacity, 복잡도 관리가 향후 평가를 좌우할 가능성도 가장 크다.

## 추천 사용자 유형

- 개인 개발자의 즉시 생산성은 Cursor가 가장 빠르게 체감될 가능성이 높다.
- 로컬 자동화와 반복 작업의 실전 안정성은 Claude Code가 여전히 강하다.
- 재현 가능한 실행 이력, 원격 executor, 복합 도구 연결이 중요하면 Codex가 가장 흥미롭다.

## 이번 주 총평

현 시점 경쟁은 더 이상 "누가 더 잘 코드를 쓰는가"만의 싸움이 아니다. 진짜 승부는 더 긴 작업을 더 안전하게, 더 연결성 있게, 더 재현 가능하게 굴리는 운영 능력에 있다.

## 주요 트렌드 관찰

1. memory, compression, browsing reach, skill security, sandbox runtime이 새로운 보조 인프라 층으로 급부상하고 있다.
2. 공식 릴리스와 GitHub 현장 수요, 외부 기사 서사가 모두 운영성과 안전성 중심으로 수렴하고 있다.
3. 기사량이 적은 제품일수록 언론 보도보다 공식 changelog를 더 세밀하게 읽어야 실제 방향을 놓치지 않는다.
