## 1. Claude Code

### 강점

- 2026-04-17과 2026-04-18 연속 릴리스로 네이티브 바이너리 실행, 도메인 차단 규칙, 서브에이전트 실패 처리, 리뷰 시작 병렬화, 권한 대화상자 크래시 수정까지 실제 운영면 개선이 빠르게 이어졌다.
- 최근 해설 글들도 Claude Code의 경쟁력이 모델보다 하네스, 메모리, 권한 경계, self-healing loop에 있다고 본다.
- 규칙 파일, 승인 흐름, 검증 루프를 직접 설계하고 싶은 팀에게 여전히 가장 선명한 선택지다.
- 터미널 중심 작업, 리팩터링, 테스트, 반복 실행, 리뷰 자동화 같은 텍스트 주도 업무와 잘 맞는다.

### 약점

- 시각 작업면이나 브라우저 중심 설명 경험은 Cursor나 Codex에 비해 상대적으로 덜 전면적이다.
- 릴리스 속도가 빠른 만큼 운영 정책과 권한 규칙을 따라잡는 비용이 있다.
- 잘 쓰려면 사용자가 워크플로를 직접 설계해야 하므로 진입 장벽이 낮지는 않다.

### 추천 사용 시나리오

- 로컬 규칙과 승인 정책을 강하게 적용해야 하는 팀
- 장기 작업을 단계별로 나누고, 테스트와 리뷰를 반복 루프로 돌려야 하는 환경
- 하네스와 메모리 구조를 직접 통제하고 싶은 고급 사용자

### 피해야 할 사용 시나리오

- 시각적 결과 비교와 브라우저 상호작용이 가장 중요한 프론트엔드 중심 협업
- 모든 것을 GUI 안에서 끝내고 싶고 정책 설계 비용을 최소화해야 하는 팀

### 최근 업데이트 반영 관점

- Claude Code의 최근 메시지는 “새로운 큰 기능”보다 “오래 돌아가는 세션을 덜 깨지게 만들기”다.
- 이 방향은 화려하지 않지만 실제 팀 생산성에는 가장 직접적인 영향을 준다.

### 현재 시점 총평

Claude Code는 이번 주에도 가장 강한 실행 엔진에 가깝다. 특히 규칙, 검증, 승인, 장기 작업 재개를 중요하게 보는 팀이라면 여전히 가장 설득력 있다.

## 2. Cursor

### 강점

- 공식 신규 항목은 없었지만, Canvases와 최근 연구 글이 모두 Cursor의 초점을 “병렬 에이전트 작업면”에 둔다는 점을 다시 보여준다.
- 500개 기업 데이터 연구 글은 더 나은 모델이 더 복잡한 작업, 더 많은 문서화, 더 많은 코드 리뷰를 유도했다고 제시한다.
- Canvases는 에이전트 결과를 표, 차트, 박스, diff, to-do 형태로 남기며 결과 비교와 설명에 강하다.
- 멀티에이전트 연구 글은 Cursor가 제품 서사를 IDE 보조에서 오케스트레이션 하네스로 확장하고 있다는 점을 강화한다.

### 약점

- 이번 공식 업데이트 창 안에는 새 제품 릴리스가 없어서 단기 변화량은 상대적으로 약하다.
- 제품 가치가 점점 “에이전트 작업면”으로 이동하면서, 전통 IDE 감각을 선호하는 사용자에게는 전환 비용이 생길 수 있다.
- 하부 모델 품질과 비용 구조는 여전히 외부 모델 공급망의 영향을 크게 받는다.

### 추천 사용 시나리오

- 여러 에이전트 결과를 한 화면에서 비교하고 설명해야 하는 제품/프론트엔드 팀
- 코드 변경뿐 아니라 문서, 시각화, 검수 흐름이 함께 중요한 조직
- 작업 분배와 결과 조율을 UI 중심으로 운영하고 싶은 사용자

### 피해야 할 사용 시나리오

- 강한 텍스트 기반 통제와 로컬 규칙 파일만으로 모든 것을 관리하려는 환경
- 최신 IDE 감각보다 안정적이고 단순한 CLI 중심 흐름이 더 중요한 팀

### 최근 업데이트 반영 관점

- Cursor는 여전히 “에이전트를 어디서 어떻게 보여주고 관리할 것인가”를 푸는 제품이다.
- 이번 주 공식 변화량은 적지만, 방향성 자체는 가장 일관되게 유지되고 있다.

### 현재 시점 총평

Cursor는 지금 가장 작업면 지향적인 도구다. 실행 엔진만 따로 보면 다른 선택지가 있을 수 있지만, 병렬 에이전트 관리와 시각적 산출물 경험에서는 여전히 독보적인 위치가 있다.

## 3. OpenAI Codex

### 강점

- 2026-04-16 제품 글과 직후 도움말 갱신이 함께 이어지면서, Codex가 앱, CLI, IDE, 웹, 클라우드 위임을 묶는 넓은 작업 시스템으로 설명되고 있다.
- 최근 문서 갱신은 플랜 포함 범위, 2x rate limits, Codex seat, 토큰 기반 요금 체계까지 빠르게 정리했다.
- 컴퓨터 사용, 메모리, 반복 작업, 브라우저, 이미지 생성, PR 리뷰 등 기능 범위가 가장 넓다.
- 상용 제품뿐 아니라 OpenAI Agents SDK 같은 오픈소스 인프라도 같이 성장하고 있어 생태계 확장성이 높다.

### 약점

- 기능 표면이 넓어진 만큼 어떤 표면에서 무엇을 해야 하는지 이해 비용도 빠르게 커진다.
- 로컬, 앱, IDE, 클라우드, 좌석 정책, 크레딧 구조를 함께 봐야 해서 단순한 도구로 느끼기 어렵다.
- 엄격하고 좁은 텍스트 기반 워크플로를 선호하는 사용자에게는 과하게 넓고 복합적으로 보일 수 있다.

### 추천 사용 시나리오

- 로컬 작업과 클라우드 위임을 섞어 쓰고 싶은 팀
- 개발, 리뷰, 문서, 프론트엔드 반복, 자동화를 한 플랫폼으로 묶고 싶은 조직
- 도구 선택보다 서비스 계층 통합이 더 중요한 환경

### 피해야 할 사용 시나리오

- 복잡한 권한/요금/좌석 구조를 다루기 어려운 작은 팀
- 넓은 작업면보다 단일 CLI 흐름의 예측 가능성을 더 중요하게 보는 사용자

### 최근 업데이트 반영 관점

- Codex는 이번 주에 “코딩 에이전트”에서 “조직 단위 작업 시스템”으로 서술 범위를 더 넓혔다.
- 도움말과 관리 문서가 빠르게 따라오고 있다는 점은 상용화와 확장 속도가 빠르다는 신호다.

### 현재 시점 총평

Codex는 가장 빠르게 넓어지고 있다. 특히 앱, 로컬, 웹, 클라우드, 좌석 정책을 함께 운영할 수 있는 팀이라면 가장 큰 확장성을 제공한다. 반대로 단순함을 원할수록 부담도 커진다.

## 이번 주 총평

이번 주에도 승부는 모델 이름이 아니라 운영 방식에서 갈렸다. Claude Code는 실행과 검증을 더 촘촘하게 다듬었고, Cursor는 병렬 작업면을 더 분명하게 밀고 있으며, Codex는 조직 배치와 기능 범위를 동시에 넓히고 있다.

GitHub Trending도 같은 신호를 보였다. 상위권에는 스킬 프레임워크, 메모리 시스템, 브라우저 MCP, SRE 자동화, 멀티에이전트 조직도가 올랐다. 즉, 지금 중요한 질문은 “어느 모델이 더 똑똑한가”보다 “에이전트를 어떤 절차와 도구 조합으로 운영할 것인가”에 더 가깝다.

## 주요 트렌드 관찰

1. 스킬 팩과 운영 하네스가 모델 성능만큼 중요한 차별화 축이 됐다.
2. 메모리, self-healing loop, 승인 규칙, 브라우저 도구 체인이 장기 실행 에이전트의 기본 조건으로 올라왔다.
3. 작업면과 실행 엔진이 분리되면서, 하나의 도구보다 조합 전략이 더 현실적인 선택지가 되고 있다.
4. 개발 자동화 다음 단계로 SRE, incident response, 운영 도메인이 빠르게 부상하고 있다.
