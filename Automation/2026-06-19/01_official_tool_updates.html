요약

2026-06-19 기준 최신 공식 신호는 세 제품 모두 “더 오래 실행되는 작업”과 “운영 제어” 쪽으로 무게가 이동하고 있다는 점이다. Cursor는 자동화와 클라우드 워커를 제품 전면으로 끌어올렸고, Codex는 브라우저/컴퓨터 사용과 재사용 가능한 스킬 생성 기능을 넓혔으며, Claude Code는 화려한 기능 추가보다 장기 세션, 권한, 파일 쓰기 신뢰성을 다지는 릴리즈를 이어갔다.

확인 불가 항목

Claude Code의 최신 changelog 맨 위 항목인 2.1.181은 버전과 변경 내용은 공식 원문으로 확인되지만, raw changelog 자체에는 해당 항목의 게시일이 적혀 있지 않았다. 따라서 이 보고서에서는 “2026-06-19 확인 시점의 최신 공식 항목”으로만 표기한다.

Claude Code

최신 공식 항목은 2.1.181, 초점은 세션 신뢰성과 파일 쓰기 안정화

공식 changelog 2026-06-19 확인 게시일 확인 불가

Anthropic의 raw changelog 최상단 최신 항목은 2.1.181이다. 이번 항목에는 프롬프트에서 직접 설정을 바꾸는 /config key=value, macOS Apple Events 샌드박스 예외 옵션, CLAUDE_CLIENT_PRESENCE_FILE, Bun 1.4 업그레이드, 장문 스트리밍 개선, 자동 재시도 강화, 서브에이전트 패널 정리, MCP OAuth 브라우저 경험 개선이 포함됐다.

특히 실무적으로 중요한 부분은 “네트워크 드라이브·클라우드 동기화 폴더에서 0바이트 또는 잘린 파일이 생기던 문제 수정”, “느린 네트워크에서 시작 시 공백 터미널로 멈추는 문제 수정”, “유휴 장기 세션 히스토리 손실 수정”이다. 즉 새 모델 과시보다 실제 작업 루프의 파손점을 줄이는 쪽이 이번 공식 메시지의 중심이다.

2.1.179 2.1.178 운영 안정성

직전 공식 항목 2.1.179는 mid-stream connection drop 보존, WSL2 스크롤 복구, 원격 세션 백그라운드 작업 표시 개선에 초점이 있었다. 2.1.178은 파라미터 기반 권한 규칙 문법과 nested skills 로딩, auto mode의 서브에이전트 검토 강화까지 포함했다.

이 흐름을 합치면 Claude Code는 2026년 6월 중순 현재 “에이전트가 더 멀리 가도록 하는 기능” 못지않게 “멈추지 않고, 잘못 쓰지 않고, 권한을 설명 가능하게 유지하는 것”을 릴리즈 핵심으로 삼고 있다.

Cursor

2026-06-18: Automations를 전면 기능으로 승격

공식 changelog 2026-06-18 자동화

Cursor의 최신 공식 항목은 2026-06-18 “Improvements to Cursor Automations”다. 로컬 에이전트 세션에서 바로 자동화를 만드는 /automate 스킬, Slack 이모지 트리거, GitHub 이슈/리뷰/워크플로 완료 트리거, 그리고 automation용 computer use가 추가됐다.

의미는 분명하다. Cursor는 IDE 안에서 코드를 도와주는 제품을 넘어서, GitHub와 Slack 이벤트를 받아 항상 켜져 있는 에이전트가 후속 작업을 수행하는 오케스트레이션 제품으로 확장 중이다.

2026-06-17: 클라우드 개발환경과 클라우드 서브에이전트 강화

공식 changelog 2026-06-17 클라우드 워커

하루 전 항목은 cloud environment setup과 /in-cloud 서브에이전트다. Cursor는 10분 이하의 클라우드 환경 준비, 재사용 가능한 환경 스냅샷, 별도 VM/브랜치에서 돌아가는 클라우드 서브에이전트, 로컬-클라우드 handoff를 전면에 배치했다.

이제 Cursor의 가치 제안은 “에디터 안 자동완성”보다 “로컬 세션은 가볍게 두고 긴 작업은 클라우드 워커가 붙잡는 구조”에 더 가까워졌다.

2026-06-10: Bugbot은 여전히 리뷰 자동화 ROI의 핵심 카드

공식 changelog 2026-06-10 코드 리뷰

2026-06-10 공식 항목에서 Cursor는 Bugbot 평균 리뷰 시간을 약 5분에서 90초 수준으로 줄이고, 리뷰당 발견 버그 수를 0.56에서 0.62로 늘렸으며, 비용은 약 22% 절감했다고 밝혔다. 여기에 /review, /review-bugbot, /review-security 흐름을 GitHub/GitLab PR과 연결했다.

즉 Cursor는 이번 주 “자동화 오케스트레이션”을, 지난주에는 “리뷰 에이전트 경제성”을 밀었다. 이 둘이 합쳐져 제품의 실제 매출 포인트를 만든다.

OpenAI Codex

2026-06-18: Record & Replay로 시연한 작업을 재사용 가능한 스킬로 전환

공식 changelog 2026-06-18 Codex app 26.616

Codex의 최신 공식 앱 릴리즈는 2026-06-18, app 26.616이다. 핵심은 macOS용 Record & Replay다. 사용자가 한 번 시연한 워크플로를 재사용 가능한 스킬로 바꾸며, bulk actions로 automation run history를 한꺼번에 읽음 처리하거나 보관할 수도 있다.

이 업데이트는 Codex가 단순히 명령을 수행하는 것을 넘어, 사람이 보여 준 절차를 내재화해 재사용 자산으로 만드는 방향을 강화한다.

2026-06-16: EEA·영국·스위스까지 Computer Use, Chrome extension, Memories, Chronicle 확대

공식 changelog 2026-06-16 지역 확장

2026-06-16 항목에서 Codex는 EEA, 영국, 스위스 사용자를 대상으로 Computer Use, Codex Chrome extension, Memories, Chronicle을 확장했다. Chronicle은 macOS용 opt-in research preview로, 최근 화면 맥락에서 기억을 만드는 실험 기능으로 설명된다.

즉 Codex는 브라우저/데스크톱 조작과 기억 축적을 더 넓은 지역에서 제공하며, “한 번성 답변”보다 “지속되는 작업 상태”를 제품 기본값으로 만들고 있다.

2026-06-11: Browser use Developer mode와 Windows 제어 강화

공식 changelog 2026-06-11 디버깅/제어

2026-06-11 app 26.609에는 Browser use Developer mode가 추가됐다. Chrome과 Codex 인앱 브라우저에서 CDP 기반 접근을 통해 네트워크 트래픽, 콘솔 출력, 런타임 오류, 페이지 상태까지 더 깊게 디버깅할 수 있다. 같은 릴리즈에는 /init, Windows per-app access controls, Enterprise용 Computer Use, Browser use 최대 2배 속도 개선도 포함됐다.

OpenAI가 Codex를 브라우저 디버깅과 컴퓨터 조작이 가능한 범용 작업 플랫폼으로 확장한다는 방향이 가장 명확하게 드러난 항목이다.
