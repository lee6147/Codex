Claude Code

2.1.158 / 2.1.157 / 2.1.154

핵심: Claude Code는 Opus 4.8, dynamic workflows, 자동 모드 범위 확대, 플러그인 초기화/자동 로딩, worktree와 background agent 안정성 개선을 연속 업데이트했습니다.

  • 2026-05-30 2.1.158: Bedrock, Vertex, Foundry에서 Opus 4.7/4.8 자동 모드를 opt-in으로 제공.
  • 2026-05-29 2.1.157: .claude/skills 플러그인 자동 로딩, claude plugin init, claude agents 설정 존중, worktree 전환 개선.
  • 2026-05-28 2.1.154: Opus 4.8 기본 high effort, dynamic workflows, 수십~수백 background agent 오케스트레이션, /simplify 정리 전용 리뷰, MCP pending approval 표시 개선.

운영 해석: Claude Code는 단일 터미널 보조도구에서 background agent/workflow 런타임으로 빠르게 이동하고 있습니다. 다만 조직 적용 시 worktree 격리, sandbox 승인, background 세션 정리 정책을 먼저 점검해야 합니다.

Cursor

3.6 Auto-review Run Mode

핵심: Cursor는 2026-05-29에 Auto-review Run Mode를 공개했습니다. 목표는 더 긴 agent 실행을 더 적은 승인 프롬프트와 더 안전한 실행 정책으로 처리하는 것입니다.

  • Shell, MCP, Fetch 도구 호출에 적용됩니다.
  • allowlist 호출은 즉시 실행되고, sandbox 가능한 호출은 sandbox에서 실행됩니다.
  • 나머지 agent 액션은 classifier subagent가 허용, 다른 접근, 사용자 승인 요청 중 하나를 판단합니다.
  • 설정 위치는 Cursor Settings > Agents > Run Mode입니다.

운영 해석: Cursor도 승인 비용을 낮추는 방향으로 이동하지만, 완전 자동화가 아니라 classifier 기반 안전 게이트를 앞세웁니다. 팀 환경에서는 allowlist와 classifier instruction 품질이 실제 안정성을 좌우합니다.

공식 출처: Cursor Changelog

OpenAI Codex

0.135.0 릴리스

핵심: GitHub 릴리스 기준 Codex의 최신 공식 태그는 2026-05-28의 0.135.0입니다. 지원 진단, 원격 연결 상태, Vim 모드, 권한 프로필, Python SDK sandbox preset, 비대화형 설치가 중심입니다.

  • codex doctor가 환경, Git, 터미널, app-server, thread inventory 진단을 더 자세히 보고합니다.
  • /status가 remote transport 연결 세부 정보와 서버 버전을 표시합니다.
  • /permissions가 named permission profile과 custom profile 표시를 지원합니다.
  • Python SDK가 thread/turn API용 Sandbox preset을 제공합니다.
  • install.sh/install.ps1가 CODEX_NON_INTERACTIVE=1 비대화형 설치를 지원합니다.

운영 해석: 이번 Codex 업데이트는 대형 기능보다 진단/권한/설치/SDK 표면을 다지는 성격입니다. 반복 자동화와 지원 케이스에서는 codex doctor 출력 품질이 특히 중요합니다.
