Cursor

Cursor SDK가 2026-04-28 공식 출시

Cursor Changelog 2026-04-28 확인 완료

Cursor는 2026-04-28에 SDK를 공개하며, 데스크톱 앱·CLI·웹 앱을 구동하는 동일 런타임과 하네스를 코드에서 직접 호출할 수 있게 했다. 몇 줄의 TypeScript로 로컬 또는 Cursor 클라우드 VM에서 에이전트를 띄우고, 같은 에이전트를 Agents Window나 웹 앱으로 이어받는 그림이 공식화됐다.

특히 durable agents, run-scoped follow-up과 streaming, archive/unarchive, self-hosted worker와 local runtime까지 함께 설명한 점이 중요하다. Cursor는 더 이상 IDE 안의 보조 도구만이 아니라 조직용 프로그래매틱 에이전트 플랫폼 쪽으로 발을 넓히고 있다.

Cursor Changelog 2026-04-24 창 밖 참고

Symphony가 Codex 오케스트레이션을 공개 스펙으로 밀어 올림

OpenAI Engineering 2026-04-27 확인 완료

OpenAI는 2026-04-27 Engineering 글에서 Symphony를 공개했다. 핵심은 Linear 같은 이슈 트래커를 코딩 에이전트의 control plane으로 삼아, 각 이슈마다 전용 작업공간과 에이전트를 붙이고, 중단되면 재시작하며, 병렬 의존성을 티켓 상태로 관리하는 오케스트레이션 방식이다.

글 안에서는 일부 팀에서 landed PR 수가 500% 늘었다고 설명하고, 운영 철학도 명확히 적는다. 세션을 인간이 직접 계속 조정하는 대신, 티켓 단위 목표와 워크플로 규약을 저장소 안에 두고 에이전트가 계속 움직이게 하자는 방향이다. 오늘 Codex의 가장 큰 공식 신호는 이 오케스트레이션 공개다.
