요약

지난 3일 공식 업데이트 기준으로는 OpenAI Codex가 가장 뚜렷하다. 2026-05-28 CLI 0.135.0과 2026-05-26 CLI 0.134.0이 연속으로 나왔고, 진단, 권한 프로필, 원격 연결, MCP, 로컬 대화 검색이 강화됐다. Claude Code는 공식 changelog 최신 항목에서 Opus 4.8, 동적 워크플로, fast mode, background agents 개선이 확인되지만 게시 날짜가 공식 문서에 노출되지 않아 `확인 불가`로 처리했다. Cursor는 최신 공식 changelog가 2026-05-20이라 이번 3일 창에는 새 항목이 없다.

OpenAI Codex

Codex CLI 0.135.0

확인 완료 공식 changelog 2026-05-28

지원 사례 분석을 위한 `codex doctor` 진단 범위가 넓어졌고, `/status`가 원격 연결 세부 정보와 서버 버전을 표시한다. Vim 모드의 텍스트 오브젝트 편집과 interrupt binding, `/permissions`의 named permission profile 표시, Python SDK의 `Sandbox` preset도 추가됐다.

실무 의미: 팀 단위 운영에서 장애 신고, 원격 세션 확인, 권한 프로필 점검이 쉬워진다. 특히 Windows/원격/자동화 환경에서 “무엇이 연결되어 있는지”를 설명하는 진단 정보가 늘어난 점이 중요하다.

발표일: 2026-05-28 | 출처: OpenAI Developers Codex changelog | URL: developers.openai.com/codex/changelog

Codex CLI 0.134.0

확인 완료 공식 changelog 2026-05-26

로컬 대화 기록 검색, CLI/TUI 전반의 `--profile` 중심 정리, MCP 서버별 환경 타깃과 OAuth 옵션, 커넥터 도구 스키마 보존, read-only MCP 도구 병렬 실행 지원이 추가됐다.

실무 의미: Codex를 장기 세션과 MCP 중심 워크플로에 붙여 쓰는 팀에서 재탐색 비용과 설정 혼선을 줄일 수 있다. read-only MCP 병렬 실행은 조사형 자동화에서 체감 성능을 높일 수 있다.

발표일: 2026-05-26 | 출처: OpenAI Developers Codex changelog | URL: developers.openai.com/codex/changelog

Claude Code

Claude Code 2.1.154 최신 changelog 항목

날짜 확인 불가 공식 GitHub changelog 게시일 미표기

공식 changelog 최신 항목은 Opus 4.8 기본 고노력 모드, 동적 워크플로, 더 저렴한 fast mode, lean system prompt 기본화, `/simplify`의 cleanup-only review, background shell session 실행, Chrome 연결 브라우저 선택, plugin 기본 비활성화 설정, MCP 표시 개선 등을 담고 있다.

실무 의미: Claude Code는 “단일 세션 보조자”에서 다수 background agents와 workflow orchestration 쪽으로 더 강하게 이동하고 있다. 다만 공식 changelog 원문에 항목별 날짜가 없어 이번 3일 창에 포함되는지는 확인 불가로 표시한다.
