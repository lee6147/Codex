주목 저장소

1. cocoindex-io / cocoindex

Python 8,339 stars 오늘 +434

cocoindex는 스스로를 Incremental engine for long horizon agents라고 정의한다. 즉 한 번의 답변보다 오래 이어지는 에이전트 세션을 중심에 두는 설계다.

지금 주목할 이유: 장기 작업, 재실행, 상태 유지가 생산성 경쟁의 핵심 축이 되고 있음을 가장 직접적으로 보여 준다.

연결점: Codex의 automations, Claude Code의 장기 CLI 세션, Cursor의 Cloud Agents와 같은 흐름과 맞닿는다.

저장소: github.com/cocoindex-io/cocoindex | 한줄 총평: “긴 작업을 버티는 실행 엔진” 수요가 메인스트림으로 올라왔다.

2. LearningCircuit / local-deep-research

Python 5,096 stars 오늘 +200

local-deep-research는 로컬과 클라우드 LLM을 모두 쓰면서 검색 엔진, 사설 문서, 암호화 저장까지 묶는 에이전트형 리서치 런타임이다.

지금 주목할 이유: 단순 질의응답이 아니라 “도구를 붙인 장기 조사 에이전트”가 별도 카테고리로 굳고 있다.

연결점: 코딩 에이전트도 결국 브라우징, 문서, 비공개 자료 연결이 중요하다는 점에서 Claude Code, Cursor, Codex의 도구 연결 경쟁과 이어진다.

저장소: github.com/LearningCircuit/local-deep-research | 한줄 총평: 리서치형 에이전트도 이제 로컬 보안과 장기 상태를 기본 전제로 한다.

3. ruvnet / ruflo

TypeScript 43,387 stars 오늘 +2,441

ruflo는 Claude용 multi-agent orchestration platform을 전면에 내세운다. swarms, autonomous workflows, RAG, Claude Code / Codex integration을 한 덩어리로 설명한다.

지금 주목할 이유: 시장의 관심이 “좋은 단일 에이전트”에서 “역할을 나눠 오래 굴리는 플랫폼”으로 이동했다는 점이 수치로 드러난다.

연결점: Cursor의 Agents Window, Codex의 멀티에이전트 앱/오토메이션, Claude Code의 sub-agent 흐름과 직접 연결된다.

저장소: github.com/ruvnet/ruflo | 한줄 총평: 에이전트 운영층이 독립 제품 범주로 굳었다.

4. mksglu / context-mode

TypeScript 12,956 stars 오늘 +344

context-mode는 AI coding agents의 context window optimization을 전면에 둔다. tool output을 샌드박싱해 토큰 낭비를 줄인다는 메시지가 선명하다.

지금 주목할 이유: 대형 리포에서 실제 성능을 가르는 것은 모델 IQ보다 컨텍스트 유지력이라는 현장 문제를 정확히 찌른다.

연결점: Claude Code, Cursor, Codex 모두 장기 세션에서 compaction과 context decay를 겪기 때문에 직접적인 레퍼런스가 된다.

저장소: github.com/mksglu/context-mode | 한줄 총평: 컨텍스트 관리가 이제 보조 기능이 아니라 핵심 운영 레이어다.

5. czlonkowski / n8n-mcp

TypeScript 20,087 stars 오늘 +335

n8n-mcp는 Claude Desktop, Claude Code, Windsurf, Cursor에서 n8n 워크플로를 만들게 하는 MCP 서버다. 외부 자동화 시스템과 코딩 에이전트를 붙이는 연결면 자체가 주목받고 있다.

지금 주목할 이유: 생산성 차이는 더 이상 모델만으로 나지 않는다. 실제 업무 시스템과 연결되는 정도가 훨씬 중요해졌다.

연결점: 코드 에이전트를 사내 자동화, 워크플로 엔진, 알림 시스템까지 넓히고 싶은 팀에 직접적인 시사점이 있다.

저장소: github.com/czlonkowski/n8n-mcp | 한줄 총평: MCP는 이제 “좋으면 쓰는 옵션”이 아니라 업무 연결의 표준 언어에 가깝다.

6. Hmbown / DeepSeek-TUI

Rust 6,966 stars 오늘 +2,389

DeepSeek-TUI는 터미널에서 도는 coding agent를 전면에 둔다. 여전히 많은 사용자가 GUI보다 셸 안에서 길게 이어지는 작업 흐름을 선호한다는 뜻이다.

지금 주목할 이유: CLI형 에이전트 작업면은 검증 가능성, 속도, 복구성 면에서 아직 강한 수요를 유지한다.

연결점: Claude Code와 Codex CLI가 강한 이유를 보여 주는 외부 시장 신호다.

저장소: github.com/Hmbown/DeepSeek-TUI | 한줄 총평: 터미널형 에이전트 경험은 여전히 확실한 시장 축이다.

7. 1jehuang / jcode

Rust 4,183 stars 오늘 +432

jcode는 설명 자체가 Coding Agent Harness다. 규칙, 검증, 반복 실행, 리뷰 가능한 변경이라는 하네스 자산이 독립적인 가치가 되고 있다는 신호다.

지금 주목할 이유: 모델은 바뀌어도 하네스는 남는다. 팀 경쟁력이 모델 선택보다 실행 계약에 더 많이 쌓이고 있다.

연결점: 로컬 계약과 검증 루프를 먼저 두는 Codex/Claude Code/Cursor 운영 방식과 잘 맞는다.

저장소: github.com/1jehuang/jcode | 한줄 총평: “하네스가 곧 제품”이라는 흐름이 더 이상 특이하지 않다.
