공식 AI 코딩 도구 업데이트

생성일: 2026-06-03 · 조사 범위: 공식 업데이트 2026-06-01 ~ 2026-06-03 · 대상: Claude Code, Cursor, OpenAI Codex
핵심 이번 공식 업데이트의 중심은 장시간 에이전트 운용의 안정성과 운영 통제다. OpenAI Codex는 0.136.0에서 원격 실행, 아카이브, Windows 샌드박스, 앱 서버 기능을 넓혔다. Claude Code 2.1.160은 셸 시작 파일과 빌드 설정 파일 쓰기 전 확인을 강화했고, Cursor는 strict 3일 범위에는 새 공식 릴리스가 없어서 2026-05-20의 3.5 업데이트를 인접 공식 맥락으로만 기록했다.

목차

1. Claude Code 2. Cursor 3. OpenAI Codex 출처

1. Claude Code

공식일자 확인 제한

Claude Code 2.1.160

발표일: 공식 changelog에는 날짜가 직접 노출되지 않는다. 보조 검증 기사 기준으로 v2.1.160의 사용자 영향 변경은 2026-06-01 릴리스로 정리되어 있다.

  • 주요 변화: 셸 시작 파일, ~/.config/git/, 빌드 도구 설정 파일에 쓰기 전에 확인 프롬프트를 추가했다. acceptEdits 모드에서도 코드 실행 가능 설정 파일은 자동 승인되지 않는다.
  • 운영 의미: 장시간 에이전트가 권한이 높은 설정 파일을 수정하는 위험을 낮춘다. 특히 팀 환경에서 자동 수정 권한을 열어 둔 경우 안전장치가 더 중요해졌다.
  • 실무 영향: 단일 파일을 grep으로 확인한 뒤 별도 Read 없이 Edit가 가능해져 반복 수정 흐름은 조금 짧아졌다.
  • 확인 상태: 공식 changelog 내용은 확인 완료, 공식 발표일은 확인 불가.

2. Cursor

strict 3일 내 신규 없음

Cursor 3.5: Shared canvases, /loop, Automations

발표일: 2026-05-20. 2026-06-01 ~ 2026-06-03 공식 changelog에서 새 항목은 확인되지 않았다.

  • 주요 변화: 공유 캔버스, /loop 스킬, Agents Window 안의 Automations, multi-repo/no-repo automation을 제공한다.
  • 운영 의미: Cursor는 IDE 안의 일회성 코딩 보조를 넘어, 반복 실행과 팀 공유 가능한 에이전트 산출물 쪽으로 방향을 잡고 있다.
  • 실무 영향: 여러 저장소를 동시에 보는 자동화와 외부 도구 모니터링은 백엔드, 데이터, 운영 팀의 반복 업무에 유용하다.
  • 확인 상태: 공식 changelog 확인 완료. 단, 이번 strict 3일 공식 업데이트에는 포함하지 않고 인접 맥락으로만 사용한다.

3. OpenAI Codex

공식 확인 완료

Codex CLI 0.136.0

발표일: 2026-06-01.

  • 주요 변화: TUI 링크 렌더링, 세션 아카이브, app-server --stdio, 원격 실행 API 키 등록, Windows 샌드박스 setup 경로, 독립 이미지 생성 확장 경로가 추가됐다.
  • 운영 의미: Codex는 단일 CLI 보조 도구보다, 데스크톱 앱, 원격 제어, 앱 서버, Bedrock 환경까지 이어지는 운영형 에이전트 플랫폼 성격이 강해지고 있다.
  • 보안 및 안정성: ChatGPT auth 갱신, 명령 안전성, sandbox cleanup, Bedrock 인증 fallback 등이 보강됐다.
  • 확인 상태: GitHub 공식 릴리스 확인 완료.

이번 업데이트 해석

세 도구 모두 더 긴 작업 시간, 더 많은 자동화, 더 넓은 실행 권한을 다루는 방향으로 움직이고 있다. 따라서 이번 주 실무 체크포인트는 새 기능 자체보다 권한 경계, 재실행 방지, 작업 이력 보존, 운영자가 검토할 수 있는 산출물 구조다.
