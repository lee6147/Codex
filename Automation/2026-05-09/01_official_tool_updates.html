핵심 요약

이번 3일 공식 업데이트의 중심은 “병렬 에이전트 운영”과 “운영 안정성”이다. Cursor는 PR 리뷰와 계획 기반 병렬 빌드를 전면에 냈고, Codex는 Chrome 확장과 CLI 0.129.0으로 브라우저·터미널 작업면을 넓혔다. Claude Code는 빠른 CLI 패치 흐름 속에서 worktree 기준, 프록시/MCP OAuth, 원격 제어 중단, Windows/네트워크 드라이브 권한 같은 실제 운영 마찰을 줄였다.

Claude Code

Claude Code 2.1.133

확인 완료공식 GitHub 원문

  • 무엇이 바뀌었나: worktree.baseRef 설정이 추가되어 --worktree, EnterWorktree, 에이전트 격리 worktree의 분기 기준을 origin 또는 로컬 HEAD 중 선택할 수 있다.
  • 운영 의미: 병렬 에이전트가 새 worktree를 만들 때 미푸시 커밋을 포함할지 여부를 정책으로 고정할 수 있어, 팀 단위 자동화에서 재현성과 누락 방지를 같이 관리하기 쉬워졌다.
  • 추가 포인트: MCP OAuth에서 프록시와 mTLS 적용 범위를 넓혔고, 원격 제어 중단이 로컬 Esc와 더 가깝게 동작하도록 수정했다.

Claude Code 2.1.132

확인 완료터미널 안정성

  • 무엇이 바뀌었나: Bash subprocess 환경에 CLAUDE_CODE_SESSION_ID가 추가되고, fullscreen alternate-screen 렌더러를 끄는 환경변수가 제공됐다.
  • 운영 의미: 훅, 로그, Bash 실행을 같은 세션 ID로 연결하기 쉬워져 자동화 감사 추적이 좋아진다.
  • 위험 완화: 외부 SIGINT, SSH 종료, Windows/IDE 터미널 입력, MCP 서버 과출력 같은 장시간 세션 장애를 줄이는 수정이 포함됐다.

Cursor

Cursor 3.3: PR Review, Build Plan in Parallel, Split PRs

2026-05-07공식 changelog

  • 무엇이 바뀌었나: PR 리뷰 탭에서 인라인 리뷰 스레드와 상위 PR 댓글을 보고, Commits/Changes 탭으로 커밋·변경 파일을 탐색할 수 있다.

  • 실무 의미: Cursor가 단순 IDE 보조에서 PR 단위 검토·분할·병렬 빌드 워크플로로 이동하고 있다.

Codex CLI 0.129.0

2026-05-07공식 Codex changelog

  • 주요 기능: TUI composer에 modal Vim 편집이 들어갔고, resume/fork picker, raw scrollback mode, /ide context injection, workspace-aware /diff가 개선됐다.
  • 운영 기능: status line에 테마 인식 색상, PR 요약, branch-change 요약을 표시할 수 있고, plugin management와 hooks 관리가 확대됐다.
  • 수정 영역: tmux 복사, Windows 입력/붙여넣기 지연, Linux/Windows sandbox, custom CA login, MCP/hook 출력 증가 문제가 다뤄졌다.
