1. Claude Code

확인 완료2.1.131

안정성 중심의 최신 릴리스

공식 GitHub 릴리스에서 2.1.131 버전이 최신으로 확인됩니다. 릴리스 노트는 압축되어 있어 세부 기능보다 버전 흐름과 패치 반영 여부 확인에 적합합니다.

실무 의미: 팀 표준 설치 버전을 고정하는 경우 2.1.131을 기준으로 재현성을 맞추는 것이 좋습니다.

발표일: 공식 페이지에서 상대 시간만 확인됨. 확인일: 2026-05-07.

확인 완료2.1.129

플러그인 URL 설치 흐름 개선

최근 릴리스 흐름에서 플러그인 URL 설치와 관련한 개선이 확인됩니다. Claude Code를 사내 확장·워크플로 자동화와 묶어 쓰는 팀에는 배포 편의성과 온보딩 안정성에 영향을 줍니다.

주의: 플러그인 배포는 신뢰 경계가 넓어지므로 내부 허용 목록과 설치 경로 검증이 필요합니다.

2. Cursor

확인 완료2026-05-06

Cursor 3.3: 컨텍스트 사용량 표시

Cursor 3.3은 에이전트가 사용 중인 컨텍스트를 드롭다운으로 보여주는 기능을 추가했습니다. 긴 작업에서 모델이 왜 느려졌는지, 어떤 컨텍스트가 비용을 쓰는지 파악하기 쉬워집니다.

실무 의미: 대형 리포지토리에서 불필요한 파일 주입을 줄이고, 에이전트 세션을 끊을 타이밍을 판단하는 데 유용합니다.

확인 완료2026-05-06

에이전트 터미널 제어와 병렬성 개선

에이전트가 여러 터미널을 병렬로 만들고, 각각의 작업을 추적할 수 있게 된 점이 핵심입니다. 자동 테스트·서버 실행·로그 감시를 동시에 돌리는 개발 흐름에 맞습니다.

관리 포인트: 병렬 실행은 빠르지만 실패 원인 분리가 어려워질 수 있으므로 명령 로그와 종료 상태를 함께 남겨야 합니다.

3. OpenAI Codex

확인 완료0.129.x alpha

alpha 릴리스가 2026-05-04 ~ 2026-05-06에 집중

OpenAI Codex GitHub 릴리스에서 0.129.0-alpha.6부터 0.129.0-alpha.10까지의 프리릴리스 흐름이 확인됩니다. 자동화·검증 환경에서는 alpha 채널을 운영 표준으로 쓰기보다 별도 실험 브랜치에서 확인하는 편이 안전합니다.

실무 의미: CLI 동작, 샌드박스, 도구 호출 계약이 바뀔 수 있으므로 고정 버전과 회귀 체크가 필요합니다.

기준선0.128.0

안정판 기준선은 2026-04-30 릴리스

최근 3일 안의 안정판 릴리스는 확인되지 않았습니다. 오늘 기준 운영 안정성을 비교할 때는 0.128.0을 기준선으로 두고 alpha 흐름을 별도 관찰하는 것이 합리적입니다.

확인 상태: 공식 GitHub 릴리스 기준 확인 완료.

확인 불가 항목

  • Claude Code 일부 릴리스의 정확한 게시일은 공식 GitHub 화면에서 상대 시간 중심으로 확인되어, 보고서에는 확인일을 함께 표기했습니다.
  • OpenAI Codex alpha 릴리스별 세부 변경 내용은 릴리스 제목만으로는 충분히 분리되지 않아, 운영 판단은 alpha 채널 존재와 버전 흐름 중심으로 정리했습니다.
