# Follow-up

## 바로 실행할 것
- 현재 반복 업무를 `라우터`, `평가자`, `실행자`, `랩업` 4단계로 쪼개고 각 단계의 입력과 출력 한 줄 정의를 먼저 적습니다.
- 사람이 반드시 개입해야 하는 지점을 문서로 박습니다. 예를 들면 `최종 대외 발신`, `법무/개인정보 판단`, `파일럿 진행 여부 결정`은 사람 승인 없이는 넘기지 않도록 합니다.
- `합의된 사항 / 미결 사항 / 액션 아이템 / 다음 담당자` 네 칸으로 된 랩업 템플릿을 만들어 모든 단계 끝에 고정 출력하게 합니다.

## 같이 보면 좋은 영상
- 메모리 주입, 권한 제한, 검증 루프까지 실제로 보여주는 멀티에이전트 아키텍처 영상
- 실패 사례, 지연시간, 비용, 사람 검토 시간까지 공개하는 운영 회고 영상

## 다음 비교 후보
- `멀티에이전트 데모`와 `실제 운영 시스템`의 차이를 설명하는 영상
- `단일 에이전트 + 강한 하네스` 대 `다중 에이전트 오케스트레이션` 비교 영상

## 메모
- 새로움은 멀티에이전트 자체보다 `운영 흐름을 데모 서사로 잘 보여준 점`에 있습니다.
- 약한 부분은 실제 병렬성, 독립 컨텍스트 여부, 운영 비용과 실패율 같은 핵심 지표가 transcript만으로는 드러나지 않는다는 점입니다.
- 외부 팩트체크는 수행하지 않았습니다. 이번 평가는 사용자 제공 transcript와 역할별 내부 분석 결과를 기준으로 정리했습니다.
