LLM-Wiki란?

LLM-Wiki는 유튜브 대본, 슬라이드, 메모 같은 원본을 그대로 보관하면서, AI가 다시 읽고 답변에 재사용하기 쉬운 중간 지식 노트와 인덱스를 만드는 방식이다.

  • 원본은 Raw Sources에 남겨 검증 기준으로 둔다.
  • 핵심 설명은 Wiki Notes로 분리한다.
  • 에이전트는 Index를 통해 필요한 노트를 빠르게 찾는다.

이 화면에서 봐야 할 핵심

Obsidian스러운 느낌은 그래프의 예쁜 선이 아니라, 지식이 파일, 링크, 백링크, 검색 가능한 인덱스로 계속 움직이는 감각이다.

원본은 보관한다

요약만 남기지 않는다. 나중에 AI 답변이 맞는지 확인하려면 대본과 슬라이드 같은 Raw Source가 필요하다.

위키는 재사용한다

긴 대본을 매번 다시 읽히지 않고, 핵심 주장과 근거가 정리된 Wiki Note를 반복 질문의 출발점으로 삼는다.

그래프는 지도다

그래프는 이해를 돕는 지도다. 좋은 답변 자체는 원본, 정리 품질, 인덱스, 검증 절차가 함께 만들 때 나온다.

Obsidian 안에서 에이전트를 부르는 기본 흐름

초보자는 “Obsidian이 AI를 실행한다”고 이해하기 쉽지만, 실제로는 Obsidian 볼트 폴더를 터미널에서 열고 Codex나 Claude Code가 그 폴더의 Markdown을 읽게 하는 방식에 가깝다.

1. 볼트 폴더에서 터미널 열기

Obsidian의 볼트는 결국 Markdown 파일 폴더다. 그 폴더에서 터미널을 열어 에이전트를 실행한다.

PowerShell 예시 개념용 
cd "C:\Users\user\Documents\Obsidian Vault"

# Codex CLI 설치 예시
npm install -g @openai/codex
codex

# Claude Code 설치 예시
npm install -g @anthropic-ai/claude-code
claude

설치 명령은 2026-06-01 기준 확인한 예시다. 실제 인증, 요금제, 지원 OS는 각 공식 문서를 확인해야 한다.

2. 프롬프트는 “그래프를 읽는 순서”를 지정한다

그냥 “요약해줘”라고 하지 말고, 원본, 위키 노트, 인덱스, 백링크를 어떤 순서로 읽을지 알려줘야 LLM-Wiki답게 작동한다.

역할 너는 내 Obsidian 볼트의 LLM-Wiki 관리자다.
읽을 순서 먼저 20_schema_index를 보고 관련 노트를 찾은 뒤, 10_wiki_notes를 읽고, 필요한 경우 00_raw_sources로 근거를 확인해라.
출력 형식 답변은 핵심 요약, 근거가 된 노트, 추가로 만들어야 할 wikilink, 확인이 필요한 원본 항목으로 나눠라.

새 강의가 들어오면 볼트가 갱신된다

아래 흐름은 LLM-Wiki의 “살아 있는 노트” 성격을 보여준다. 새 자료가 들어오면 파일 하나가 추가되는 데서 끝나지 않고, 연결과 질문 경로가 같이 바뀐다.

Ingest

대본, 슬라이드, 메모를 Raw Sources에 넣는다.

Distill

AI가 핵심 주장, 근거, 관련 개념을 노트로 뽑는다.

Link

기존 노트와 새 노트 사이에 wikilink와 backlinks를 만든다.

Query

다음 질문부터는 정리된 위키와 인덱스를 먼저 읽는다.

실행 예시/wiki ingest transcript-part-01.md
결과Raw Source가 저장되고, LLM-Wiki란.md와 concept-index.base가 갱신된다.
주의이 명령은 개념 예시다. 실제 도구명은 사용 환경에 맞춰 바뀔 수 있다.
