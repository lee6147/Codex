영상이 실제로 말하는 것

영상의 메시지는 Markdown 폐기론이 아니라, 사람이 실제로 읽고 판단해야 하는 복잡한 AI 산출물에는 HTML artifact가 더 나을 수 있다는 주장이다.

핵심 긴 Markdown은 잘 안 읽힌다

AI가 만든 100줄 이상의 계획서나 스펙은 작성은 쉽지만 검토율이 낮아질 수 있다.

전환 HTML은 시각과 인터랙션을 담는다

색, 레이아웃, SVG, 슬라이더, 버튼, 드래그 카드 같은 요소로 판단 비용을 줄인다.

주의 권위 인용은 아직 검증 전

Anthropic 엔지니어 글과 Karpathy 트윗은 원문 확인 전까지 그대로 인용하면 안 된다.

Markdown과 HTML의 선택 기준

HTML-first가 항상 답은 아니다. 짧은 기록은 Markdown이 낫고, 판단해야 하는 복잡한 산출물은 HTML 실험 가치가 있다.

기준
Markdown 유지
HTML artifact 요청
문서 목적

가볍고 검색/버전관리가 쉽다. 원문 근거와 결정 기록을 오래 남기기 좋다.
정보 밀도, 시각적 구분, 인터랙션, 복사 버튼으로 다음 행동까지 빠르게 이어진다.
비용
긴 문서는 읽지 않고 넘길 위험이 있다. 표와 다이어그램 표현력이 제한된다.

추천 방식
결론과 근거는 Markdown으로 남긴다.
복잡한 판단 화면은 HTML로 만들고, 최종 결정은 짧은 Markdown으로 다시 기록한다.

바로 쓸 프롬프트

아래 문구는 follow-up 노트의 추천 프롬프트를 실험용으로 다듬은 것이다. 버튼을 누르면 클립보드에 복사된다.

복잡한 분석을 HTML로 바꾸기

이 내용을 Markdown 보고서가 아니라 한 파일짜리 HTML artifact로 만들어줘.
탭 구조, 위험도 색상, 비교표, 근거/미확인 주장 구분, copy prompt 버튼을 포함해줘.
민감 정보가 있으면 별도 섹션으로 표시하고 외부 공유 금지 문구를 넣어줘.

PR 리뷰 대시보드 만들기

이 PR을 reviewer가 빠르게 판단할 수 있는 HTML 대시보드로 설명해줘.
파일별 변경 의도, 위험도, 테스트 공백, 확인 질문, 승인 전 체크리스트를 포함해줘.

검증 전에는 믿지 말 것

이 HTML은 분석 Markdown을 시각화한 것이지, 외부 fact check를 끝낸 결과가 아니다. 권위 인용과 제품 능력 주장은 별도 확인이 필요하다.

인용 주의 자동 자막 기반 실무 팁은 실험 가능
  • Anthropic 또는 Claude Code 엔지니어 원문 글의 실제 작성자, 문맥, 문구 확인
  • Karpathy 트윗의 정확한 표현과 영상의 해석이 일치하는지 확인
  • HTML이 Markdown보다 토큰을 2-4배 더 쓴다는 수치 검증
  • 로컬 파일, git history, MCP 데이터 시각화가 어떤 권한과 보안 경계에서 가능한지 확인
  • S3나 링크 공유 시 내부 데이터가 HTML에 포함되지 않는지 확인

실험 설계

영상의 조언은 논쟁보다 실험으로 판단하는 편이 빠르다. 기존 긴 Markdown 하나를 골라 같은 입력으로 HTML 버전을 생성하고 아래 네 가지를 비교한다.

1 읽은 시간

처음부터 끝까지 실제로 검토하는 데 걸린 시간을 비교한다.

2 놓친 위험

Markdown에서는 지나쳤고 HTML에서는 발견한 위험이나 결정을 기록한다.

3 다음 액션 속도

복사 버튼, 체크리스트, 비교표가 실제 후속 작업을 줄였는지 본다.

4 보존 비용

git diff, 장기 보관, 접근성, 보안 공유 비용이 감당 가능한지 판단한다.

결론 둘 다 남기는 전략

복잡한 판단은 HTML, 최종 결정과 근거는 짧은 Markdown으로 남기는 조합이 가장 안전하다.
