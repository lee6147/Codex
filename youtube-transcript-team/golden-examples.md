# Golden Examples

이 파일은 정답지가 아니라 출력 톤과 밀도를 맞추기 위한 기준 예시다.

## Example 1

### 입력 상황
- 주제: "Codex와 Claude Code 비교"
- 스크립트 품질: 전체 스크립트
- 사용자의 관심사: 실무 적용, 툴 선택

### 기대되는 좋은 출력 특성
- 두 도구의 장단점을 길게 설명하기보다 `어떤 상황에서 누가 유리한지` 먼저 말한다.
- "직접 볼 가치"는 데모가 실제 비교 판단에 중요한지 기준으로 판정한다.
- 실행 액션은 `같은 태스크를 두 도구에 돌려 비교해라`처럼 바로 실험 가능한 형태여야 한다.
- 최신 기능 우열을 단정하지 않고 `검증 필요`를 남긴다.

### 피해야 할 출력
- "둘 다 좋다" 같은 무난한 결론
- 발표자 톤을 따라 과하게 흥분하는 문장
- 제품 최신 기능을 사실처럼 단정하는 문장

## Example 2

### 입력 상황
- 주제: "AI 에이전트로 개발 생산성 10배"
- 스크립트 품질: 일부만 있음
- 사용자의 관심사: 과장 여부, 실제 적용성

### 기대되는 좋은 출력 특성
- 입력 신뢰도와 판단 신뢰도를 낮게 시작한다.
- `10배` 같은 수치는 Skeptic Reviewer와 Fact Checker 관점에서 바로 경계한다.
- 실행 가능성은 높게 주기보다 `조건부` 또는 `낮음`으로 보수적으로 평가할 수 있어야 한다.
- 최종 판정에서 `요약만으로 충분` 또는 `필요한 구간만 시청`을 냉정하게 말할 수 있어야 한다.

### 피해야 할 출력
- 일부 스크립트만 보고 영상 전체를 확정 평가하는 것
- 측정 기준 없는 생산성 수치를 그대로 받아쓰는 것

## Example 3

### 입력 상황
- 주제: "MCP 실전 연결 데모"
- 스크립트 품질: 전체 스크립트
- 사용자의 이전 시청 기록: 입문용 MCP 영상 2개

### 기대되는 좋은 출력 특성
- `직접 볼 가치`를 높게 줄 수 있다. 이유는 화면 흐름과 연결 실수, 수정 과정이 중요하기 때문이다.
- Companion Recommender는 `입문용 -> 실전 데모용 -> 실패 사례용`처럼 순서를 제안해야 한다.
- Action Curator는 `작은 로컬 도구 하나부터 MCP로 연결해봐라`처럼 작은 실험 단위로 변환해야 한다.

### 피해야 할 출력
- 이전 영상을 모르는 척하고 일반론만 하는 것
- "MCP만 붙이면 된다" 식의 단순화
